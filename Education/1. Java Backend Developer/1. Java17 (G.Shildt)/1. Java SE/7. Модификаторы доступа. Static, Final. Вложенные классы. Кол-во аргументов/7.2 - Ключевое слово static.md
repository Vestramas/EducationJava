Ключевое слово `static` в Java — ==создает члены класса, принадлежащих самому классу==, а не объекту. 

> [!NOTE] ⭐`static` поля по-умолчанию НЕ СЕРИАЛИЗУЮТСЯ
> 
# 📕 0. Что можно и нельзя использовать В `static`-методе?

| Внутри `static` метода можно             | Внутри `static` метода нельзя                                                               |
| ---------------------------------------- | ------------------------------------------------------------------------------------------- |
| ✅ Вызывать другие `static`-методы        | ❌ Вызывать нестатические методы напрямую (только через создание объекта и вызов через него) |
| ✅ Использовать `static` поля             | ❌ Использовать нестатические поля напрямую                                                  |
| ✅ Создавать объекты классов              | ❌ Использовать `this` или `super`                                                           |
| ✅ Использовать `static` вложенные классы | ❌ Получить доступ к экземпляру объекта (если он не передан явно)                            |

## Можно ли переопределить `static` методы?
> [!NOTE] 🛑`НЕТ`! В Java `static` методы **не переопределяются**, а **скрываются (hidden)**:
> 

```java
class A {
    static void show() { System.out.println("A"); }
}
class B extends A {
    static void show() { System.out.println("B"); }
}

A obj = new B();
obj.show(); // 👉 "A", а не "B"
```
📌 Потому что ==выбор `static`-метода ==происходит ==**на этапе компиляции**==, а не во время выполнения (нет полиморфизма).

То есть у **static методов НЕТ ДИНАМИЧЕСКОГО СВЯЗЫВАНИЯ** (когда вызов метода определяется по типу объекта, а не по типу ссылки)
, **метод определяется ссылкой**

# 📕 1. **Общий порядок загрузки и инициализации класса и объекта в Java**

## 🔷 При первом использовании класса:
1. Статические поля (default-значения)
2. Статические поля (инициализация) **в порядке их объявления в коде** сверху вниз.
3. Статические блоки **в порядке их объявления в коде** сверху вниз.
## 🔶 При создании объекта:
4. Нестатические поля (default-значения)
5. Нестатические поля (инициализация) **в порядке их объявления в коде** сверху вниз.
6. Нестатические блоки **в порядке их объявления в коде** сверху вниз.
7. Конструктор

## 💡 Обобщённый порядок 💡

| Этап                | Что выполняется                                   |
| ------------------- | ------------------------------------------------- |
| 1. Загрузка класса  | Static-поля и static-блоки                        |
| 2. Создание объекта | Нестатические поля → instance-блоки → конструктор |

# 📕 2. Частые ошибки

| Ошибка                                                  | Объяснение                                                                         |
| ------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| Обращение к НЕстатической переменной из `static` метода | Нужно создать объект                                                               |
| Попытка переопределить `static` метод через `override`  | ==`static` методы НЕ ПОЛИМОРФНЫ== — **они не переопределяются**, только скрываются |
| Переопределение `static` метода как нестатического      | Нельзя — сигнатура должна совпадать                                                |
| Обращение к this и super                                | Поскольку static методы/переменные относятся к классу а не объекту                 |

# 📕 3. Спецификатор `Static` применяется к :

| Элемент                         | Пример применения                 |
| ------------------------------- | --------------------------------- |
| Статические переменные (поля)   | `static int count;`               |
| Статические методы              | `static void main(String[] args)` |
| Статические блоки инициализации | `static { ... }`                  |
| ==Вложенные классы==            | `static class Inner {}`           |

***
## 🔶 1. **Статические переменные (поля)**

- ==Общие для всех объектов класса.==
- ==Инициализируются один раз при загрузке класса==.
- Часто используются как **счетчики**, **константы**, **кэш** и т.д.
```java
class MyClass {
    static int count = 0;
	
    MyClass() {
        count++;
    }
}

public class Main {
    public static void main(String[] args) {
        new MyClass();
        new MyClass();
        System.out.println(MyClass.count); // 2
    }
}
```

***
## 🔶 2. **Статические методы

- ==Вызываются **без создания экземпляра**.==
- ==Могут обращаться ТОЛЬКО к другим ==`static`.
- Часто используются в **утилитарных классах** (`Math`, `Collections`, и т.д.)
```java
class Utils {
    static int square(int x) {
        return x * x;
    }
}

int result = Utils.square(5); // 25
```
### 💡 Ограничения `static`-методов: 💡
```java
class Example {
    int a = 5;

    static void show() {
        // System.out.println(a); ❌ ошибка!
        // нельзя обращаться к нестатическим переменным
    }
}
```

***
## 🔶 3. **Статический блок инициализации**

- ==Выполняется один раз при **загрузке класса**.==
- Используется для **инициализации сложных статических переменных**.

```java
class Config {
    static final Map<String, String> SETTINGS;

    static {
        SETTINGS = new HashMap<>();
        SETTINGS.put("mode", "production");
        SETTINGS.put("version", "1.0");
    }
}
```

***
## 🔶 4. **Статические вложенные классы**

- Это вложенные классы, ==**не имеющие доступа к нестатическим полям внешнего класса**==.
- Их ==можно использовать **без создания экземпляра внешнего класса**==.
```java
class Outer {
    static class Inner {
        void show() {
            System.out.println("Static nested class");
        }
    }
}

// Вызов:
Outer.Inner obj = new Outer.Inner();
obj.show();
```

***
## 🔶 5. **static final** — Константы

Если вы хотите объявить **константу**, используйте `static final`.
```java
class MathConstants {
    static final double PI = 3.14159;
}
```
