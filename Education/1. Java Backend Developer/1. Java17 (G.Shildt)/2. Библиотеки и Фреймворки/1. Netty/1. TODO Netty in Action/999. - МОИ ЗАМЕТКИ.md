
В Netty ключевой разработкой является - настройка обработчиков (ChannelHandler) канала (Channel)

Каждый Channel содержит:
- ChannelHandler
- EventLoop 
- ChannelPipeline (цепочка ChannelHandler)

ChannelHandler позволяет обрабатывать событие операции в Netty. А более точно, то у него есть два под вида `ChannelInboundHandler` и `ChannelOutboundHandler` (исходящие/входящие сообщения)

Также существует и `SimpleChannelInboundHandler`, который освобождает буфер с сообщением, после завершения метода где он используется

Чтобы написать собственный обработчик, нужно реализовать необходимый интерфейс. 
НО, есть уже "БАЗОВЫЕ" реализации для Inbound\Outbound (входящих/исходящий) сообщений `ChannelInboundHandlerAdapter` или `ChannelOutboundHandlerAdapter`

```java
//Для сервера
ServerBootstrap bootstrap = new ServerBootstrap();  
  
bootstrap.group(group)  
         .channel(NioServerSocketChannel.class)  
         .localAddress(new InetSocketAddress(port))  
         .childHandler(new ChannelInitializer<SocketChannel>() {  
             @Override  
             protected void initChannel(SocketChannel ch) throws Exception {  
                 ch.pipeline().addLast(echoServerHandler);  
             }  
         });

//Для клиента
bootstrap.group(group)  
    .channel(NioSocketChannel.class)  
    .remoteAddress(new InetSocketAddress(host, port))  
    .handler(new ChannelInitializer<SocketChannel>() {  
  
        @Override  
        protected void initChannel(SocketChannel ch) throws Exception {  
            ch.pipeline().addLast(new EchoClientHandler());  
        }  
    });
```
