Tomcat определяет строгие зависимости между версией **Servlet API** и версией Spring. Вот **таблица совместимости**:

| Версия Tomcat    | Пространство имен   | Версия Servlet API | Версия Spring   |
| ---------------- | ------------------- | ------------------ | --------------- |
| **Tomcat 8.5**   | `javax.servlet.*`   | **Servlet 3.1**    | Spring 4.x, 5.x |
| **Tomcat 9**     | `javax.servlet.*`   | **Servlet 4.0**    | Spring 5.x      |
| **Tomcat 10**    | `jakarta.servlet.*` | **Servlet 5.0**    | Spring 6.x      |
| **Tomcat 10.1+** | `jakarta.servlet.*` | **Servlet 6.0**    | Spring 6.1+     |
### **Другие важные зависимости для Tomcat**
Кроме Servlet API и Spring, есть **дополнительные библиотеки**, которые могут вызывать проблемы совместимости.

|Компонент|Совместимость с Tomcat 8.5/9 (javax)|Совместимость с Tomcat 10+ (jakarta)|
|---|---|---|
|**JSP API**|`javax.servlet.jsp.*`|`jakarta.servlet.jsp.*`|
|**EL API**|`javax.el.*`|`jakarta.el.*`|
|**JPA (Hibernate, EclipseLink)**|`javax.persistence.*`|`jakarta.persistence.*`|
|**JAX-RS (REST API, Jersey, RESTEasy)**|`javax.ws.rs.*`|`jakarta.ws.rs.*`|
|**JAX-B (XML парсинг)**|`javax.xml.bind.*`|`jakarta.xml.bind.*`|
