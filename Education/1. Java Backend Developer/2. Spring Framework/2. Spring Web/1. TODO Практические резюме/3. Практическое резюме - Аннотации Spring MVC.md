# 1. Аннотации для контроллеров
#### `@Controller`

Используется для определения класса как контроллера в Spring MVC.
```java
@Controller
public class MyController {
    @RequestMapping("/")
    public String home() {
        return "index";
    }
}
```
***
#### `@RestController`

Объединяет `@Controller` и `@ResponseBody`, чтобы методы автоматически возвращали JSON/XML-ответ.
```java
@RestController
public class MyRestController {
    @GetMapping("/hello")
    public String sayHello() {
        return "Hello, World!";
    }
}
```

# 2. Аннотации для обработки HTTP-запросов

#### `@RequestMapping`

Определяет соответствие метода HTTP-запросу.
```java
@Controller
@RequestMapping("/api")
public class ApiController {
    @RequestMapping(value = "/users", method = RequestMethod.GET)
    public String getUsers() {
        return "users";
    }
}
```
***
#### `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`, `@PatchMapping`

Более удобные версии `@RequestMapping` для различных HTTP-методов.
```java
@RestController
@RequestMapping("/products")
public class ProductController {

    @GetMapping("/{id}")
    public String getProduct(@PathVariable Long id) {
        return "Product " + id;
    }

    @PostMapping
    public String addProduct(@RequestBody String product) {
        return "Added product: " + product;
    }
}
```

# 3. Аннотации для ПАРАМЕТРОВ запроса

#### `@RequestParam`

Используется для извлечения параметров запроса.
```java
@GetMapping("/search")
public String search(@RequestParam String query) {
    return "Searching for: " + query;
}
```
***
#### `@PathVariable`

Используется для передачи переменных пути.
```java
@GetMapping("/users/{id}")
public String getUser(@PathVariable int id) {
    return "User ID: " + id;
}
```
***
#### `@RequestBody`

Позволяет автоматически десериализовать JSON в объект.
```java
@PostMapping("/users")
public String createUser(@RequestBody User user) {
    return "Created user: " + user.getName();
}
```
***
#### `@RequestHeader`

Извлекает заголовки запроса.
```java
@GetMapping("/header")
public String getHeader(@RequestHeader("User-Agent") String userAgent) {
    return "User-Agent: " + userAgent;
}
```
***
#### `@CookieValue`

Позволяет получить значение cookie.
```java
@GetMapping("/cookie")
public String getCookie(@CookieValue("sessionId") String sessionId) {
    return "Session ID: " + sessionId;
}
```

# 4. Аннотации для обработки ответов

#### `@ResponseBody`

Преобразует возвращаемые данные в JSON/XML (если не используется `@RestController`).
```java
@Controller
public class MyController {
    @GetMapping("/json")
    @ResponseBody
    public String getJson() {
        return "{\"message\":\"Hello, JSON!\"}";
    }
}
```
***
#### `@ResponseStatus`

Позволяет задать HTTP-статус ответа.
```java
@ResponseStatus(HttpStatus.NOT_FOUND)
public class ResourceNotFoundException extends RuntimeException {
    public ResourceNotFoundException(String message) {
        super(message);
    }
}
```

# 5. **Аннотации для валидации**

#### `@Valid` и `@Validated`

Используются для валидации входных данных.
```java
@PostMapping("/users")
public String addUser(@Valid @RequestBody User user, BindingResult result) {
    if (result.hasErrors()) {
        return "Validation failed";
    }
    return "User added";
}
```

Для валидации полей, требует зависимость
```xml
<dependency>
    <groupId>org.hibernate.validator</groupId>
    <artifactId>hibernate-validator</artifactId>
    <version>6.1.6.Final</version>
</dependency>
```

> [!NOTE] Класс `BindingResult`
> Объект этого класса, ==ДОЛЖЕН идти СРАЗУ== после аргумента, помеченного аннотацией `Valid
> 
> В него помещаются сообщения об ошибках валидации

# 6. Аннотации для конфигурации и фильтрации

#### `@ModelAttribute`

Позволяет добавлять общие данные в `Model`.
```java
@ModelAttribute("appName")
public String getAppName() {
    return "My Application";
}
```
***
#### `@ExceptionHandler`

Обрабатывает исключения на уровне контроллера
```java
@ControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(Exception.class)
    @ResponseBody
    public String handleException(Exception e) {
        return "Error: " + e.getMessage();
    }
}
```
***
#### `@InitBinder`

Позволяет задать кастомные редакторы для привязки данных.
```java
@InitBinder
public void initBinder(WebDataBinder binder) {
    binder.registerCustomEditor(Date.class, new CustomDateEditor(new SimpleDateFormat("yyyy-MM-dd"), false));
}
```
