Классы `SimpleJdbcInsert` и `SimpleJdbcCall` используют метаданные базы данных, получаемые через JDBC-драйвер, чтобы минимизировать конфигурацию. 

==Вы можете переопределить или отключить обработку метаданных==, если хотите явно указать все детали в коде.

## 1.1 Вставка данных с помощью `SimpleJdbcInsert`

Класс создаётся в методе инициализации (например, `setDataSource`) с fluent API. 

> [!NOTE] СУТЬ работы `SimpleJdbc...`
> ```java
> new SimpleJdbcInsert(dataSource)
> 	.withTableName("t_actor");
> ```
> При создании `SimpleJdbcInsert`, указывается таблица, с которой будем работать.

Минимальная конфигурация — указать имя таблицы:
```java
public class JdbcActorDao implements ActorDao {
    private SimpleJdbcInsert insertActor;

    public void setDataSource(DataSource dataSource) {
        this.insertActor = new SimpleJdbcInsert(dataSource)
		    .withTableName("t_actor");
    }

    public void add(Actor actor) {
        Map<String, Object> parameters = new HashMap<>(3);
        
        parameters.put("id", actor.getId());
        parameters.put("first_name", actor.getFirstName());
        parameters.put("last_name", actor.getLastName());
        
        insertActor.execute(parameters);
    }
}
```

> [!WARNING] СОВПАДЕНИЕ КЛЮЧЕЙ В `Map` и столбцов таблицы БД
> **Важно:** Ключи в `Map` должны совпадать с именами колонок в базе данных, так как Spring читает метаданные для построения запроса.

---
## 1.2 Получение автосгенерированных ключей

Метод `usingGeneratedKeyColumns` указывает имя колонки с автосгенерированным ключом. 

Вместо `execute` используется `executeAndReturnKey`, который возвращает `java.lang.Number`:
```java
public void setDataSource(DataSource dataSource) {
    this.insertActor = new SimpleJdbcInsert(dataSource)
            .withTableName("t_actor")
            .usingGeneratedKeyColumns("id");
}

public void add(Actor actor) {
    Map<String, Object> parameters = new HashMap<>(2);
    
    parameters.put("first_name", actor.getFirstName());
    parameters.put("last_name", actor.getLastName());
    
    Number newId = insertActor.executeAndReturnKey(parameters);
    actor.setId(newId.longValue());
}
```

> [!WARNING] Несколько автосгенерированных колонов
> **Примечание:** Если автосгенерированных колонок несколько или значения нечисловые, используйте `executeAndReturnKeyHolder`, который возвращает `KeyHolder`.

---
## 1.3 Явное указание колонок
Метод `usingColumns` ограничивает колонки для вставки. 

Остальные колонки не будут включены в запрос:
```java
this.insertActor = new SimpleJdbcInsert(dataSource)
        .withTableName("t_actor")
        .usingColumns("first_name", "last_name")
        .usingGeneratedKeyColumns("id");
```

---
## 1.4 Использование `SqlParameterSource` для передачи параметров

Вместо `Map` ==удобнее использовать реализации== `SqlParameterSource`:

---
### Класс: `BeanPropertySqlParameterSource`

**`BeanPropertySqlParameterSource`** — ==извлекает значения из геттеров JavaBean==:
```java
public void add(Actor actor) {
    SqlParameterSource parameters = 
	    new BeanPropertySqlParameterSource(actor);
	    
    Number newId = insertActor.executeAndReturnKey(parameters);
    
    actor.setId(newId.longValue());
}
```

---
### Класс: `MapSqlParameterSource`

**`MapSqlParameterSource`** — **похож на** `Map`, ==но с цепочечным методом== `addValue`:
```java
public void add(Actor actor) {
    SqlParameterSource parameters = new MapSqlParameterSource()
            .addValue("first_name", actor.getFirstName())
            .addValue("last_name", actor.getLastName());
            
    Number newId = insertActor.executeAndReturnKey(parameters);
    
    actor.setId(newId.longValue());
}
```

# 2. Вызов хранимых процедур с `SimpleJdbcCall`

Создаётся аналогично `SimpleJdbcInsert`, но с указанием имени процедуры:
```sql
-- Пример хранимой процедуры (MySQL)
CREATE PROCEDURE read_actor (
    IN in_id INTEGER,
    OUT out_first_name VARCHAR(100),
    OUT out_last_name VARCHAR(100),
    OUT out_birth_date DATE)
BEGIN
    SELECT first_name, last_name, birth_date
    INTO out_first_name, out_last_name, out_birth_date
    FROM t_actor where id = in_id;
END;
```

```java
public class JdbcActorDao implements ActorDao {
    private SimpleJdbcCall procReadActor;

    public void setDataSource(DataSource dataSource) {
        this.procReadActor = new SimpleJdbcCall(dataSource)
                .withProcedureName("read_actor");
    }

    public Actor readActor(Long id) {
        SqlParameterSource in = new MapSqlParameterSource()
                .addValue("in_id", id);
                
        Map out = procReadActor.execute(in);
        
        Actor actor = new Actor();
        actor.setId(id);
        actor.setFirstName((String) out.get("out_first_name"));
        actor.setLastName((String) out.get("out_last_name"));
        actor.setBirthDate((Date) out.get("out_birth_date"));
        
        return actor;
    }
}
```

**Чувствительность к регистру:** Имена параметров в результатах зависят от БД. 

Для переносимости используйте `LinkedCaseInsensitiveMap`:
```java
JdbcTemplate jdbcTemplate = new JdbcTemplate(dataSource);
jdbcTemplate.setResultsMapCaseInsensitive(true);

this.procReadActor = new SimpleJdbcCall(jdbcTemplate)
        .withProcedureName("read_actor");
```

## 2.1 Явное объявление параметров
Если метаданные недоступны (база не поддерживается Spring) или вы хотите полный контроль:
```java
this.procReadActor = new SimpleJdbcCall(jdbcTemplate)
        .withProcedureName("read_actor")
        .withoutProcedureColumnMetaDataAccess()// отключаем чтение метаданных
        .useInParameterNames("in_id")
        .declareParameters(
                new SqlParameter("in_id", Types.NUMERIC),
                new SqlOutParameter("out_first_name", Types.VARCHAR),
                new SqlOutParameter("out_last_name", Types.VARCHAR),
                new SqlOutParameter("out_birth_date", Types.DATE)
        );
```

**Поддерживаемые БД для автоматического метаданных:** Apache Derby, DB2, MySQL, MS SQL Server, Oracle, Sybase. Хранимые функции поддерживаются для MySQL, MS SQL Server, Oracle.

---
## 2.2 Типы `SqlParameter` - IN\OUT параметры в хранимой процедуре

- **`SqlParameter`** — IN-параметр.
    
- **`SqlOutParameter`** — OUT-параметр (только возврат значения).
    
- **`SqlInOutParameter`** — INOUT-параметр (и вход, и выход).
    

**Важно:** Только `SqlParameter` и `SqlInOutParameter` используются для входных значений.

Параметры задаются с именем и SQL-типом (константы `java.sql.Types`). Для OUT-параметров можно указать `RowMapper` (для REF CURSOR) или `SqlReturnType` (для кастомной обработки).

---
## 2.3 Вызов хранимых функций
Используется `withFunctionName` и `executeFunction` (возвращает значение напрямую, без извлечения из Map):

```sql
-- Пример хранимой функции (MySQL)
CREATE FUNCTION get_actor_name (in_id INTEGER)
RETURNS VARCHAR(200) READS SQL DATA
BEGIN
    DECLARE out_name VARCHAR(200);
    SELECT concat(first_name, ' ', last_name)
        INTO out_name
        FROM t_actor where id = in_id;
    RETURN out_name;
END;
```

```java
this.funcGetActorName = new SimpleJdbcCall(jdbcTemplate)
        .withFunctionName("get_actor_name");

public String getActorName(Long id) {
    SqlParameterSource in = new MapSqlParameterSource()
	    .addValue("in_id", id);
	    
    return funcGetActorName.executeFunction(String.class, in);
}
```

**Примечание:** Для процедур с одним OUT-параметром есть аналогичный метод `executeObject`.

## 2.4 Возврат `ResultSet`/REF CURSOR

Используется `returningResultSet` с `RowMapper`:
```sql
CREATE PROCEDURE read_all_actors()
BEGIN
    SELECT a.id, a.first_name, a.last_name, a.birth_date FROM t_actor a;
END;
```

```java
this.procReadAllActors = new SimpleJdbcCall(jdbcTemplate)
        .withProcedureName("read_all_actors")
        .returningResultSet("actors",
                BeanPropertyRowMapper.newInstance(Actor.class));

public List getActorsList() {
    Map m = procReadAllActors.execute(new HashMap<String, Object>(0));
    return (List) m.get("actors");
}
```
