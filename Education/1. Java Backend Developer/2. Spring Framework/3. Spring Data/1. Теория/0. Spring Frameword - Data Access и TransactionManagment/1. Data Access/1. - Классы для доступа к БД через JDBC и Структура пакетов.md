# 1. Классы для доступа к БД
Spring Framework предоставляет несколько подходов для работы с JDBC. 

**Все они**, так или иначе, **основаны на** `JdbcTemplate`, но различаются по уровню абстракции и удобству использования. Вы можете комбинировать их в одном проекте.

#### 1.1. `JdbcTemplate`
- **Классический и самый популярный** подход.
    
- Самый «низкоуровневый» из всех — все остальные подходы внутри себя используют именно его.
    
- Работает с традиционными JDBC-плейсхолдерами `?`.
    
- **Когда использовать:** Когда нужен полный контроль над SQL и минимальные накладные расходы.


#### 1.2. `NamedParameterJdbcTemplate`
- **Обёртка над `JdbcTemplate`**.
    
- Использует **именованные параметры** (например, `:firstName`) вместо `?`.
    
- **Плюсы:** Лучшая читаемость и удобство при большом количестве параметров в SQL-запросе. Сразу видно, какой параметр за что отвечает.
    
- **Когда использовать:** Когда запросы содержат много параметров и важна читаемость кода.

#### 1.3. `SimpleJdbcInsert` и `SimpleJdbcCall`
- Используют **метаданные базы данных** для минимизации конфигурации.
    
- **`SimpleJdbcInsert`:** ==Для вставки данных== — **достаточно указать имя таблицы и передать Map с параметрами**, имена которых совпадают с именами колонок.
    
- **`SimpleJdbcCall`:** ==Для вызова хранимых процедур== — аналогично, минимум конфигурации.
    
- **Требование:** Работает только если БД предоставляет достаточные метаданные. Если нет — придётся настраивать параметры явно.
    
- **Когда использовать:** Когда нужно быстро выполнить вставку или вызов процедуры без написания полного SQL.

#### 1.4. RDBMS Objects
- Объектно-ориентированный подход: `MappingSqlQuery`, `SqlUpdate`, `StoredProcedure`.
    
- Эти объекты создаются **один раз при инициализации** (потокобезопасные, переиспользуемые). Вы определяете строку запроса, объявляете параметры, компилируете запрос.
    
- Затем методы `execute(...)`, `update(...)`, `findObject(...)` можно вызывать многократно с разными значениями параметров.
    
- **Когда использовать:** Когда нужно создать переиспользуемый, скомпилированный однажды объект для частых типовых операций.

# 2. Структура пакетов
Фреймворк Spring JDBC состоит из четырёх основных пакетов:

#### 2.1. `core` (Ядро)
- **`org.springframework.jdbc.core`** — содержит `JdbcTemplate` и его callback-интерфейсы.
    
- **`org.springframework.jdbc.core.simple`** — содержит `SimpleJdbcInsert` и `SimpleJdbcCall`.
    
- **`org.springframework.jdbc.core.namedparam`** — содержит `NamedParameterJdbcTemplate`.

#### 2.2. `datasource` (Источники данных)
- **`org.springframework.jdbc.datasource`** — **утилиты для доступа к** `DataSource`, простые реализации для тестирования и работы вне Java EE-контейнера.
    
- **`org.springframework.jdbc.datasource.embedded`** — ==поддержка встроенных БД== (HSQL, H2, Derby).

#### 2.3. `object` (RDBMS-объекты)
- **`org.springframework.jdbc.object`** — содержит `MappingSqlQuery`, `SqlUpdate`, `StoredProcedure`.
    
- Представляет запросы, обновления и хранимые процедуры как **потокобезопасные, переиспользуемые объекты**.
    
- Это более высокоуровневая абстракция, которая зависит от пакета `core`.

#### 2.4. `support` (Поддержка)
- **`org.springframework.jdbc.support`** — содержит `SQLExceptionTranslator` и утилиты.
    
- **Главная функция:** Преобразование JDBC/RDBMS-специфичных исключений в **непроверяемые** исключения из пакета `org.springframework.dao`.
    
- Вам не нужно писать RDBMS-специфичную обработку ошибок. Все исключения становятся **unchecked**, и вы можете ловить только те, которые реально можете обработать.
