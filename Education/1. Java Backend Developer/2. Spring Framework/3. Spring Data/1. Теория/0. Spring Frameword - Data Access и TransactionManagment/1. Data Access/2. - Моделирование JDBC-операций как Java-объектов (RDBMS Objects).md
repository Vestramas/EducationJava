Пакет `org.springframework.jdbc.object` **предоставляет классы для доступа к БД** ==в объектно-ориентированном стиле.== 
> Однако многие разработчики считают, что эти классы (кроме `StoredProcedure`) можно заменить прямыми вызовами `JdbcTemplate`. 

Используйте их, только если это даёт измеримую выгоду.

# 1 `SqlQuery` и `MappingSqlQuery`
`SqlQuery` — переиспользуемый, потокобезопасный класс, инкапсулирующий SQL-запрос. 

На практике чаще используют его подкласс **`MappingSqlQuery`**, который требует реализовать метод `mapRow(..)` для преобразования строки `ResultSet` в объект.

> [!IMPORTANT] Метод `declareParameter()`
> Метод `declareParameter` используется для **явного объявления параметров** SQL-запроса перед его компиляцией в классах RDBMS-объектов (`SqlQuery`, `SqlUpdate`, `StoredProcedure`).


> [!IMPORTANT] Метод `compile()`
> После объявления всех параметров вызов `compile()` фиксирует структуру запроса и делает объект **потокобезопасным**. Его можно безопасно хранить как поле экземпляра DAO и использовать многократно без дополнительной синхронизации.


**Пример `MappingSqlQuery`:**
```java
public class ActorMappingQuery extends MappingSqlQuery<Actor> {
    public ActorMappingQuery(DataSource ds) {
        super(
	        ds, 
	        "select id, first_name, last_name from t_actor where id = ?"
	    );
	    
        declareParameter(new SqlParameter("id", Types.INTEGER));
        
        compile(); // Компиляция запроса, после этого объект потокобезопасен
    }

    @Override
    protected Actor mapRow(ResultSet rs, int rowNumber) throws SQLException {
        Actor actor = new Actor();
        actor.setId(rs.getLong("id"));
        actor.setFirstName(rs.getString("first_name"));
        actor.setLastName(rs.getString("last_name"));
        return actor;
    }
}
```

**Использование в DAO:**
```java
private ActorMappingQuery actorMappingQuery;

@Autowired
public void setDataSource(DataSource dataSource) {
    this.actorMappingQuery = new ActorMappingQuery(dataSource);
}

// Для одного объекта
public Actor getActor(Long id) {
    return actorMappingQuery.findObject(id);
}

// Для списка объектов
public List<Actor> searchForActors(int age, String namePattern) {
    return actorSearchMappingQuery.execute(age, namePattern);
}
```

# 2. `SqlUpdate`

`SqlUpdate` инкапсулирует SQL-обновление. 

Переиспользуем, потокобезопасен после компиляции. Можно не создавать подкласс, а параметризовать напрямую.

**Пример:**
```java
public class UpdateCreditRating extends SqlUpdate {
    public UpdateCreditRating(DataSource ds) {
        setDataSource(ds);
        
        setSql("update customer set credit_rating = ? where id = ?");
        
        declareParameter(new SqlParameter("creditRating", Types.NUMERIC));
        declareParameter(new SqlParameter("id", Types.NUMERIC));
        
        compile();
    }

    public int execute(int id, int rating) {
        return update(rating, id); // Параметры в порядке плейсхолдеров
    }
}
```

# 3. `StoredProcedure`

Абстрактный суперкласс для объектных абстракций хранимых процедур.

**Параметры:**
- `SqlParameter(name, type)` — IN-параметр.
    
- `SqlOutParameter(name, type)` — OUT-параметр.
    
- `SqlInOutParameter(name, type)` — INOUT-параметр.
    
- Для `OUT`-параметров можно указать `RowMapper` (для REF CURSOR) или `SqlReturnType`.


**Пример вызова ФУНКЦИИ Oracle `sysdate` (без входных параметров):**
```java
private class GetSysdateProcedure extends StoredProcedure {
    public GetSysdateProcedure(DataSource dataSource) {
        setDataSource(dataSource);
        
        setFunction(true);// Это ФУНКЦИЯ, не процедура
        setSql("sysdate");
        
        declareParameter(new SqlOutParameter("date", Types.DATE));
        compile();
    }

    public Date execute() {
        Map<String, Object> results = 
	        execute(new HashMap<String, Object>());
	        
        return (Date) results.get("date");
    }
}
```


**Пример с OUT-параметрами типа REF CURSOR:**
```java
public class TitlesAndGenresStoredProcedure extends StoredProcedure {
    public TitlesAndGenresStoredProcedure(DataSource dataSource) {
        super(dataSource, "AllTitlesAndGenres");
        
        declareParameter(
	        new SqlOutParameter("titles", 
		        OracleTypes.CURSOR,
		        new TitleMapper()
		    )
	    );
        declareParameter(
	        new SqlOutParameter("genres", 
		        OracleTypes.CURSOR, 
		        new GenreMapper()
		    )
		);
		
        compile();
    }

    public Map<String, Object> execute() {
        return super.execute(new HashMap<String, Object>());
    }
}
```


**Пример с входными параметрами:**
```java
public class TitlesAfterDateStoredProcedure extends StoredProcedure {
    private static final String CUTOFF_DATE_PARAM = "cutoffDate";

    public TitlesAfterDateStoredProcedure(DataSource dataSource) {
        super(dataSource, "TitlesAfterDate");
        
        declareParameter(
	        new SqlParameter(CUTOFF_DATE_PARAM, Types.DATE)
	    );
        declareParameter(
	        new SqlOutParameter(
		        "titles",
		        OracleTypes.CURSOR,
		        new TitleMapper()
		    )
		);
        compile();
    }

    public Map<String, Object> execute(Date cutoffDate) {
        Map<String, Object> inputs = new HashMap<>();
        inputs.put(CUTOFF_DATE_PARAM, cutoffDate);
        return super.execute(inputs);
    }
}
```
