# 1. Указание SQL-типа для параметров
Обычно **Spring определяет SQL-тип параметра, по типу переданного Java-объекта.** 

Но иногда (особенно при установке `NULL`) нужно указать тип явно. 

> [!NOTE] 
> Передали параметры в `PreparedStatement`, и теперь М**ОЖЕМ ЯВНО УКАЗАТЬ ТИП КАЖДОГО ПАРАМЕТРА**


Есть три способа:

## **Способ 1: Массив `int[]` с типами**
Многие методы `JdbcTemplate` ***принимают дополнительный параметр — массив констант из*** `java.sql.Types` (по одному элементу на каждый параметр):
```java
jdbcTemplate.update(
    "insert into my_table (name, age) values (?, ?)",
    new Object[]{"John", null},
    new int[]{Types.VARCHAR, Types.INTEGER}  // age может быть NULL
);
```

## **Способ 2: Класс `SqlParameterValue`**
Оборачивает значение и его SQL-тип. Можно указать масштаб (scale) для чисел:
```java
SqlParameterValue value = new SqlParameterValue(Types.INTEGER, null);

jdbcTemplate.update(
	"insert into my_table (age) values (?)", 
	value
);
```

## **Способ 3: Через `SqlParameterSource`**

`BeanPropertySqlParameterSource` и `MapSqlParameterSource` имеют методы для регистрации SQL-типа именованного параметра:
```java
MapSqlParameterSource params = new MapSqlParameterSource();

params.addValue("name", "John");
params.addValue("age", null);

params.registerSqlType("age", Types.INTEGER); // Указываем тип для NULL
```

# 2. Обработка BLOB и CLOB
Для работы с большими объектами (изображения, файлы, большие тексты) Spring использует интерфейс `LobHandler` и `LobCreator`. Обычно используется реализация `DefaultLobHandler`.

## **Вставка BLOB/CLOB:**
Используется `AbstractLobCreatingPreparedStatementCallback`:
```java
final File blobIn = new File("spring2004.jpg");
final InputStream blobIs = new FileInputStream(blobIn);

final File clobIn = new File("large.txt");
final InputStream clobIs = new FileInputStream(clobIn);

final InputStreamReader clobReader = new InputStreamReader(clobIs);

jdbcTemplate.execute(
    "INSERT INTO lob_table (id, a_clob, a_blob) VALUES (?, ?, ?)",
    new AbstractLobCreatingPreparedStatementCallback(lobHandler) {
        protected void setValues(
	        PreparedStatement ps, 
	        LobCreator lobCreator) throws SQLException 
	    {
            ps.setLong(1, 1L);
            
            lobCreator.setClobAsCharacterStream(
	            ps, 
	            2, 
	            clobReader, 
	            (int)clobIn.length()
	        );
            lobCreator.setBlobAsBinaryStream(
	            ps, 
	            3, 
	            blobIs, 
	            (int)blobIn.length()
	        );
        }
    }
);

blobIs.close();

clobReader.close();
```

> **Примечание:** Если передать отрицательную длину в `setBlobAsBinaryStream`/`setClobAsCharacterStream`, `DefaultLobHandler` использует JDBC 4.0-варианты методов без параметра длины. 
> 
> Проверьте документацию вашего драйвера на поддержку такой возможности.

## **Чтение BLOB/CLOB:**
```java
List<Map<String, Object>> l = jdbcTemplate.query(
    "select id, a_clob, a_blob from lob_table",
    (rs, rowNum) -> {
        Map<String, Object> results = new HashMap<>();
        
        // CLOB как строка
        String clobText = lobHandler.getClobAsString(rs, "a_clob");
        
        // BLOB как массив байт   
        byte[] blobBytes = lobHandler.getBlobAsBytes(rs, "a_blob");  
         
        results.put("CLOB", clobText);
        results.put("BLOB", blobBytes);
        
        return results;
    });
```

## **Методы `LobHandler`:**

| Тип  | Ввод                                                                         | Вывод                                                                           |
| ---- | ---------------------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| BLOB | `setBlobAsBytes` / <br>`setBlobAsBinaryStream`                               | `getBlobAsBytes` / <br>`getBlobAsBinaryStream`                                  |
| CLOB | `setClobAsString`/<br>`setClobAsAsciiStream` /<br>`setClobAsCharacterStream` | `getClobAsString` / <br>`getClobAsAsciiStream` / <br>`getClobAsCharacterStream` |

# 3. Передача списков значений для IN-клаузы
JDBC не поддерживает переменное количество плейсхолдеров для `IN (...)` напрямую. 

`NamedParameterJdbcTemplate` решает это: он генерирует нужное количество плейсхолдеров автоматически при передаче `java.util.List`
```java
// Автоматически превратится в 
//"select * from t_actor where id in (?, ?, ?)"
List<Long> ids = Arrays.asList(1L, 2L, 3L);

MapSqlParameterSource params = new MapSqlParameterSource("ids", ids);

namedTemplate.query(
	"select * from t_actor where id in (:ids)", 
	params, 
	...
);
```

**Ограничения:**
- JDBC не гарантирует более 100 значений в `IN`. Разные БД имеют разные лимиты (Oracle — до 1000).
    
- Можно передавать `List<Object[]>` для составных выражений: `WHERE (id, last_name) IN ((1, 'Johnson'), (2, 'Harrop'))` — если БД это поддерживает.

# 4. Обработка сложных типов для хранимых процедур
Для специфичных типов БД (например, пользовательские типы Oracle) есть два интерфейса:

## SqlReturnType
**`SqlReturnType` — для ПОЛУЧЕНИЯ сложного типа из процедуры (OUT-параметр):**

Реализует единственный метод `getTypeValue`. Используется при объявлении `SqlOutParameter`:
```java
new SqlOutParameter("item", Types.STRUCT, "ITEM_TYPE",
    (CallableStatement cs, int colIndx, int sqlType, String typeName) -> {
        Struct struct = (Struct) cs.getObject(colIndx);
        Object[] attr = struct.getAttributes();
        TestItem item = new TestItem();
        item.setId(((Number) attr[0]).longValue());
        item.setDescription((String) attr[1]);
        item.setExpirationDate((java.util.Date) attr[2]);
        return item;
    })
```

## SqlTypeValue
**`SqlTypeValue` — для ПЕРЕДАЧИ сложного типа в процедуру (IN-параметр):**

Реализует метод `createTypeValue`, имеющий доступ к `Connection` для создания объектов БД (`Struct`, `Array`):
```java
TestItem testItem = new TestItem(123L, "A test item", ...);

SqlTypeValue value = new AbstractSqlTypeValue() {
    protected Object createTypeValue(
	    Connection conn, 
	    int sqlType, 
	    String typeName) throws SQLException 
	{
        Object[] item = new Object[] {
            testItem.getId(),
            testItem.getDescription(),
            new java.sql.Date(testItem.getExpirationDate().getTime())
        };
        return conn.createStruct(typeName, item);
    }
};
// value можно передать в Map параметров для execute()
```

**Oracle-специфичный пример (создание массива):**
```java
Long[] ids = new Long[] {1L, 2L};

SqlTypeValue value = new AbstractSqlTypeValue() {
    protected Object createTypeValue(
	    Connection conn, 
	    int sqlType, 
	    String typeName
	)throws SQLException 
	{
        return conn
        .unwrap(
	        OracleConnection.class
	    )
	    .createOracleArray(typeName, ids);
    }
};
```
