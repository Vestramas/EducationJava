Пакет `org.springframework.jdbc.datasource.embedded` предоставляет поддержку встроенных Java-движков баз данных: **HSQL**, **H2** и **Derby**. Также есть расширяемый API для подключения новых типов БД.

## Зачем нужна встроенная БД?
Встроенные БД **полезны на этапе разработки** благодаря:
- Лёгкости конфигурации
    
- Быстрому запуску
    
- Удобству тестирования
    
- Возможности быстро изменять SQL

# 1. Создание встроенной БД
Используется `EmbeddedDatabaseBuilder` (Java) или XML-элемент `<jdbc:embedded-database>`:
```java
@Bean
DataSource dataSource() {
    return new EmbeddedDatabaseBuilder()
            .generateUniqueName(true)  // Уникальное имя (рекомендуется)
            .setType(EmbeddedDatabaseType.H2)
            .addScripts("schema.sql", "test-data.sql")
            .build();
}
```

```xml
<jdbc:embedded-database id="dataSource" generate-name="true" type="H2">
    <jdbc:script location="classpath:schema.sql"/>
    <jdbc:script location="classpath:test-data.sql"/>
</jdbc:embedded-database>
```

Это создаёт встроенную БД, наполняет её данными из скриптов и регистрирует как бин типа `javax.sql.DataSource`.

# 2. Выбор типа встроенной БД
Spring поддерживает три типа:

#### Использование HSQL
Spring поддерживает HSQL версии 1.8.0 и выше. 
**HSQL является встроенной базой данных по умолчанию**, если тип не указан явно. 
Чтобы явно указать HSQL, установите атрибут `type` тега `embedded-database` в значение `HSQL`. 
Если вы используете Builder API, вызовите метод `setType(EmbeddedDatabaseType)` с аргументом `EmbeddedDatabaseType.HSQL`.

#### Использование H2
Spring поддерживает базу данных H2. Чтобы включить H2, установите атрибут `type` тега `embedded-database` в значение `H2`. Если вы используете Builder API, вызовите метод `setType(EmbeddedDatabaseType)` с аргументом `EmbeddedDatabaseType.H2`.

#### Использование Derby
Spring поддерживает Apache Derby версии 10.5 и выше. Чтобы включить Derby, установите атрибут `type` тега `embedded-database` в значение `DERBY`. Если вы используете Builder API, вызовите метод `setType(EmbeddedDatabaseType)` с аргументом `EmbeddedDatabaseType.DERBY`


> В Java: `setType(EmbeddedDatabaseType.H2)`, в XML: `type="H2"`.

# 3. Кастомизация конфигурации
Можно переопределить настройки подключения для поддерживаемых типов:
```java
@Bean
public DataSource dataSource() {
    return new EmbeddedDatabaseBuilder()
            .setDatabaseConfigurer(EmbeddedDatabaseConfigurers
                    .customizeConfigurer(H2, this::customize))
            .addScript("schema.sql")
            .build();
}

private EmbeddedDatabaseConfigurer customize(
	EmbeddedDatabaseConfigurer defaultConfigurer
) {
    return new EmbeddedDatabaseConfigurerDelegate(defaultConfigurer) {
        @Override
        public void configureConnectionProperties(
                ConnectionProperties properties, 
                String databaseName
        ) {
            super.configureConnectionProperties(properties, databaseName);
            properties.setDriverClass(CustomDriver.class); // Свой драйвер
        }
    };
}
```

# 4. Тестирование с встроенной БД
Шаблон интеграционного теста:
```java
public class DataAccessIntegrationTestTemplate {
    private EmbeddedDatabase db;

    @BeforeEach
    public void setUp() {
        // Создаёт in-memory HSQL, наполняет из classpath:schema.sql и classpath:data.sql
        db = new EmbeddedDatabaseBuilder()
                .generateUniqueName(true)
                .addDefaultScripts()
                .build();
    }

    @Test
    public void testDataAccess() {
        JdbcTemplate template = new JdbcTemplate(db);
        template.query( /* ... */ );
    }

    @AfterEach
    public void tearDown() {
        db.shutdown();
    }
}
```

# 5. Генерация уникальных имён
По умолчанию `EmbeddedDatabaseFactory` называет БД `testdb`, а `<jdbc:embedded-database>` — как `id` бина. 

Это приводит к конфликтам при повторном создании в рамках одной JVM (например, в тестах). 

**Решение (Spring 4.2+):**
- `EmbeddedDatabaseBuilder.generateUniqueName(true)`
    
- `EmbeddedDatabaseFactory.setGenerateUniqueDatabaseName()`
    
- XML: `<jdbc:embedded-database generate-name="true" ...>`

---
# 6. Расширение поддержки
Можно добавить поддержку новых типов БД или `DataSource`:

- Реализовать `EmbeddedDatabaseConfigurer` (для нового типа БД).
    
- Реализовать `DataSourceFactory` (для нового `DataSource`, например с пулом соединений).
