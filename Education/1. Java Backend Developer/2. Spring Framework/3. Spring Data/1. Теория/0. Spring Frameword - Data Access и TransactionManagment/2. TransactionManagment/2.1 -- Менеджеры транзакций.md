В Spring есть несколько разных менеджеров транзакций — **каждый предназначен для своего типа ресурса** (JDBC, JPA, Hibernate, JMS, JTA и т.д.).

# 1. DataSourceTransactionManager
**Для чего:** JDBC (чистый SQL), JdbcTemplate.
**Уровень:** уровень соединения с БД (Connection).

**Особенности:**
- управляет транзакциями через `java.sql.Connection`
- поддерживает savepoints
- быстрый и лёгкий
- используется с пулами соединений (HikariCP, C3P0 и т.д.)

**Когда использовать:**
- простые приложения на JDBC
- Spring MVC + JdbcTemplate
- batch-процессы

---
# 2. JpaTransactionManager
**Для чего:** JPA (Hibernate, EclipseLink).
**Уровень:** уровень EntityManager.

**Особенности:**
- интеграция с `EntityManager` и persistence context
- управляет flush(), synchronization
- поддерживает lazy-loading
    

**Когда использовать:**
- любые приложения на JPA/Hibernate
    
- Spring Boot JPA starter использует его по умолчанию

---
# 3. HibernateTransactionManager
**Для чего:** Hibernate Core (когда JPA не используется).
**Уровень:** SessionFactory / Session (native API).


**Особенности:**
- прямое управление Hibernate Session
- поддерживает специфические для Hibernate функции
- даёт больше контроля, чем JPA
    

**Когда использовать:**
- если работаешь с Hibernate напрямую (без JPA)
- если нужны кастомные Hibernate-фичи

---

# 4. JtaTransactionManager
**Для чего:** распределённые транзакции (`XA`).
**Уровень:** JTA (Java Transaction API).


**Особенности:**
- глобальные транзакции (2PC — двухфазный commit)
    
- координация между несколькими ресурсами:
    - несколько БД
    - JMS + DB
    - Kafka + DB

- может использовать внешние координаторы:
    - Atomikos
    - Bitronix
    - Narayana
    - WebLogic
    - WebSphere

**Когда использовать:**
- микросервисы в контейнерах (Narayana + Spring)
    
- интеграция нескольких разных источников данных
    
- сложные enterprise-системы

---

# 5. ChainedTransactionManager (Spring Data)
**Редкий, но полезный.**

Позволяет объединять **несколько локальных транзакционных менеджеров** в один логический:
- DataSourceTransactionManager
    
- HibernateTransactionManager
    
- MongoTransactionManager (Spring Data Mongo)  
    и т.д.


**Работает без XA**, но:
- commit идёт в цепочке
- rollback тоже идёт в цепочке
- не гарантирует атомарность, но подходит для простых случаев

---
# 6. ReactiveTransactionManager
**Для чего:** R2DBC, реактивные базы данных.

Поддерживается в WebFlux.
Реализации:
- `R2dbcTransactionManager`
    
- `CouchbaseReactiveTransactionManager`
    
- `MongoReactiveTransactionManager`
    

Использует **reactor context**, а не ThreadLocal.
