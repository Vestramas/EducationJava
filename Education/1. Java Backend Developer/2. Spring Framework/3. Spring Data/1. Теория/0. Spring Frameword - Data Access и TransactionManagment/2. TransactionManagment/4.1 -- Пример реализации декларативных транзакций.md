Рассмотрим интерфейс `FooService` и его реализацию `DefaultFooService`. 

Методы 
- `get*` должны выполняться ***в транзакции*** **только для чтения**, 
- а `insertFoo`/`updateFoo` — ***в транзакции*** **для чтения-записи**.

```java
package x.y.service;

public interface FooService {
    Foo getFoo(String fooName);
    Foo getFoo(String fooName, String barName);
    void insertFoo(Foo foo);
    void updateFoo(Foo foo);
}

//-------------------------
package x.y.service;

public class DefaultFooService implements FooService {

	@Override
	public Foo getFoo(String fooName) {
		// ...
	}

	@Override
	public Foo getFoo(String fooName, String barName) {
		// ...
	}

	@Override
	public void insertFoo(Foo foo) {
		// ...
	}

	@Override
	public void updateFoo(Foo foo) {
		// ...
	}
}
```

**XML-конфигурация:**
```xml
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:aop="http://www.springframework.org/schema/aop"
    xmlns:tx="http://www.springframework.org/schema/tx"
    xsi:schemaLocation="
        http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/tx
        https://www.springframework.org/schema/tx/spring-tx.xsd
        http://www.springframework.org/schema/aop
        https://www.springframework.org/schema/aop/spring-aop.xsd">

    <!-- Сервис, который мы делаем транзакционным -->
    <bean id="fooService" class="x.y.service.DefaultFooService"/>

    <!-- Транзакционное поведение (advice) -->
    <tx:advice id="txAdvice" transaction-manager="txManager">
        <tx:attributes>
            <!-- Все методы, начинающиеся с 'get', только для чтения -->
            <tx:method name="get*" read-only="true"/>
            <!-- Остальные методы с настройками по умолчанию -->
            <tx:method name="*"/>
        </tx:attributes>
    </tx:advice>

    <!-- Применяем advice к методам FooService -->
    <aop:config>
        <aop:pointcut id="fooServiceOperation" expression="execution(* x.y.service.FooService.*(..))"/>
        <aop:advisor advice-ref="txAdvice" pointcut-ref="fooServiceOperation"/>
    </aop:config>

    <!-- DataSource -->
    <bean id="dataSource" class="org.apache.commons.dbcp.BasicDataSource" destroy-method="close">
        <property name="driverClassName" value="oracle.jdbc.driver.OracleDriver"/>
        <property name="url" value="jdbc:oracle:thin:@rj-t42:1521:elvis"/>
        <property name="username" value="scott"/>
        <property name="password" value="tiger"/>
    </bean>

    <!-- TransactionManager -->
    <bean id="txManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
        <property name="dataSource" ref="dataSource"/>
    </bean>

</beans>
```

**Java-конфигурация:**
```java
@Configuration
@EnableTransactionManagement
public class AppConfig {

    @Bean
    public DataSource dataSource() {
        BasicDataSource ds = new BasicDataSource();
        ds.setDriverClassName(
	        "oracle.jdbc.driver.OracleDriver"
	    );
        ds.setUrl("jdbc:oracle:thin:@rj-t42:1521:elvis");
        ds.setUsername("scott");
        ds.setPassword("tiger");
        return ds;
    }

    @Bean
    public PlatformTransactionManager txManager(
	    DataSource dataSource
	) {
        return new DataSourceTransactionManager(dataSource);
    }

    @Bean
    public FooService fooService() {
        return new DefaultFooService();
    }

    // Транзакционный advice с декларативными правилами
    @Bean
    public TransactionInterceptor txAdvice(
	    PlatformTransactionManager txManager
	) {
		//Класс, для ИМЕННОВАННОГО МЕТЧА методов и типа транзакции
        NameMatchTransactionAttributeSource source = 
	        new NameMatchTransactionAttributeSource();

        // Все методы, начинающиеся с 'get', только для чтения
        RuleBasedTransactionAttribute readOnly = 
	        new RuleBasedTransactionAttribute();
        readOnly.setReadOnly(true);
        
        source.addTransactionalMethod("get*", readOnly);

        // Остальные методы с настройками по умолчанию (чтение-запись)
        RuleBasedTransactionAttribute readWrite = 
	        new RuleBasedTransactionAttribute();
        source.addTransactionalMethod("*", readWrite);

        return new TransactionInterceptor(txManager, source);
    }

    // AOP: применяем txAdvice к методам FooService
    @Bean
    public Advisor txAdvisor(TransactionInterceptor txAdvice) {
        AspectJExpressionPointcut pointcut = 
	        new AspectJExpressionPointcut();
	        
        pointcut.setExpression(
	        "execution(* x.y.service.FooService.*(..))"
	    );
        return new DefaultPointcutAdvisor(pointcut, txAdvice);
    }
}
```

### Важные замечания по конфигурации
- **В Java-конфигурации** обязательно `@EnableTransactionManagement` + бин `TransactionInterceptor` + `Advisor` с pointcut'ом.
    
- **В XML** атрибут `transaction-manager` в `<tx:advice/>` указывает на имя бина `TransactionManager`. Если бин называется **`transactionManager`** — атрибут можно **опустить**.
    
- **Pointcut** определяет, к каким методам применяется транзакционный advice. Чтобы сделать транзакционным весь сервисный слой:
    - **Java:** `"execution(* x.y.service.*.*(..))"` в `AspectJExpressionPointcut`.
        
    - **XML:** `<aop:pointcut ... expression="execution(* x.y.service.*.*(..))"/>`.

### Что происходит при вызове (императивный пример)
При запуске программы:
```java
public final class Boot {
    public static void main(final String[] args) {
        ApplicationContext ctx = 
	        new ClassPathXmlApplicationContext("context.xml");
	        
        FooService fooService = ctx.getBean(FooService.class);
        
        fooService.insertFoo(new Foo());
    }
}
```

1. Создаётся **прокси** для `DefaultFooService`.
    
2. При вызове `insertFoo(...)` на прокси, `TransactionInterceptor` начинает транзакцию.

4. Метод `insertFoo(...)` бросает `UnsupportedOperationException` (unchecked).
    
5. `TransactionInterceptor` применяет правила отката (по умолчанию — откат на `RuntimeException`).
    
6. Транзакция откатывается, `Connection` освобождается.

## Реактивная версия
Для реактивных транзакций интерфейс возвращает реактивные типы:
```java
package x.y.service;

public interface FooService {
    Flux<Foo> getFoo(String fooName);
    Publisher<Foo> getFoo(String fooName, String barName);
    Mono<Void> insertFoo(Foo foo);
    Mono<Void> updateFoo(Foo foo);
}
```

---
## Ключевое различие императивных и реактивных транзакций
- **Императивные транзакции:** 
  ***Результат возвращается*** **после успешного завершения** метода. Частично вычисленные результаты не покидают метод.
    
- **Реактивные транзакции:** 
  Метод возвращает реактивную обёртку (`Mono`/`Flux`), которая представляет **последовательность вычислений** и обещание начать/завершить вычисление. 
  
  `TransactionInterceptor` **декорирует** возвращаемый реактивный тип транзакционным оператором, который начнёт и завершит транзакцию при **подписке** (subscription). 
  
  Фактическое управление транзакцией **откладывается** до активации обработки реактивного типа.
    
- **Проблема «утечки данных»:** `Publisher` может испускать данные, **пока транзакция продолжается, но ещё не завершена**. Методы, зависящие от успешного завершения всей транзакции, должны гарантировать завершение и буферизовать результаты в вызывающем коде.
