Если у вас несколько сервисных объектов и к каждому нужно применить **разную транзакционную конфигурацию**, это делается через отдельные `<aop:advisor/>` с разными `pointcut` и `advice-ref`.

---
## 1.0 ПРОСТАЯ настройка НО...
Можно создать несколько бинов `TransactionManager` и в `@Transactional` указать, какой именно использовать:

```java
@Bean
public PlatformTransactionManager txManager1(DataSource dataSource1) {
    return new DataSourceTransactionManager(dataSource1);
}

@Bean
public PlatformTransactionManager txManager2(DataSource dataSource2) {
    return new DataSourceTransactionManager(dataSource2);
}

// В сервисе указываем, какой менеджер использовать
@Transactional("txManager1")
public void doSomething() { ... }

@Transactional("txManager2")
public void doSomethingElse() { ... }
```

НО

| Подход                                                            | Что настраивается                                                                         | Когда использовать                                                                                     |
| ----------------------------------------------------------------- | ----------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **Разные <br>`TransactionManager`** <br>+ `@Transactional("имя")` | Разные **источники данных** (разные БД), разные типы транзакций (JDBC vs JTA).            | Когда у вас физически разные БД или разные типы менеджеров.                                            |
| **Разные <br>`TransactionInterceptor`** <br>(как в главе)         | Разная **семантика** (read-only, propagation, rollback-правила) для разных групп методов. | Когда БД одна, но к разным сервисам/методам нужно применить разные правила транзакций централизованно. |

---
## 1.1. Пример: одна транзакционная конфигурация для всех сервисов
Pointcut охватывает все бины в пакете `x.y.service` (и подпакетах), чьи имена заканчиваются на `Service`:


> [!NOTE] **AspectJ pointcut expression**
> ```java
> pointcut.setExpression(
> 	"execution(* x.y.service..*Service.*(..))"
> );
> ```
> - execution(...) - Конструкция «при выполнении метода».
> - `*` (первый) - **Любой возвращаемый тип** метода
> - x.y.service.. - Пакет `x.y.service` и **все его подпакеты** на любую глубину. Две точки `..` — это AnyDepth-паттерн.
> - `.*` (перед скобками) - **Любое имя метода**.
> - (..) - **Любые параметры** метода (ноль, один, много, любых типов).


XML-config:
```xml
<aop:config>
    <aop:pointcut id="serviceOperation"
            expression="execution(* x.y.service..*Service.*(..))"/>
    <aop:advisor pointcut-ref="serviceOperation" advice-ref="txAdvice"/>
</aop:config>

<!-- Эти бины будут транзакционными -->
<bean id="fooService" class="x.y.service.DefaultFooService"/>
<bean id="barService" class="x.y.service.extras.SimpleBarService"/>

<!-- А эти — НЕТ -->
<bean id="anotherService" class="org.xyz.SomeService"/>       <!-- не в том пакете -->
<bean id="barManager" class="x.y.service.SimpleBarManager"/>  <!-- имя не заканчивается на 'Service' -->

<tx:advice id="txAdvice">
    <tx:attributes>
        <tx:method name="get*" read-only="true"/>
        <tx:method name="*"/>
    </tx:attributes>
</tx:advice>
```


Java-config:
```java
@Configuration
@EnableTransactionManagement
public class AppConfig {

    //----------------- СЕРВИСЫ ---------------------------
    @Bean
    public FooService fooService() { 
	    return new DefaultFooService(); 
	}

    @Bean
    public BarService barService() { 
	    return new SimpleBarService(); 
	}

    @Bean
    public SomeService anotherService() { 
	    return new SomeService(); 
	}

    @Bean
    public BarManager barManager() { 
	    return new SimpleBarManager(); 
	}
	//_____________________ СЕРВИСЫ _________________________
	
	//----------_________________________________________
    // Общий транзакционный advice для *Service в x.y.service
    @Bean
    public TransactionInterceptor txAdvice(
	    PlatformTransactionManager txManager
	) {
        NameMatchTransactionAttributeSource source = 
	        new NameMatchTransactionAttributeSource();
	        
        RuleBasedTransactionAttribute readOnly = 
	        new RuleBasedTransactionAttribute();
        readOnly.setReadOnly(true);
        
        source.addTransactionalMethod(
	        "get*", 
	        readOnly
	    );
        source.addTransactionalMethod(
	        "*", 
	        new RuleBasedTransactionAttribute()
	    );
	    
        return new TransactionInterceptor(txManager, source);
    }

    // AOP: применяем txAdvice только к сервисам из x.y.service с именами *Service
    @Bean
    public Advisor serviceAdvisor(TransactionInterceptor txAdvice) {
        AspectJExpressionPointcut pointcut = 
	        new AspectJExpressionPointcut();
	        
        pointcut.setExpression(
	        "execution(* x.y.service..*Service.*(..))"
	    );
        return new DefaultPointcutAdvisor(pointcut, txAdvice);
    }
}
```

## 1.2. Пример: две разные транзакционные конфигурации
Один бин (`DefaultFooService`) использует стандартные транзакционные настройки, другой (`DefaultDdlManager`) — с `propagation="NEVER"` (вообще без транзакций):

XML-config:
```xml
<aop:config>
    <!-- Pointcut для обычных сервисов -->
    <aop:pointcut id="defaultServiceOperation"
            expression="execution(* x.y.service.*Service.*(..))"/>

    <!-- Pointcut для DDL-менеджера (без транзакций) -->
    <aop:pointcut id="noTxServiceOperation"
            expression="execution(* x.y.service.ddl.DefaultDdlManager.*(..))"/>

    <aop:advisor pointcut-ref="defaultServiceOperation" advice-ref="defaultTxAdvice"/>
    <aop:advisor pointcut-ref="noTxServiceOperation" advice-ref="noTxAdvice"/>
</aop:config>

<bean id="fooService" class="x.y.service.DefaultFooService"/>
<bean id="anotherFooService" class="x.y.service.ddl.DefaultDdlManager"/>

<!-- Стандартный advice -->
<tx:advice id="defaultTxAdvice">
    <tx:attributes>
        <tx:method name="get*" read-only="true"/>
        <tx:method name="*"/>
    </tx:attributes>
</tx:advice>

<!-- Advice без транзакций -->
<tx:advice id="noTxAdvice">
    <tx:attributes>
        <tx:method name="*" propagation="NEVER"/>
    </tx:attributes>
</tx:advice>
```


Java-config:
```java
@Configuration
@EnableTransactionManagement
public class AppConfig {

    @Bean
    public FooService fooService() { 
	    return new DefaultFooService(); 
	}

    @Bean
    public DefaultDdlManager anotherFooService() { 
	    return new DefaultDdlManager(); 
	}


    @Bean
    public PlatformTransactionManager txManager(
	    DataSource dataSource
	) {
        return new DataSourceTransactionManager(dataSource);
    }

    // Стандартный транзакционный advice
    @Bean
    public TransactionInterceptor defaultTxAdvice(
	    PlatformTransactionManager txManager
	) {
        NameMatchTransactionAttributeSource source = 
	        new NameMatchTransactionAttributeSource();
	       
 
        RuleBasedTransactionAttribute readOnly = 
	        new RuleBasedTransactionAttribute();
	        
        readOnly.setReadOnly(true);
        
        
        source.addTransactionalMethod(
	        "get*", 
	        readOnly
	    );
        source.addTransactionalMethod(
	        "*", 
	        new RuleBasedTransactionAttribute()
	    );
        return new TransactionInterceptor(txManager, source);
    }

    // Advice без транзакций (propagation = NEVER)
    @Bean
    public TransactionInterceptor noTxAdvice(
	    PlatformTransactionManager txManager
	) {
        NameMatchTransactionAttributeSource source = 
	        new NameMatchTransactionAttributeSource();
	        
        RuleBasedTransactionAttribute neverTx = 
	        new RuleBasedTransactionAttribute();
	        
        neverTx.setPropagationBehavior(
	        TransactionDefinition.PROPAGATION_NEVER
	    );
	    
        source.addTransactionalMethod("*", neverTx);
        return new TransactionInterceptor(txManager, source);
    }



    // Применяем defaultTxAdvice к *Service в x.y.service
    @Bean
    public Advisor defaultServiceAdvisor(
	    TransactionInterceptor defaultTxAdvice
	) {
        AspectJExpressionPointcut pointcut = 
	        new AspectJExpressionPointcut();
	        
        pointcut.setExpression(
	        "execution(* x.y.service.*Service.*(..))"
	    );
        return new DefaultPointcutAdvisor(
	        pointcut, 
	        defaultTxAdvice
	    );
    }

    // Применяем noTxAdvice к DefaultDdlManager
    @Bean
    public Advisor noTxServiceAdvisor(
	    TransactionInterceptor noTxAdvice
	) {
        AspectJExpressionPointcut pointcut = 
	        new AspectJExpressionPointcut();
	        
        pointcut.setExpression(
	        "execution(* x.y.service.ddl.DefaultDdlManager.*(..))");
        return new DefaultPointcutAdvisor(
	        pointcut, 
	        noTxAdvice
	    );
    }
}
```

**Примечание:** Если используется Spring Boot или `@Transactional` непосредственно на классах/методах, ==такая детальная AOP-конфигурация часто не нужна== — достаточно аннотаций. 

Но если нужно централизованно применить правила по паттернам имён методов, `TransactionInterceptor` с `NameMatchTransactionAttributeSource` — правильный путь.
