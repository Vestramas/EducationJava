# 1. `EntityManager` — центральный интерфейс JPA
**`EntityManager`** — это **главный объект** для работы с JPA. 

Если проводить аналогию с JDBC, который мы изучили ранее:

| JDBC           | JPA                    | Назначение                                  |
| -------------- | ---------------------- | ------------------------------------------- |
| `DataSource`   | `EntityManagerFactory` | Фабрика для создания соединений/менеджеров  |
| `Connection`   | `EntityManager`        | **Основной объект для выполнения операций** |
| `JdbcTemplate` | Методы `EntityManager` | Упрощает выполнение типовых операций        |

**`EntityManager` отвечает за:**
- `persist(entity)` — **сохранение** новой сущности (аналог `INSERT`).
    
- `merge(entity)` — **обновление** существующей сущности (аналог `UPDATE`).
    
- `remove(entity)` — **удаление** сущности (аналог `DELETE`).
    
- `find(Class, id)` — *поиск по первичному ключу*.
    
- `createQuery(jpql)` — *создание запросов на JPQL* (Java Persistence Query Language).
    
- Управление Persistence Context (контекстом персистентности).
    
---
# 2. `EntityManagerFactory` — фабрика менеджеров

> [!NOTE] EntityManagerFactory
> **`EntityManagerFactory`** — это **потокобезопасная фабрика**, которая создаёт экземпляры `EntityManager`. 
> 
> Она настраивается **один раз при запуске приложения** и живёт всё время его работы.


**Важное различие:**
- `EntityManagerFactory` — **потокобезопасен** (можно использовать из разных потоков).
    
- `EntityManager` — **НЕ потокобезопасен**. Каждый поток должен использовать свой экземпляр.


**Как создаётся:**
- Вручную: через `persistence.xml` и `Persistence.createEntityManagerFactory(...)`.
    
- В Spring: через `LocalContainerEntityManagerFactoryBean`
    
- В Spring Boot: автоматически из настроек `application.properties`

# 3. Persistence Context (контекст персистентности)

Это **самая важная концепция JPA**, которой не было в чистом JDBC. 

> [!NOTE] Persistence Context - кэш 
> Persistence Context — это **кэш первого уровня**, которым управляет `EntityManager`.

**Что он делает:**
- Хранит **все сущности**, загруженные или созданные в рамках текущей транзакции.
    
- **Гарантирует**, что для одного первичного ключа в рамках одного Persistence Context существует **только один экземпляр** сущности.
    
- **Отслеживает изменения** (dirty checking): если вы изменили поле у загруженной сущности, JPA сама сгенерирует `UPDATE` при коммите транзакции.


**Типы Persistence Context:**

|Тип|Описание|Потокобезопасность|
|---|---|---|
|**Transaction-scoped** (по умолчанию)|Живёт в рамках одной транзакции. При коммите все изменения сбрасываются в БД.|Да (через прокси)|
|**Extended**|Живёт дольше одной транзакции. Используется в stateful-компонентах.|**Нет!**|

**Пример работы:**
```java
@Transactional
public void updateUser(Long id, String newName) {
	
	// Загрузили из БД, сущность в Persistence Context
    User user1 = em.find(User.class, id);  
    
    // Изменили поле
    user1.setName(newName);   
        
    // JPA отследила изменение. При коммите автоматически выполнит UPDATE.
    // user1 == user2 (это ОДИН объект в памяти)
}
```

# 4. JPQL (Java Persistence Query Language)

> [!NOTE] JPQL (Java Persistence Query Language)
> JPQL — это **объектно-ориентированный язык запросов** JPA. В отличие от SQL, который работает с таблицами и колонками, JPQL работает с **сущностями и их полями**.


**Пример:**
```java
// SQL:      SELECT * FROM users WHERE email = ?
// JPQL:     SELECT u FROM User u WHERE u.email = :email

TypedQuery<User> query = em.createQuery(
    "SELECT u FROM User u WHERE u.email = :email", User.class);
    
query.setParameter("email", "test@example.com");

List<User> users = query.getResultList();
```

> **Основное преимущество:** 
> JPQL *не зависит от конкретной БД*. 
> 
> Один и тот же запрос будет работать на PostgreSQL, MySQL, Oracle (при правильном диалекте провайдера).
