Спецификация JPA 2.1 ввела поддержку вызова хранимых процедур через JPA Criteria Query API. Spring Data JPA предоставляет аннотацию **`@Procedure`** для объявления метаданных хранимой процедуры на методе репозитория.

## 1. Пример хранимой процедуры (HSQL DB)
```sql
DROP procedure IF EXISTS plus1inout;

CREATE procedure plus1inout (IN arg int, OUT res int)
BEGIN ATOMIC
 set res = arg + 1;
END;
```

## 2. Объявление метаданных на сущности
**Метаданные хранимых процедур можно настроить через аннотацию** **`@NamedStoredProcedureQuery`** на типе сущности:
```java
@Entity
@NamedStoredProcedureQuery(
	name = "User.plus1", 
	procedureName = "plus1inout", 
	parameters = {
	  @StoredProcedureParameter(
		  mode = ParameterMode.IN, 
		  name = "arg", 
		  type = Integer.class
	),
	@StoredProcedureParameter(
		mode = ParameterMode.OUT, 
		name = "res", 
		type = Integer.class) 
	}
)
public class User {}
```

**Важное различие:** У `@NamedStoredProcedureQuery` есть **два разных имени**:
- **`name`** — ***имя, которое использует JPA для ссылки на эту конфигурацию.***
    
- **`procedureName`** — ==имя, которое хранимая процедура имеет== **в базе данных**.

---
## 3. СПОСОБЫ ВЫЗОВА хранимых процедур из репозитория

### **Способ 1: Прямая ссылка на имя процедуры в БД через `value`**
```java
@Procedure("plus1inout")
Integer explicitlyNamedPlus1inout(Integer arg);
```

---
### **Способ 2: Прямая ссылка через атрибут `procedureName` (псевдоним `value`)**
```java
@Procedure(procedureName = "plus1inout")
Integer callPlus1InOut(Integer arg);
```

----
### **Способ 3: Неявное сопоставление по имени метода**
Если ни `value`, ни `procedureName`, ни `name` не указаны — **имя метода репозитория** используется как `name` для поиска `@NamedStoredProcedureQuery`.
```java
@Procedure
Integer plus1inout(@Param("arg") Integer arg);
```

Этот вызов найдёт `@NamedStoredProcedureQuery` с `name = "User.plus1inout"` (имя сущности + имя метода).

---
### **Способ 4: Ссылка на `@NamedStoredProcedureQuery` по атрибуту `name`**
```java
@Procedure(name = "User.plus1IO")
Integer entityAnnotatedCustomNamedProcedurePlus1IO(
	@Param("arg") Integer arg
);
```

---
# 2. Получение значений
## 2.1. Возврат одного OUT-параметра
Если у хранимой процедуры **один OUT-параметр**, он может быть возвращён как **возвращаемое значение метода**:
```java
@Procedure("plus1inout")
Integer plus1(Integer arg);  // OUT-параметр "res" возвращается как Integer
```

## 2.2. Возврат нескольких OUT-параметров
Если в `@NamedStoredProcedureQuery` указано **несколько OUT-параметров**, их можно вернуть как **`Map<String, Object>`**, где ключи — имена параметров из аннотации.

**Объявление сущности с несколькими OUT-параметрами:**
```java
@Entity
@NamedStoredProcedureQuery(
	name = "User.multiple_out_parameters",
	procedureName = "multiple_out_parameters", 
	parameters = {
		@StoredProcedureParameter(
			mode = ParameterMode.IN, 
			name = "arg", 
			type = Integer.class
		),
		@StoredProcedureParameter(
			mode = ParameterMode.REF_CURSOR, 
			name = "some_cursor", 
			type = void.class
		),
		@StoredProcedureParameter(
			mode = ParameterMode.OUT, 
			name = "res", 
			type = Integer.class) 
	}
)
public class User {}
```

**Метод репозитория:**
```java
@Procedure(name = "User.multiple_out_parameters")
Map<String, Object> returnsMultipleOutParameters(
	@Param("arg") Integer arg
);
```

> **Важное примечание:** 
> Если хранимая процедура возвращает `ResultSet`, **любые OUT-параметры опускаются**, так как Java может вернуть только одно возвращаемое значение (если метод не объявляет возврат `Map`).
