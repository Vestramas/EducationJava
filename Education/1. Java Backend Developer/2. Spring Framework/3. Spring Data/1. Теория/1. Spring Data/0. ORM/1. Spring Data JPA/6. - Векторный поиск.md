С развитием генеративного ИИ векторные базы данных получили широкое распространение. 

Они позволяют эффективно хранить и запрашивать многомерные векторы, что делает их подходящими для задач **семантического поиска, рекомендательных систем и понимания естественного языка**.

***Векторный поиск извлекает*** **семантически похожие данные**, сравнивая векторные представления (эмбеддинги), а не полагаясь на традиционные запросы точного совпадения. 

Spring Data позволяет интегрировать векторный поиск, используя привычные абстракции репозиториев.

**Зависимость для Hibernate Vector Search:**
```xml
<dependency>
    <groupId>org.hibernate.orm</groupId>
    <artifactId>hibernate-vector</artifactId>
    <version>${hibernate.version}</version>
</dependency>
```

> **Важно:** `Vector` можно использовать как тип для запросов, но **не в доменной модели**. Hibernate требует массивы `float[]` или `double[]` в качестве типов векторных колонок.

---
## 1. Модель векторных данных
**`Vector`** — лёгкая иммутабельная **обёртка над массивом чисел с плавающей точкой**. 

Может использоваться как вход для поисковых запросов или как свойство доменной сущности для хранения связанного векторного представления.
```java
class Comment {
  @Id String id;
  String country;
  String comment;

  @Column(name = "the_embedding")
  @JdbcTypeCode(SqlTypes.VECTOR)
  @Array(length = 5)
  Vector embedding;
}
```

> **Особенность:** 
> Связывание вектора с доменным объектом означает, что вектор загружается и сохраняется как часть жизненного цикла сущности, что может создавать дополнительную нагрузку.


**`SearchResults<T>` и `SearchResult<T>`** — инкапсулируют результаты векторного поиска. 
Каждый `SearchResult` включает как совпавший доменный объект, так и **оценку релевантности (score)**. 

Оценка релевантности — числовое значение, показывающее, насколько близко совпавший вектор соответствует вектору запроса. **Оценки не являются частью доменной модели** и лучше всего представляются как внешние (out-of-band) данные.

---
## 2. Score, Similarity и Scoring Functions

**`Score`** содержит числовое значение релевантности. 

Интерпретация зависит от конкретной функции схожести. **`ScoringFunction`** вычисляет `Score`.

**Стандартные функции схожести:**

|Функция|Описание|
|---|---|
|**Euclidean Distance**|Расстояние по прямой в n-мерном пространстве (сумма квадратов разностей).|
|**Cosine Similarity**|Измеряет угол между векторами (нормализованное скалярное произведение).|
|**Dot Product**|Сумма поэлементных умножений.|

Hibernate транслирует вызовы функций расстояния в нативные функции БД для PGvector и Oracle. При использовании **`Similarity`** вместо `Score`, Spring Data **нормализует** значения расстояния в оценку схожести от 0 до 1 (чем выше, тем более похожи векторы).

**Примеры использования Score и Similarity:**
```java
interface CommentRepository extends Repository<Comment, String> {
	SearchResults<Comment> searchByEmbeddingNear(
		Vector vector, 
		ScoringFunction function
	);
	SearchResults<Comment> searchByEmbeddingNear(
		Vector vector, 
		Score score
	);
	SearchResults<Comment> searchByEmbeddingNear(
		Vector vector, 
		Similarity similarity
	);
	SearchResults<Comment> searchByEmbeddingNear(
		Vector vector, 
		Range<Similarity> range
	);
}

// Поиск с косинусной схожестью
repository.searchByEmbeddingNear(Vector.of(…), ScoringFunction.cosine());

// Поиск с ограничением по score (дистанция ≤ 0.9)
repository.searchByEmbeddingNear(
	Vector.of(…), 
	Score.of(0.9, ScoringFunction.cosine())
);

// Поиск с нормализацией в similarity (схожесть ≥ 0.9)
repository.searchByEmbeddingNear(
	Vector.of(…), 
	Similarity.of(0.9, ScoringFunction.cosine())
);

// Поиск с диапазоном similarity от 0.5 до 1.0 (евклидово расстояние)
repository.searchByEmbeddingNear(
	Vector.of(…), 
	Similarity.between(0.5, 1, ScoringFunction.euclidean())
);
```

> JPA требует указывать `ScoringFunction` при создании экземпляров `Score` или `Similarity`.

## 3. Методы векторного поиска
Определяются с использованием тех же соглашений, что и стандартные методы запросов Spring Data. Возвращают `SearchResults<T>` и требуют параметр `Vector`.

**Два подхода к объявлению:**

### **1. Производные методы поиска (Derived Search Methods)**
Ключевые слова: `Near`/`IsNear`, `Within`/`IsWithin`.
```java
interface CommentRepository extends Repository<Comment, String> {
	SearchResults<Comment> searchByEmbeddingNear(
		Vector vector, 
		Score score
	);
	SearchResults<Comment> searchByEmbeddingWithin(
		Vector vector, 
		Range<Similarity> range
	);
	SearchResults<Comment> searchByCountryAndEmbeddingWithin(
		String country, 
		Vector vector, 
		Range<Similarity> range
	);
}
```

Производные методы **требуют** наличия `Score`, `Range<Score>` или `ScoringFunction` как второго аргумента для `Near`/`Within`.

### **2. Аннотированные методы поиска (Annotated Search Methods)**

Предоставляют полный контроль над семантикой запроса через `@Query` с JPQL.
```java
interface CommentRepository extends Repository<Comment, String> {
	@Query("""
		SELECT c, cosine_distance(c.embedding, :embedding) as distance FROM Comment c
		WHERE c.country = ?1
		AND cosine_distance(c.embedding, :embedding) <= :distance
		ORDER BY distance asc
	""")
	SearchResults<Comment> searchAnnotatedByCountryAndEmbeddingWithin(
		String country, 
		Vector embedding, 
		Score distance
	);
	
	@Query("""
		SELECT c FROM Comment c
		WHERE c.country = ?1
		AND cosine_distance(c.embedding, :embedding) <= :distance
		ORDER BY cosine_distance(c.embedding, :embedding) asc
	""")
	List<Comment> findAnnotatedByCountryAndEmbeddingWithin(
		String country, 
		Vector embedding, 
		Score distance
	);
}
```

**Особенности аннотированных методов:**
- Если метод возвращает `SearchResults` и в проекции есть **вторая колонка** — она считается значением score.
    
- Если запрос **не определяет score**, значение score в `SearchResult<T>` будет равно **нулю**.
    
- Нормализация `Similarity` использует нативную score-функцию внутри запроса.

## 4. Сортировка

По умолчанию результаты сортируются по score. Можно переопределить через `Sort` (только по свойствам доменной модели, **score как критерий сортировки не допускается**):
```java
interface CommentRepository extends Repository<Comment, String> {
	SearchResults<Comment> searchByEmbeddingNearOrderByCountry(
		Vector vector, 
		Score score
	);
	SearchResults<Comment> searchByEmbeddingWithin(
		Vector vector, 
		Score score, 
		Sort sort
	);
}
```
