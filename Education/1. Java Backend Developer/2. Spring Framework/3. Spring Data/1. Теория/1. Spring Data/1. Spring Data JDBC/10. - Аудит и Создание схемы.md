# 1. Аудит
Для активации аудита добавьте **`@EnableJdbcAuditing`** в конфигурацию:

```java
@Configuration
@EnableJdbcAuditing
class Config {

    @Bean
    AuditorAware<AuditableUser> auditorProvider() {
        return new AuditorAwareImpl();
    }
}
```

Если вы предоставляете бин типа `AuditorAware` в `ApplicationContext`, инфраструктура аудита **автоматически подхватывает его** и использует для определения текущего пользователя. 

Если зарегистрировано несколько реализаций, нужную можно выбрать через атрибут `auditorAwareRef` в `@EnableJdbcAuditing`.

# 2. Создание схемы (Schema Creation)

Spring Data JDBC поддерживает интеграцию с **Liquibase** для создания изменений схемы БД на основе доменной модели (code-first подход).

**Пример доменной сущности:**
```java
@Table
class Person {
    @Id long id;
    String firstName;
    String lastName;
    LocalDate birthday;
    boolean active;
}
```

**Генерация ChangeSet:**
```java
RelationalMappingContext context = … // Контекст содержит сущность Person

LiquibaseChangeSetWriter writer = new LiquibaseChangeSetWriter(context);

writer.writeChangeSet(new FileSystemResource(new File(…)));
```

**Результат — сгенерированный change log:**
```yaml
databaseChangeLog:
- changeSet:
    id: '1685969572426'
    author: Spring Data Relational
    objectQuotingStrategy: LEGACY
    changes:
    - createTable:
        columns:
        - column:
            autoIncrement: true
            constraints:
              nullable: false
              primaryKey: true
            name: id
            type: BIGINT
        - column:
            constraints:
              nullable: true
            name: first_name
            type: VARCHAR(255 BYTE)
        - column:
            constraints:
              nullable: true
            name: last_name
            type: VARCHAR(255 BYTE)
        - column:
            constraints:
              nullable: true
            name: birthday
            type: DATE
        - column:
            constraints:
              nullable: false
            name: active
            type: TINYINT
        tableName: person
```

**Как это работает:**
- Типы колонок вычисляются из реализации стратегии `SqlTypeMapping`.
    
- **Nullability** выводится из типа: `false` для примитивных типов, `true` для ссылочных.
    

**Дифференциальный режим (differential mode):** Можно передать существующую `Liquibase Database` — writer сравнит существующие таблицы с маппингом сущностей и выведет разницу (какие таблицы/колонки создать или удалить). **По умолчанию ничего не удаляется** — для удаления нужно настроить фильтры:
```java
writer.setDropTableFilter(tableName -> …);
writer.setDropColumnFilter((tableName, columnName) -> …);
```

**Ограничения:** Поддержка схемы определяет только добавления и удаления (удаление не маппящихся таблиц/колонок, добавление отсутствующих). **Переименование колонок и миграция данных не поддерживаются**, так как маппинг сущностей не содержит информации о том, как схема эволюционировала.
