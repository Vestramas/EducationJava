Spring Data JDBC позволяет создавать **конвертеры** для преобразования значений между Java-типами и типами, понятными базе данных.

**Источник:** Это **значение из колонки таблицы в БД**. 
Например, если у вас есть таблица:
```sql
CREATE TABLE person (
  id BIGINT,
  name VARCHAR(255),
  email VARCHAR(255)   -- Вот отсюда берётся строка
);
```

Когда Spring Data JDBC выполняет `SELECT * FROM person`, он получает `ResultSet`, **в котором для каждой строки**, ***есть значение колонки*** `email` — обычная строка: `"user@example.com"`.

**Без конвертера** Spring Data JDBC сохранит email как обычную `String` в поле `String email` вашего Java-класса.

**С конвертером** вы можете превратить эту строку в специализированный тип.

## 1. `@ReadingConverter` — чтение из БД

Преобразует значение из БД в Java-объект:
```java
@ReadingConverter
public class EmailReadConverter implements Converter<String, Email> {

	public Email convert(String source) {
		return Email.valueOf(source);
	}
}
```

---
## 2. `@WritingConverter` — запись в БД
Преобразует Java-объект в значение для БД.

---
## 3. Разрешение неоднозначности
Если и исходный, и целевой тип являются нативными — Spring Data **не может определить**, читающий это конвертер или пишущий. 

Например, `Converter<String, Long>` — неоднозначен. Чтобы явно указать направление, используются аннотации **`@ReadingConverter`** и **`@WritingConverter`**.

---
## 4. Регистрация конвертеров
Конвертеры **НЕ ПОДБИРАЮТСЯ АВТОМАТИЧЕСКИ** из classpath или сканирования контейнера. 

**Их нужно регистрировать** **явно** через **`CustomConversions`** — центральный компонент для регистрации и поиска конвертеров по исходному и целевому типу.

---
## 5. Предопределённые конвертеры
`CustomConversions` поставляется с набором предопределённых конвертеров:

- **JSR-310:** преобразование между `java.time`, `java.util.Date` и `String`.
    
- Конвертеры по умолчанию для локальных временных типов (например, `LocalDateTime` → `java.util.Date`) **зависят от системного часового пояса**. Можно переопределить, зарегистрировав свой конвертер.

---
## 6. Определение направления по типам
Spring Data анализирует исходный и целевой типы конвертера:

- Если **только целевой тип** может быть обработан нативно → **Writing Converter**: `Converter<Person, String>`.
    
- Если **только исходный тип** может быть обработан нативно → **Reading Converter**: `Converter<String, Person>`.
