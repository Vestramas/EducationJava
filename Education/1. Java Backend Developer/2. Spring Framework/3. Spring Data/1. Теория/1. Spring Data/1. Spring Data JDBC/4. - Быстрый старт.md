## 1. Требования
- **Spring Framework** 7.0.7 и выше.
    
- **База данных:** Spring Data JDBC требует **диалект** для абстрагирования общих SQL-функций поверх специфичных для вендора особенностей. 
  Прямая поддержка: **DB2, H2, HSQLDB, MariaDB, Microsoft SQL Server, MySQL, Oracle, PostgreSQL**. 
  Если используется другая БД — приложение **не запустится** без дополнительных действий (см. раздел «Диалекты»).

## 3. Логирование
Spring Data JDBC ==самостоятельно почти НЕ ВЕДЕТ ЛОГИРОВАННИЕ.== 

SQL-запросы логируются через `JdbcTemplate`. 

Для отладки добавьте в `application.properties`:
```
logging.level.org.springframework.jdbc=DEBUG
```

## 4. Конфигурация
Активация поддержки репозиториев Spring Data JDBC через аннотации - `@EnableJdbcRepositories`:

### Что делает `@EnableJdbcRepositories`?
Эта аннотация **сканирует пакет** в поисках интерфейсов, унаследованных от `Repository`, и для каждого **генерирует реализацию**, которая **внутри использует `JdbcTemplate`**. 

Мы избавляемся от ручного написания SQL для CRUD-операций и простых запросов.

---
**Пример конфигурации**
```java
@Configuration
@EnableJdbcRepositories // (1)
class ApplicationConfig extends AbstractJdbcConfiguration {// (2)

    @Bean
    DataSource dataSource() { // (3)
        EmbeddedDatabaseBuilder builder = new EmbeddedDatabaseBuilder();
        return builder.setType(EmbeddedDatabaseType.HSQL).build();
    }

    @Bean
    NamedParameterJdbcOperations namedParameterJdbcOperations(
	    DataSource dataSource
	) { // (4)
        return new NamedParameterJdbcTemplate(dataSource);
    }

    @Bean
    TransactionManager transactionManager(DataSource dataSource) { // (5)
        return new DataSourceTransactionManager(dataSource);
    }
}
```

1. **`@EnableJdbcRepositories`** — создаёт реализации для интерфейсов, производных от `Repository`.
    
2. **`AbstractJdbcConfiguration`** — предоставляет различные default-бины, необходимые Spring Data JDBC.
    
3. **`DataSource`** — соединение с БД.
    
4. **`NamedParameterJdbcOperations`** — используется Spring Data JDBC для доступа к БД.
    
5. **`TransactionManager`** — Spring Data JDBC использует управление транзакциями из Spring JDBC.
    

Конфигурация использует `EmbeddedDatabaseBuilder` для встроенной HSQL. Расширение `AbstractJdbcConfiguration` регистрирует различные бины. Переопределяя его методы, можно настроить поведение (см. ниже).

**Упрощение через Spring Boot:** Достаточно `DataSource` и зависимости `spring-boot-starter-data-jdbc` — всё остальное Spring Boot настраивает автоматически.

### Отличие Spring JDBC и Spring Data JDBC
#### **Раньше (Spring JDBC):**
Мы ==писали DAO вручную==, используя `JdbcTemplate`:
```java
@Repository
public class ProductDaoImpl implements ProductDao {

    private final JdbcTemplate jdbcTemplate;

    public ProductDaoImpl(DataSource dataSource) {
        this.jdbcTemplate = new JdbcTemplate(dataSource);
    }

    public List<Product> findByCategory(String category) {
        return jdbcTemplate.query(
            "SELECT * FROM product WHERE category = ?",
            (rs, rowNum) -> new Product(...),
            category);
    }

    public void save(Product product) {
        jdbcTemplate.update(
            "INSERT INTO product (...) VALUES (...)",
            product.getName(), ...);
    }
}
```

---
#### **Сейчас (Spring Data JDBC):**
Мы ==пишем только интерфейс==, а **реализацию создаёт Spring:**
```java
// Только интерфейс — без единой строчки кода реализации!
public interface ProductRepository extends CrudRepository<Product, Long> {

    List<Product> findByCategory(String category);  // Запрос генерируется автоматически
}
```

И используем:
```java
@Service
public class ProductService {
    private final ProductRepository repository;

    // save, findAll, findById, findByCategory — всё готово
}
```

## 5. Диалекты

Spring Data JDBC использует интерфейс **`JdbcDialect`** для инкапсуляции поведения, специфичного для конкретной БД или её JDBC-драйвера.

- **По умолчанию:** `AbstractJdbcConfiguration` пытается определить диалект автоматически, получая соединение и регистрируя правильный `JdbcDialect`.
    
- **Кастомизация:** Переопределите `AbstractJdbcConfiguration.jdbcDialect(NamedParameterJdbcOperations)`.
    
- **Если БД без диалекта:** Приложение не запустится. Нужно либо попросить вендора предоставить реализацию `JdbcDialect`, либо реализовать свой.
    

**Авто-обнаружение пользовательского диалекта** через `META-INF/spring.factories`:

1. Реализуйте свой `JdbcDialect`.
    
2. Реализуйте `JdbcDialectProvider`, возвращающий ваш `JdbcDialect`.
    
3. Зарегистрируйте провайдер в `META-INF/spring.factories`:
```
org.springframework.data.jdbc.core.dialect.DialectResolver$JdbcDialectProvider=<полное_имя_вашего_JdbcDialectProvider>
```
