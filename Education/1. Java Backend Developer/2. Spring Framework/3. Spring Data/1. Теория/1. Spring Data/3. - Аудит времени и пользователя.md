Spring Data предоставляет прозрачный механизм для автоматического отслеживания того, **кто** и **когда** ***создал или изменил сущность.***

## 1. Основы
Чтобы включить аудит, нужно:
1. Добавить метаданные аудита в классы сущностей (через аннотации или интерфейс).
    
2. Включить аудит в конфигурации (через `@EnableJpaAuditing` для JPA или XML).

**Важно:** Если вам нужно отслеживать только даты создания/изменения, сущности **не обязаны** реализовывать `AuditorAware`.

---
## 2. Аннотации аудита
Spring Data предоставляет четыре аннотации:

|Аннотация|Назначение|
|---|---|
|`@CreatedBy`|Кто создал сущность|
|`@LastModifiedBy`|Кто последним изменил сущность|
|`@CreatedDate`|Дата и время создания|
|`@LastModifiedDate`|Дата и время последнего изменения|
**Пример использования:**
```java
class Customer {
    @CreatedBy
    private User user;

    @CreatedDate
    private Instant createdDate;

    // ... другие поля
}
```

- Аннотации можно применять **выборочно** (только те, что нужны).
    
- Для дат поддерживаются: типы JDK 8 Date/Time (`Instant`, `LocalDateTime` и т.д.), `long`, `Long`, а также устаревшие `Date` и `Calendar`.
    
- Текущее время предоставляется `DateTimeProvider`. По умолчанию это `CurrentDateTimeProvider`. Его можно изменить через атрибут `dateTimeProviderRef` при включении аудита.

**Вложенные метаданные:** Метаданные аудита ==не обязательно должны быть в самом классе сущности==. Их можно вынести во встраиваемый объект:
```java
class Customer {
    private AuditMetadata auditingMetadata;
    // ...
}

class AuditMetadata {
    @CreatedBy
    private User user;

    @CreatedDate
    private Instant createdDate;
}
```

## 3. Отслеживание пользователя (`AuditorAware`)

Если используются `@CreatedBy` или `@LastModifiedBy`, ***Spring Data должен знать***, **кто** является текущим пользователем. 

Для этого **нужно реализовать интерфейс** `AuditorAware<T>`, где `T` — тип свойства, помеченного этими аннотациями.

**Пример с Spring Security:**
```java
class SpringSecurityAuditorAware implements AuditorAware<User> {
    @Override
    public Optional<User> getCurrentAuditor() {
        return Optional.ofNullable(SecurityContextHolder.getContext())
            .map(SecurityContext::getAuthentication)
            .filter(Authentication::isAuthenticated)
            .map(Authentication::getPrincipal)
            .map(User.class::cast);
    }
}
```

Этот код берёт текущего аутентифицированного пользователя из контекста безопасности Spring Security.

**Для реактивных приложений** существует аналог — `ReactiveAuditorAware<T>`, возвращающий `Mono<T>`:
```java
class SpringSecurityAuditorAware implements ReactiveAuditorAware<User> {
    @Override
    public Mono<User> getCurrentAuditor() {
        return ReactiveSecurityContextHolder.getContext()
            .mapNotNull(SecurityContext::getAuthentication)
            .filter(Authentication::isAuthenticated)
            .mapNotNull(Authentication::getPrincipal)
            .map(User.class::cast);
    }
}
```
