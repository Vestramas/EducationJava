# 1. Работа с данными пользователя
Когда приходит запрос в приложение, нужно как-то извлечь данные о пользователе (логин+пароль/jwt/еще что-то) и проверить эти данные

## 1.1 `UserDetailsService` — для аутентификации по логину и паролю
`UserDetailsService` — это интерфейс с единственным методом `loadUserByUsername(String username)`. 

![[Pasted image 20260507213858.png]]

**Когда пользователь вводит логин и пароль** *через форму входа или HTTP Basic*, фильтр `UsernamePasswordAuthenticationFilter` (или `BasicAuthenticationFilter`) создаёт `UsernamePasswordAuthenticationToken` и передаёт его в `AuthenticationManager`. 

Тот в свою очередь вызывает `DaoAuthenticationProvider`, который и дёргает `UserDetailsService.loadUserByUsername()`.

**Как это работает**
1. Провайдер получает `username` из токена (`UsernamePasswordAuthenticationToken`).
    
2. Вызывает ваш `UserDetailsService` и получает объект `UserDetails`
	
3. Сравнивает введённый пароль (с помощью `PasswordEncoder`) с тем, что хранится в `UserDetails`.
    
4. Если пароль совпадает — создаётся полностью аутентифицированный `UsernamePasswordAuthenticationToken`, который помещается в `SecurityContext`.

### 1.1.1 Связка UserDetailsService → UserDetails → PasswordEncoder

#### 1. UserDetailsService

> [!NOTE] Сервис для получения `UserDetails`
> **`UserDetailsService`** — интерфейс с единственным методом `loadUserByUsername(String username)`. 

Ваша **реализация должна найти пользователя в БД** (*или любом другом хранилище*) и вернуть объект `UserDetails`. 
Если пользователь не найден — выбросить `UsernameNotFoundException`.
```java
@Service
public class JpaUserDetailsService implements UserDetailsService {
    
    private final UserRepository userRepository; //JPA репозиторий или др
    
    @Override
    public UserDetails loadUserByUsername(
	    String username
	) throws UsernameNotFoundException 
	{
        return userRepository.findByUsername(username)
            .map(SecurityUser::new)  // оборачиваем в UserDetails
            .orElseThrow(() ->
	             new UsernameNotFoundException("User not found")
	        );
    }
}
```

----
#### 2. UserDetails
**`UserDetails`** — интерфейс, который представляет данные пользователя с точки зрения безопасности: `getUsername()`, `getPassword()`, `getAuthorities()`, `isEnabled()`, и т.д. 

Обычно вы не реализуете его напрямую, а 
- либо используете готовый класс `User` из Spring Security, 
- либо создаёте свой класс-адаптер, который **композицией включает** вашу JPA-entity.
```java
// Композиция: адаптер, который оборачивает вашу entity
public class SecurityUser implements UserDetails {

    private final User user;  // ваша JPA-entity
    
    @Override
    public String getPassword() {
        return user.getPasswordHash();  // пароль уже закодирован
    }
    
    @Override
    public Collection<? extends GrantedAuthority> getAuthorities() {
        return user.getRoles().stream()
            .map(role -> new SimpleGrantedAuthority(role.getName()))
            .toList();
    }
    // ... остальные методы
}
```

---
#### 3. PasswordEncoder
**`PasswordEncoder`** — используется в двух местах:
- **При создании/сохранении пользователя**: вы кодируете пароль перед записью в БД.
    
- **При аутентификации**: `DaoAuthenticationProvider` вызывает `passwordEncoder.matches(rawPassword, encodedPassword)` для проверки пароля.

```java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```
(либо другая реализация - см [[2.2 -- Хранение паролей (Кодировщики паролей)]])

## 1.2 `JwtAuthenticationConverter` — для JWT-аутентификации
Когда приложение является Resource Server и принимает JWT-токены, пароль не проверяется — вместо этого проверяется подпись токена. 

Поэтому `UserDetailsService` не нужен. 

![[Pasted image 20260507214824.png]]

Вместо него **используется** `JwtAuthenticationConverter`, который преобразует уже проверенный `Jwt` в объект `Authentication`, содержащий данные о пользователе и его правах.

По умолчанию Spring Security создаёт `JwtAuthenticationToken` с `principal = Jwt` и authorities, извлечёнными из claim'ов `scope` или `scp` с префиксом `SCOPE_`. 
> Чтобы адаптировать это под свою модель, **мы создаём кастомный конвертер.**

**Как это работает**
1. `BearerTokenAuthenticationFilter` извлекает токен из заголовка `Authorization`.
    
2. `JwtAuthenticationProvider` декодирует токен через `JwtDecoder` и получает объект `Jwt`.
    
3. `JwtAuthenticationConverter` преобразует `Jwt` в `AbstractAuthenticationToken` (обычно `JwtAuthenticationToken`), извлекая имя пользователя (обычно из `sub`) и authorities (из вашего claim'а).

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    // --- Декодер (JwtDecoder) — аналог UserDetailsService ---
    @Bean
    public JwtDecoder jwtDecoder() {
        return NimbusJwtDecoder
            .withJwkSetUri("https://provider.com/.well-known/jwks.json")
            .build();
    }

    // --- Конвертер (JwtAuthenticationConverter) — превращает Jwt в Authentication ---
    @Bean
    public JwtAuthenticationConverter jwtAuthenticationConverter() 
    {
        JwtAuthenticationConverter converter = 
	        new JwtAuthenticationConverter();
        
        converter
	        .setJwtGrantedAuthoritiesConverter(
		        jwtGrantedAuthoritiesConverter()
		    );
        return converter;
    }

    // --- Вспомогательный конвертер authorities (JwtGrantedAuthoritiesConverter) ---
    @Bean
    public JwtGrantedAuthoritiesConverter jwtGrantedAuthoritiesConverter() 
    {
        JwtGrantedAuthoritiesConverter converter = 
	        new JwtGrantedAuthoritiesConverter();
	    
	    // из какого claim'а брать роли
        converter.setAuthoritiesClaimName("roles");
        
        // префикс (по умолчанию
        converter.setAuthorityPrefix("ROLE_"); SCOPE_)
        
        return converter;
    }
}
```

**Что настраивается:**
- `JwtDecoder` — где брать публичный ключ и как проверять подпись.
    
- `JwtAuthenticationConverter` — как превратить `Jwt` в `Authentication`.
    
- `JwtGrantedAuthoritiesConverter` — из какого claim'а брать authorities и с каким префиксом.
