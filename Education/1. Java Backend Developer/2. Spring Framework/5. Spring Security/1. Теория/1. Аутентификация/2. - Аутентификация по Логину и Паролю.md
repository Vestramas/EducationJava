Один из **наиболее распространенных способов аутентификации пользователя** — проверка имени пользователя и пароля. 

Spring Security обеспечивает всестороннюю поддержку аутентификации с использованием имени пользователя и пароля.

Вы можете настроить аутентификацию по имени пользователя и паролю, используя следующее:

**Простой пример с именем пользователя и паролем**
```jaVA
@Configuration
@EnableWebSecurity
public class SecurityConfig {

	@Bean
	public SecurityFilterChain securityFilterChain(
		HttpSecurity http
	) throws Exception 
	{
		http
			.authorizeHttpRequests((authorize) -> 
				authorize
					.anyRequest().authenticated()
			)
			.httpBasic(Customizer.withDefaults())
			.formLogin(Customizer.withDefaults());

		return http.build();
	}

	@Bean
	public UserDetailsService userDetailsService() {
		UserDetails userDetails = 
			User.withDefaultPasswordEncoder()
				.username("user")
				.password("password")
				.roles("USER")
				.build();

		return new 
			InMemoryUserDetailsManager(userDetails);
	}
}
```

```xml
<http>
	<intercept-url pattern="/**" access="authenticated"/>
	<form-login />
	<http-basic />

	<user-service>
		<user name="user"
			password="{noop}password"
			authorities="ROLE_USER" />
	</user-service>
</http>
```

> **Примечание:** `User#withDefaultPasswordEncoder` **считается небезопасным** для production и предназначен только для примеров приложений. См. `User#withDefaultPasswordEncoder` для получения дополнительной информации.

Приведенная **выше конфигурация автоматически регистрирует** `UserDetailsService` в памяти с помощью `SecurityFilterChain`, 
регистрирует `DaoAuthenticationProvider` с `AuthenticationManager` по умолчанию и включает аутентификацию через форму входа (Form Login) и HTTP Basic.

## 1.1 Публикация бина `AuthenticationManager`

Довольно распространенным требованием является публикация бина `AuthenticationManager` для обеспечения возможности кастомной аутентификации, например, в `@Service` или `@Controller` Spring MVC. Например, вы можете захотеть аутентифицировать пользователей через REST API вместо использования формы входа.

Вы можете опубликовать такой `AuthenticationManager` для сценариев кастомной аутентификации, используя следующую конфигурацию:

**Публикация бина `AuthenticationManager` для кастомной аутентификации**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

	@Bean
	public SecurityFilterChain securityFilterChain(
		HttpSecurity http
	) throws Exception {
		http
			.authorizeHttpRequests((authorize) -> 
				authorize
					.requestMatchers("/login").permitAll()
					.anyRequest().authenticated()
			);

		return http.build();
	}

	@Bean
	public AuthenticationManager authenticationManager(
		UserDetailsService userDetailsService,
		PasswordEncoder passwordEncoder
	) {
		DaoAuthenticationProvider authenticationProvider 
		= new DaoAuthenticationProvider(
			userDetailsService
		);
		
		authenticationProvider
			.setPasswordEncoder(passwordEncoder);

		return 
			new	ProviderManager(authenticationProvider);
	}

	@Bean
	public UserDetailsService userDetailsService() {
		UserDetails userDetails = 
			User.withDefaultPasswordEncoder()
				.username("user")
				.password("password")
				.roles("USER")
				.build();

		return 
			new InMemoryUserDetailsManager(userDetails);
	}

	@Bean
	public PasswordEncoder passwordEncoder() {
		return PasswordEncoderFactories
			.createDelegatingPasswordEncoder();
	}
}
```

Имея приведенную выше конфигурацию, вы можете создать `@RestController`, который использует `AuthenticationManager` следующим образом:

**Создание `@RestController` для аутентификации**
```java
@RestController
public class LoginController {

	private 
		final AuthenticationManager authenticationManager;

	public LoginController(
		AuthenticationManager authenticationManager
	) {
		this.authenticationManager = 
			authenticationManager;
	}

	@PostMapping("/login")
	public ResponseEntity<Void> login(
		@RequestBody LoginRequest loginRequest
	) {
	// Шаг 1. Мы создаем ОБЪЕКТ-ЗАПРОС.
	// Мы говорим: "Вот логин и пароль, которые ввел пользователь. ПРОВЕРЬ ИХ".
	// 💡Метод .unauthenticated() создает токен с флагом isAuthenticated = false.
	// Это как чистый бланк с написанными данными, но без печати.
		Authentication authenticationRequest =
			UsernamePasswordAuthenticationToken
				.unauthenticated(
					loginRequest.username(), 
					loginRequest.password()
				);
	// Шаг 2. Мы передаем этот ЗАПРОС в AuthenticationManager.
	// AuthenticationManager внутри себя идет в базу, проверяет пароль через PasswordEncoder.
	// Если все ОК, он возвращает НОВЫЙ объект Authentication.
		Authentication authenticationResponse =
			this.authenticationManager
				.authenticate(authenticationRequest);
		// ...
		
	// Шаг 3. (За кадром, но подразумевается)
	// Теперь authenticationResponse имеет флаг isAuthenticated = true
	// и содержит ВСЮ информацию о пользователе (Principal, Authorities).
	// Именно этот объект мы должны положить в SecurityContextHolder.
	}

	public record LoginRequest(
		String username, 
		String password
	) {
	}
}
```

В этом примере **вы сами отвечаете** з*а сохранение аутентифицированного пользователя в* `SecurityContextRepository`, если это необходимо. Например, если используется `HttpSession` для сохранения `SecurityContext` между запросами, вы можете использовать `HttpSessionSecurityContextRepository`.

## 1.2 Кастомизация `AuthenticationManager`

==Обычно Spring Security создает внутренни==й `AuthenticationManager`, состоящий из `DaoAuthenticationProvider` для аутентификации по имени пользователя и паролю. 

В определенных случаях может потребоваться кастомизировать экземпляр `AuthenticationManager`, используемый Spring Security. 
> Например, *вам может понадобиться просто отключить очистку учетных данных для кэшированных пользователей.*

Чтобы сделать это, **вы можете воспользоваться тем фактом, что** `AuthenticationManagerBuilder`, используемый для построения глобального `AuthenticationManager` Spring Security, публикуется как бин. 

Вы можете настроить его следующим образом:

**Настройка глобального `AuthenticationManagerBuilder`**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

	@Bean
	public SecurityFilterChain securityFilterChain(
		HttpSecurity http
	) throws Exception {
		// ...
		return http.build();
	}

	@Bean
	public UserDetailsService userDetailsService() {
		// Вернуть UserDetailsService, который кэширует пользователей
		// ...
	}

	@Autowired
	public void configure(
		AuthenticationManagerBuilder builder
	) {
		builder.eraseCredentials(false);
	}
}
```

В качестве альтернативы вы можете настроить локальный `AuthenticationManager`, чтобы переопределить глобальный.

**Настройка локального `AuthenticationManager` для Spring Security**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

	@Bean
	public SecurityFilterChain securityFilterChain(
		HttpSecurity http
	) throws Exception {
		http
			.authorizeHttpRequests((authorize) -> 
				authorize
					.anyRequest().authenticated()
			)
			.httpBasic(Customizer.withDefaults())
			.formLogin(Customizer.withDefaults())
			.authenticationManager(
				authenticationManager()
			);

		return http.build();
	}

	private AuthenticationManager authenticationManager() 
	{
		DaoAuthenticationProvider authenticationProvider 
			= new DaoAuthenticationProvider(
				userDetailsService()
			);
		authenticationProvider
			.setPasswordEncoder(passwordEncoder());

		ProviderManager providerManager =
			new ProviderManager(authenticationProvider);
		
		providerManager
			.setEraseCredentialsAfterAuthentication(
				false
			);

		return providerManager;
	}

	private UserDetailsService userDetailsService() {
		UserDetails userDetails = 
			User.withDefaultPasswordEncoder()
				.username("user")
				.password("password")
				.roles("USER")
				.build();

		return 
			new InMemoryUserDetailsManager(userDetails);
	}

	private PasswordEncoder passwordEncoder() {
		return PasswordEncoderFactories
			.createDelegatingPasswordEncoder();
	}
}
```
