# 1. Вход по одноразовому токену (One-Time Token Login)
Spring Security предлагает поддержку аутентификации по одноразовому токену (One-Time Token, OTT) через DSL-метод `oneTimeTokenLogin()`.

Прежде чем углубляться в детали реализации, важно прояснить область применения функции OTT в рамках фреймворка, выделив, что поддерживается, а что нет.

## 1.1 Понимание разницы между одноразовыми токенами и одноразовыми паролями
Часто путают одноразовые токены (One-Time Tokens, OTT) с одноразовыми паролями (One-Time Passwords, OTP), но в Spring Security эти концепции различаются по нескольким ключевым аспектам. 

Для ясности будем считать, что OTP относится к TOTP (Time-Based One-Time Password) или HOTP (HMAC-Based One-Time Password).

### Требования к настройке
- **OTT:** Начальная настройка не требуется. Пользователю не нужно ничего настраивать заранее.
    
- **OTP:** Обычно требует настройки, такой как генерация и обмен секретным ключом с внешним инструментом для создания одноразовых паролей.

### Доставка токена
- **OTT:** Обычно ***необходимо реализовать пользовательский*** `OneTimeTokenGenerationSuccessHandler`, отвечающий ***за доставку токена конечному пользователю***.
    
- **OTP:** Токен часто ***генерируется внешним инструментом,*** поэтому нет необходимости отправлять его пользователю через приложение.

### Генерация токена
- **OTT:** Метод `OneTimeTokenService.generate(GenerateOneTimeTokenRequest)` требует возврата `OneTimeToken`, что подчеркивает **генерацию на стороне сервера**.
    
- **OTP:** Токен не обязательно генерируется на стороне сервера; он часто создается клиентом с использованием общего секрета.

В итоге, одноразовые токены (OTT) предоставляют способ аутентификации пользователей без дополнительной настройки учетной записи, что отличает их от одноразовых паролей (OTP), которые обычно включают более сложный процесс настройки и полагаются на внешние инструменты для генерации токена.

Вход по одноразовому токену работает в два основных этапа:
1. Пользователь запрашивает токен, отправляя свой идентификатор пользователя (обычно имя пользователя), и токен доставляется ему, часто в виде «волшебной ссылки» (Magic Link), по электронной почте, SMS и т. д.
    
2. Пользователь отправляет токен на конечную точку входа по одноразовому токену, и, если токен действителен, пользователь входит в систему.

В следующих разделах мы рассмотрим, как настроить OTT Login в соответствии с вашими потребностями.

---
## 1.2 Страница входа по умолчанию и страница отправки одноразового токена по умолчанию
При использовании DSL-метода `oneTimeTokenLogin()` страница входа по одноразовому токену по умолчанию автоматически генерируется `org.springframework.security.web.authentication.ui.DefaultLoginPageGeneratingFilter`. 

DSL также настроит `DefaultOneTimeTokenSubmitPageGeneratingFilter` для генерации страницы отправки одноразового токена по умолчанию.

---
## 1.3 Отправка токена пользователю
Spring Security не может разумным образом определить способ доставки токена вашим пользователям. 

Поэтому **для доставки токена пользователю** в соответствии с вашими потребностями **должен быть предоставлен пользовательский** `OneTimeTokenGenerationSuccessHandler`. 

> [!NOTE] OneTimeTokenGenerationSuccessHandler
> Для отправки OTT пользователю

Одной из наиболее распространенных стратегий доставки является «волшебная ссылка» (Magic Link) по электронной почте, SMS и т. д. 

В следующем примере мы создадим волшебную ссылку и отправим ее на электронную почту пользователя.

**Конфигурация входа по одноразовому токену**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) {
        http
            // ...
            .formLogin(Customizer.withDefaults())
            .oneTimeTokenLogin(Customizer.withDefaults());
        return http.build();
    }

}

import org.springframework.mail.SimpleMailMessage;
import org.springframework.mail.javamail.JavaMailSender;

@Component // (1)
public class MagicLinkOneTimeTokenGenerationSuccessHandler implements OneTimeTokenGenerationSuccessHandler {

    private final MailSender mailSender;

    private final OneTimeTokenGenerationSuccessHandler redirectHandler = 
	    new RedirectOneTimeTokenGenerationSuccessHandler(
		    "/ott/sent"
		);

    // конструктор опущен

    @Override
    public void handle(
	    HttpServletRequest request, 
	    HttpServletResponse response, 
	    OneTimeToken oneTimeToken
	) throws IOException, ServletException 
	{
        UriComponentsBuilder builder = 
        UriComponentsBuilder
	        .fromHttpUrl(
		        UrlUtils.buildFullRequestUrl(request))
                .replacePath(request.getContextPath())
                .replaceQuery(null)
                .fragment(null)
                .path("/login/ott")
                .queryParam(
	                "token", 
	                oneTimeToken.getTokenValue());// (2)
                
                
        String magicLink = builder.toUriString();
        String email = getUserEmail(
	        oneTimeToken.getUsername()
	    ); // (3)
	    
        this.mailSender.send(
	        email, 
	        "Your Spring Security One Time Token", 
	        "Use the following link to sign in into the application: " + magicLink); // (4)
	        
        this.redirectHandler.handle(
	        request, 
	        response, 
	        oneTimeToken); // (5)
    }

    private String getUserEmail() {
        // ...
    }

}

@Controller
class PageController {

    @GetMapping("/ott/sent")
    String ottSent() {
        return "my-template";
    }
}
```

|№|Описание|
|---|---|
|(1)|Сделайте `MagicLinkOneTimeTokenGenerationSuccessHandler` Spring-бином.|
|(2)|Создайте URL-адрес обработки входа с токеном в качестве параметра запроса.|
|(3)|Получите адрес электронной почты пользователя на основе имени пользователя.|
|(4)|Используйте API `JavaMailSender` для отправки электронного письма пользователю с волшебной ссылкой.|
|(5)|Используйте `RedirectOneTimeTokenGenerationSuccessHandler` для выполнения перенаправления на желаемый URL-адрес.|

Содержимое электронного письма будет выглядеть примерно так:
```
Use the following link to sign in into the application: http://localhost:8080/login/ott?token=a830c444-29d8-4d98-9b46-6aba7b22fe5b
```

Страница отправки по умолчанию обнаружит, что URL-адрес содержит параметр запроса `token`, и автоматически заполнит поле формы значением токена.

## 1.4 Изменение URL-адреса генерации одноразового токена
По умолчанию `GenerateOneTimeTokenFilter` прослушивает запросы `POST /ott/generate`. Этот URL-адрес можно изменить с помощью DSL-метода `generateTokenUrl(String)`:

**Изменение URL-адреса генерации**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) {
        http
            // ...
            .formLogin(Customizer.withDefaults())
            .oneTimeTokenLogin((ott) -> ott
                .generateTokenUrl("/ott/my-generate-url")
            );
        return http.build();
    }

}

@Component
public class MagicLinkOneTimeTokenGenerationSuccessHandler implements OneTimeTokenGenerationSuccessHandler {
    // ...
}
```

## 1.5 Изменение URL-адреса страницы отправки по умолчанию

Страница отправки одноразового токена по умолчанию генерируется `DefaultOneTimeTokenSubmitPageGeneratingFilter` и прослушивает `GET /login/ott`. Этот URL-адрес также можно изменить следующим образом:

## 1.6 Отключение страницы отправки по умолчанию
Если вы хотите использовать свою собственную страницу отправки одноразового токена, вы можете отключить страницу по умолчанию, а затем предоставить свою собственную конечную точку.

**Отключение страницы отправки по умолчанию**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) {
        http
            .authorizeHttpRequests((authorize) -> authorize
                .requestMatchers("/my-ott-submit").permitAll()
                .anyRequest().authenticated()
            )
            .formLogin(Customizer.withDefaults())
            .oneTimeTokenLogin((ott) -> ott
                .showDefaultSubmitPage(false)
            );
        return http.build();
    }

}

@Controller
public class MyController {

    @GetMapping("/my-ott-submit")
    public String ottSubmitPage() {
        return "my-ott-submit";
    }

}

@Component
public class OneTimeTokenGenerationSuccessHandler implements OneTimeTokenGenerationSuccessHandler {
    // ...
}
```

## 1.7 Настройка способа генерации и использования одноразовых токенов
Интерфейс, определяющий общие операции для генерации и использования одноразовых токенов, — это `OneTimeTokenService`. Spring Security использует `InMemoryOneTimeTokenService` в качестве реализации этого интерфейса по умолчанию, если не предоставлено иное. Для производственных сред рекомендуется использовать `JdbcOneTimeTokenService`.

Некоторые из наиболее распространенных причин для настройки `OneTimeTokenService` включают, но не ограничиваются:
- Изменение времени истечения срока действия одноразового токена
    
- Сохранение дополнительной информации из запроса на генерацию токена
    
- Изменение способа создания значения токена
    
- Дополнительная проверка при использовании одноразового токена
    

Есть два варианта настройки `OneTimeTokenService`. Один из вариантов — предоставить его как бин, чтобы он мог быть автоматически подобран DSL-методом `oneTimeTokenLogin()`:

**Передача OneTimeTokenService как бина**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) {
        http
            // ...
            .formLogin(Customizer.withDefaults())
            .oneTimeTokenLogin(Customizer.withDefaults());
        return http.build();
    }

    @Bean
    public OneTimeTokenService oneTimeTokenService() {
        return new MyCustomOneTimeTokenService();
    }

}

@Component
public class MagicLinkOneTimeTokenGenerationSuccessHandler implements OneTimeTokenGenerationSuccessHandler {
    // ...
}
```

Второй вариант — передать экземпляр `OneTimeTokenService` в DSL, что полезно, если существует несколько `SecurityFilterChain` и для каждого из них требуется разный `OneTimeTokenService`.

**Передача OneTimeTokenService с помощью DSL**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) {
        http
            // ...
            .formLogin(Customizer.withDefaults())
            .oneTimeTokenLogin((ott) -> ott
                .oneTimeTokenService(new MyCustomOneTimeTokenService())
            );
        return http.build();
    }

}

@Component
public class MagicLinkOneTimeTokenGenerationSuccessHandler implements OneTimeTokenGenerationSuccessHandler {
    // ...
}
```

## 1.8 Настройка экземпляра GenerateOneTimeTokenRequest

Существует ряд причин, по которым вы можете захотеть настроить `GenerateOneTimeTokenRequest`. Например, вы можете захотеть установить `expiresIn` равным 10 минутам, в то время как Spring Security по умолчанию устанавливает 5 минут.

Вы можете настроить элементы `GenerateOneTimeTokenRequest`, опубликовав `GenerateOneTimeTokenRequestResolver` как `@Bean`, следующим образом:
```java
@Bean
GenerateOneTimeTokenRequestResolver generateOneTimeTokenRequestResolver() {
    DefaultGenerateOneTimeTokenRequestResolver delegate = new DefaultGenerateOneTimeTokenRequestResolver();
        return (request) -> {
		    GenerateOneTimeTokenRequest generate = delegate.resolve(request);
		    return new GenerateOneTimeTokenRequest(generate.getUsername(), Duration.ofSeconds(600));
	};
}
```
