# 1. Основная конфигурация (Core Configuration)
## 1.1 Пример с Spring Boot
Spring Boot предоставляет полные возможности автоконфигурации для OAuth 2.0 Login.

В этом разделе показано, как настроить [пример OAuth 2.0 Login](https://github.com/spring-projects/spring-security-samples/tree/main/servlet/spring-boot/java/oauth2/login) с использованием Google в качестве провайдера аутентификации, и рассматриваются следующие темы:

### 1.1.1 Начальная настройка
Чтобы использовать систему аутентификации Google OAuth 2.0 для входа, ==вы должны настроить проект в Google API Console== **для получения учетных данных OAuth 2.0.**

[Реализация OAuth 2.0 от Google](https://developers.google.com/identity/protocols/OpenIDConnect) для аутентификации соответствует спецификации [OpenID Connect 1.0](https://openid.net/connect/) и [сертифицирована OpenID](https://openid.net/certification/).

Следуйте инструкциям на странице [OpenID Connect](https://developers.google.com/identity/protocols/OpenIDConnect), начиная с раздела «Setting up OAuth 2.0».

После выполнения инструкций «Obtain OAuth 2.0 credentials» у вас должен появиться новый OAuth Client с учетными данными, состоящими из Client ID и Client Secret.

---
### 1.1.2 Настройка Redirect URI
**Redirect URI** — это ***путь в приложении***, на который пользовательский агент (user-agent) конечного пользователя **перенаправляется обратно после того, как пользователь прошел аутентификацию в Google** и предоставил доступ OAuth Client (созданному на предыдущем шаге) на странице согласия (Consent page).

В подразделе «Set a redirect URI» убедитесь, что поле **Authorized redirect URIs** установлено в значение `localhost:8080/login/oauth2/code/google`.

Шаблон redirect URI по умолчанию: `{baseUrl}/login/oauth2/code/{registrationId}`. 
> `registrationId` — это уникальный идентификатор для [`ClientRegistration`](https://../client/index.html#oauth2Client-client-registration).

Если OAuth Client работает за прокси-сервером, вам следует проверить [конфигурацию прокси-сервера](https://../features/exploits/http.html#http-proxy-server), чтобы убедиться, что приложение настроено правильно. Также см. поддерживаемые [переменные шаблона URI](https://../client/authorization-grants.html#oauth2Client-auth-code-redirect-uri) для `redirect-uri`.

---
### 1.1.3 Настройка application.yml
Теперь, когда у **вас есть новый OAuth Client с Google**, *вам нужно настроить приложение на использование этого OAuth Client* для процесса аутентификации. 

Для этого:
1. Перейдите в `application.yml` и установите следующую конфигурацию:
```yaml
spring:
  security:
    oauth2:
      client:
        registration: # (1)
          google: # (2)
            client-id: google-client-id
            client-secret: google-client-secret
```

| №   | Свойства OAuth Client                                                                                                                                             |
| --- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| (1) | `spring.security.oauth2.client.registration` — это ==базовый префикс свойства для свойств OAuth Client.==                                                         |
| (2) | За базовым префиксом свойства следует идентификатор для [`ClientRegistration`](https://../client/index.html#oauth2Client-client-registration), например `google`. |
    
2. Замените значения в свойствах `client-id` и `client-secret` на учетные данные OAuth 2.0, которые вы создали ранее.

---
### 1.1.4 Запуск приложения
Запустите пример Spring Boot и перейдите по адресу `localhost:8080`
Вы будете перенаправлены на автоматически сгенерированную страницу входа по умолчанию, на которой отображается ссылка для Google.

Нажмите на ссылку Google, и вы будете перенаправлены в Google для аутентификации.

После аутентификации с использованием учетных данных вашей учетной записи Google вы увидите экран согласия (Consent screen). Экран согласия спрашивает, разрешить ли или запретить доступ OAuth Client, который вы создали ранее. 
Нажмите **Allow**, чтобы разрешить OAuth Client доступ к вашему адресу электронной почты и основной информации профиля.

На этом этапе OAuth Client извлекает ваш адрес электронной почты и основную информацию профиля из [конечной точки UserInfo](https://openid.net/specs/openid-connect-core-1_0.html#UserInfo) и устанавливает аутентифицированную сессию.

## 1.2 Сопоставления свойств Spring Boot

В следующей таблице показано сопоставление свойств OAuth Client Spring Boot со свойствами [`ClientRegistration`](https://../client/index.html#oauth2Client-client-registration).

|`spring.security.oauth2.client.registration.[registrationId].client-name`|`clientName`|
|`spring.security.oauth2.client.provider.[providerId].authorization-uri`|`providerDetails.authorizationUri`|
|`spring.security.oauth2.client.provider.[providerId].token-uri`|`providerDetails.tokenUri`|
|`spring.security.oauth2.client.provider.[providerId].jwk-set-uri`|`providerDetails.jwkSetUri`|

## 1.3 CommonOAuth2Provider
`CommonOAuth2Provider` ==предопределяет набор свойств клиента по умолчанию== **для ряда известных провайдеров**: Google, GitHub, Facebook, X и Okta.

Например, `authorization-uri`, `token-uri` и `user-info-uri` не часто меняются для провайдера. Поэтому имеет смысл предоставить значения по умолчанию, чтобы уменьшить требуемую конфигурацию.

Как было продемонстрировано ранее, когда мы настраивали клиент Google, требуются только свойства `client-id` и `client-secret`.

Следующий листинг показывает пример:
```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          google:
            client-id: google-client-id
            client-secret: google-client-secret
```

> Автоматическое применение свойств клиента по умолчанию работает здесь без проблем, потому что `registrationId` (`google`) совпадает с перечислением `GOOGLE` (без учета регистра) в `CommonOAuth2Provider`.

Для случаев, когда вы можете захотеть указать другой `registrationId`, например `google-login`, вы все равно можете использовать автоматическое применение свойств клиента по умолчанию, настроив свойство `provider`.

Следующий листинг показывает пример:
```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          google-login: # (1)
            provider: google # (2)
            client-id: google-client-id
            client-secret: google-client-secret
```

|№|Описание|
|---|---|
|(1)|`registrationId` установлен в `google-login`.|
|(2)|Свойство `provider` установлено в `google`, что задействует автоматическое применение свойств клиента по умолчанию, заданных в `CommonOAuth2Provider.GOOGLE.getBuilder()`.|

## 1.4 Настройка свойств пользовательского провайдера
Существуют некоторые провайдеры OAuth 2.0, которые поддерживают мультитенантность (multi-tenancy), что приводит к различным конечным точкам протокола для каждого тенанта (или субдомена).

Например, OAuth Client, зарегистрированный в Okta, привязан к определенному субдомену и имеет свои собственные конечные точки протокола.

Для таких случаев Spring Boot предоставляет следующее базовое свойство **для настройки свойств пользовательского провайдера:** `spring.security.oauth2.client.provider.[providerId]`.

Следующий листинг показывает пример:
```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          okta:
            client-id: okta-client-id
            client-secret: okta-client-secret
        provider:
          okta: # (1)
            authorization-uri: https://your-subdomain.oktapreview.com/oauth2/v1/authorize
            token-uri: https://your-subdomain.oktapreview.com/oauth2/v1/token
            user-info-uri: https://your-subdomain.oktapreview.com/oauth2/v1/userinfo
            user-name-attribute: sub
            jwk-set-uri: https://your-subdomain.oktapreview.com/oauth2/v1/keys
```

|№|Описание|
|---|---|
|(1)|Базовое свойство (`spring.security.oauth2.client.provider.okta`) позволяет настраивать расположение конечных точек протокола.|

## 1.5 Переопределение автоконфигурации Spring Boot (application.yaml + переопределение стандартного поведения)
Классом автоконфигурации Spring Boot для поддержки OAuth Client является `OAuth2ClientAutoConfiguration`.

Он выполняет следующие задачи:
- Регистрирует бин `ClientRegistrationRepository@Bean`, состоящий из `ClientRegistration`(s) из настроенных свойств OAuth Client.
    
- Регистрирует бин `SecurityFilterChain@Bean` и включает OAuth 2.0 Login через `httpSecurity.oauth2Login()`.

Если вам нужно переопределить автоконфигурацию в соответствии с вашими конкретными требованиями, вы можете сделать это следующими способами:

### 1.5.1 Регистрация бина ClientRegistrationRepository
Следующий пример показывает, как зарегистрировать бин `ClientRegistrationRepository`:
```java
@Configuration
public class OAuth2LoginConfig {

	@Bean
	public ClientRegistrationRepository clientRegistrationRepository() {
		return new InMemoryClientRegistrationRepository(
			this.googleClientRegistration()
		);
	}

	private ClientRegistration googleClientRegistration() {
		return ClientRegistration.withRegistrationId("google")
			.clientId("google-client-id")
			.clientSecret("google-client-secret")
			.clientAuthenticationMethod(
				ClientAuthenticationMethod.CLIENT_SECRET_BASIC
			)
			.authorizationGrantType(
				AuthorizationGrantType.AUTHORIZATION_CODE
			)
			.redirectUri("{baseUrl}/login/oauth2/code/{registrationId}")
			.scope("openid", "profile", "email", "address", "phone")
			.authorizationUri(
				"https://accounts.google.com/o/oauth2/v2/auth"
			)
			.tokenUri("https://www.googleapis.com/oauth2/v4/token")
			.userInfoUri("https://www.googleapis.com/oauth2/v3/userinfo")
			.userNameAttributeName(IdTokenClaimNames.SUB)
			.jwkSetUri("https://www.googleapis.com/oauth2/v3/certs")
			.clientName("Google")
			.build();
	}
}
```

### 1.5.2 Регистрация бина SecurityFilterChain
Следующий пример показывает, как зарегистрировать бин `SecurityFilterChain` с `@EnableWebSecurity` и включить OAuth 2.0 login через `httpSecurity.oauth2Login()`:

**Конфигурация OAuth2 Login**
```java
@Configuration
@EnableWebSecurity //ОБЯЗАТЕЛЬНО!!!
public class OAuth2LoginSecurityConfig {

	@Bean
	public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
		http
			.authorizeHttpRequests((authorize) -> authorize
				.anyRequest().authenticated()
			)
			.oauth2Login(withDefaults());
		return http.build();
	}
}
```

### 1.5.3 Полное переопределение автоконфигурации
Следующий пример показывает, как полностью переопределить автоконфигурацию, зарегистрировав бин `ClientRegistrationRepository` и бин `SecurityFilterChain`.

**Переопределение автоконфигурации**
```java

	@Bean
	public ClientRegistrationRepository clientRegistrationRepository() {

			this.googleClientRegistration());
	}

	private ClientRegistration googleClientRegistration() {
		return ClientRegistration.withRegistrationId("google")
			.clientId("google-client-id")
			.clientSecret("google-client-secret")
			.clientAuthenticationMethod(
				ClientAuthenticationMethod.CLIENT_SECRET_BASIC
			)
			.authorizationGrantType(
				AuthorizationGrantType.AUTHORIZATION_CODE
			)
			.redirectUri("{baseUrl}/login/oauth2/code/{registrationId}")
			.scope("openid", "profile", "email", "address", "phone")
			.authorizationUri(
				"https://accounts.google.com/o/oauth2/v2/auth"
			)
			.tokenUri("https://www.googleapis.com/oauth2/v4/token")
			.userInfoUri("https://www.googleapis.com/oauth2/v3/userinfo")
			.userNameAttributeName(IdTokenClaimNames.SUB)
			.jwkSetUri("https://www.googleapis.com/oauth2/v3/certs")
			.clientName("Google")
			.build();
	}
}
```

## 1.6 Конфигурация Java без Spring Boot
Если вы не можете использовать Spring Boot и хотите настроить одного из предопределенных провайдеров в `CommonOAuth2Provider` (например, Google), примените следующую конфигурацию:

**Конфигурация OAuth2 Login**
```java
@Configuration
@EnableWebSecurity
public class OAuth2LoginConfig {

	@Bean
	public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
		http
			.authorizeHttpRequests((authorize) -> authorize
				.anyRequest().authenticated()
			)
			.oauth2Login(withDefaults());
		return http.build();
	}

	@Bean
	public ClientRegistrationRepository clientRegistrationRepository() {
		return new InMemoryClientRegistrationRepository(
			this.googleClientRegistration()
		);
	}
	private ClientRegistration googleClientRegistration() {
		return CommonOAuth2Provider.GOOGLE.getBuilder("google")
			.clientId("google-client-id")
			.clientSecret("google-client-secret")
			.build();
	}

	@Bean
	public OAuth2AuthorizedClientService authorizedClientService(
			ClientRegistrationRepository clientRegistrationRepository
	) {
		return new InMemoryOAuth2AuthorizedClientService(
			clientRegistrationRepository
		);
	}

	@Bean
	public OAuth2AuthorizedClientRepository authorizedClientRepository(
			OAuth2AuthorizedClientService authorizedClientService
	) {
		return new AuthenticatedPrincipalOAuth2AuthorizedClientRepository(
			authorizedClientService
		);
	}
}
```
