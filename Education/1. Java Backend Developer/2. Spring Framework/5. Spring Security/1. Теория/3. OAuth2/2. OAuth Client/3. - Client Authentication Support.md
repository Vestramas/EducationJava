# 1. Client Credentials (Учетные данные клиента)

## 1.1 Аутентификация с использованием `client_secret_basic`
**Аутентификация клиента с помощью HTTP Basic поддерживается «из коробки»**, и ==для ее включения не требуется никакой настройки. ==

**Реализация по умолчанию предоставляется классом** `DefaultOAuth2TokenRequestHeadersConverter`.

При следующих свойствах Spring Boot для регистрации OAuth 2.0 клиента:
```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          okta:
            client-id: client-id
            client-secret: client-secret
            client-authentication-method: client_secret_basic
            authorization-grant-type: authorization_code
            ...
```

Следующий пример показывает, как настроить `RestClientAuthorizationCodeTokenResponseClient` для отключения URL-кодирования учетных данных клиента:
```java
DefaultOAuth2TokenRequestHeadersConverter<OAuth2AuthorizationCodeGrantRequest> headersConverter =
	new DefaultOAuth2TokenRequestHeadersConverter<>();
	
headersConverter.setEncodeClientCredentials(false);


//
RestClientAuthorizationCodeTokenResponseClient tokenResponseClient =
	new RestClientAuthorizationCodeTokenResponseClient();

tokenResponseClient.setHeadersConverter(headersConverter);
```

# 2. JWT Bearer
Обратитесь к разделу JSON Web Token (JWT) Profile for OAuth 2.0 Client Authentication and Authorization Grants для получения дополнительных сведений о JWT Bearer Client Authentication.
(https://datatracker.ietf.org/doc/html/rfc7523#section-2.2)

==Реализацией по умолчанию для JWT Bearer Client Authentication является ==
`NimbusJwtClientAuthenticationParametersConverter`, который представляет собой `Converter`, **настраивающий параметры токен-запроса путем добавления подписанного JSON** Web Token (JWS) в параметр `client_assertion`.

`java.security.PrivateKey` или `javax.crypto.SecretKey`, используемый для подписи JWS, предоставляется резолвером `com.nimbusds.jose.jwk.JWK`, связанным с `NimbusJwtClientAuthenticationParametersConverter`.

## 2.1 Аутентификация с использованием `private_key_jwt`
При следующих свойствах Spring Boot для регистрации OAuth 2.0 клиента:
```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          okta:
            client-id: okta-client-id
            client-authentication-method: private_key_jwt
            authorization-grant-type: authorization_code
            ...
```

Следующий пример показывает, как настроить `RestClientAuthorizationCodeTokenResponseClient`:
```java
Function<ClientRegistration, JWK> jwkResolver = (clientRegistration) -> {
	if (clientRegistration
		.getClientAuthenticationMethod()
		.equals(
			ClientAuthenticationMethod.PRIVATE_KEY_JWT
		)
	) {
		// Предполагая RSA тип ключа
		RSAPublicKey publicKey = ...
		RSAPrivateKey privateKey = ...
		return new RSAKey.Builder(publicKey)
				.privateKey(privateKey)
				.keyID(UUID.randomUUID().toString())
				.build();
	}
	return null;
};

RestClientAuthorizationCodeTokenResponseClient tokenResponseClient =
	new RestClientAuthorizationCodeTokenResponseClient();

tokenResponseClient.addParametersConverter(
	new NimbusJwtClientAuthenticationParametersConverter<>(jwkResolver));
```

## 2.2 Аутентификация с использованием `client_secret_jwt`
При следующих свойствах Spring Boot для регистрации OAuth 2.0 клиента:
```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          okta:
            client-id: okta-client-id
            client-secret: okta-client-secret
            client-authentication-method: client_secret_jwt
            authorization-grant-type: client_credentials
            ...
```

Следующий пример показывает, как настроить `RestClientClientCredentialsTokenResponseClient`:
```java
Function<ClientRegistration, JWK> jwkResolver = (clientRegistration) -> {
	if (clientRegistration
		.getClientAuthenticationMethod()
		.equals(
			ClientAuthenticationMethod.CLIENT_SECRET_JWT
		)
	) {
		SecretKeySpec secretKey = new SecretKeySpec(
				clientRegistration
					.getClientSecret()
						.getBytes(StandardCharsets.UTF_8),
				"HmacSHA256");
		return new OctetSequenceKey.Builder(secretKey)
				.keyID(UUID.randomUUID().toString())
				.build();
	}
	return null;
};

RestClientClientCredentialsTokenResponseClient tokenResponseClient =
	new RestClientClientCredentialsTokenResponseClient();

tokenResponseClient.addParametersConverter(
	new NimbusJwtClientAuthenticationParametersConverter<>(jwkResolver));
```

## 2.3 Настройка JWT-утверждения (assertion)
JWT, создаваемый `NimbusJwtClientAuthenticationParametersConverter`, по умолчанию содержит утверждения (claims) `iss`, `sub`, `aud`, `jti`, `iat` и `exp`. Вы можете настроить заголовки и/или утверждения, предоставив `Consumer<NimbusJwtClientAuthenticationParametersConverter.JwtClientAuthenticationContext<T>>` в `setJwtClientAssertionCustomizer()`. 

Следующий пример показывает, как настроить утверждения JWT:
```java
Function<ClientRegistration, JWK> jwkResolver = ...
NimbusJwtClientAuthenticationParametersConverter<OAuth2ClientCredentialsGrantRequest> converter =
	new NimbusJwtClientAuthenticationParametersConverter<>(jwkResolver);
	
converter.setJwtClientAssertionCustomizer((context) -> 
{
	context
		.getHeaders()
			.header("custom-header", "header-value");
			
	context
		.getClaims()
			.claim("custom-claim", "claim-value");
});
```

# 3. Публичная аутентификация (Public Authentication)
**Публичная аутентификация клиента поддерживается «из коробки»**, ==и для ее включения не требуется никакой настройки.==

Следующие свойства Spring Boot для регистрации OAuth 2.0 клиента демонстрируют конфигурацию:
```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          okta:
            client-id: client-id
            client-authentication-method: none
            authorization-grant-type: authorization_code
            ...
```

Публичные клиенты поддерживаются с использованием **Proof Key for Code Exchange (PKCE)**. PKCE будет автоматически использоваться, когда `client-authentication-method` установлен в `"none"` (`ClientAuthenticationMethod.NONE`).
