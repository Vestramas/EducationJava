В этом разделе рассматриваются дополнительные возможности, предоставляемые Spring Security для OAuth2 клиента.

# 1. Получение авторизованного клиента (Resolving an Authorized Client)
Аннотация `@RegisteredOAuth2AuthorizedClient` **предоставляет возможность преобразовывать параметр метода в значение аргумента типа** `OAuth2AuthorizedClient`. 

Это удобная альтернатива по сравнению с доступом к `OAuth2AuthorizedClient` через `OAuth2AuthorizedClientManager` или `OAuth2AuthorizedClientService`. 

Следующий пример показывает, как использовать `@RegisteredOAuth2AuthorizedClient`:
```java
@Controller
public class OAuth2ClientController {

	@GetMapping("/")
	public String index(
		@RegisteredOAuth2AuthorizedClient("okta") OAuth2AuthorizedClient authorizedClient
	) {
		OAuth2AccessToken accessToken = authorizedClient
			.getAccessToken();

		...

		return "index";
	}
}
```

Аннотация `@RegisteredOAuth2AuthorizedClient` обрабатывается `OAuth2AuthorizedClientArgumentResolver`, который напрямую использует `OAuth2AuthorizedClientManager` и, следовательно, наследует его возможности.

# 2. Интеграция с RestClient
Поддержка `RestClient` обеспечивается классом `OAuth2ClientHttpRequestInterceptor`. 
> Этот ***перехватчик предоставляет возможность делать запросы к защищенным ресурсам***, **помещая Bearer-токен в заголовок** `Authorization` и**сходящего запроса.** 

Перехватчик напрямую использует `OAuth2AuthorizedClientManager` и, следовательно, наследует следующие возможности:

- **Выполняет запрос OAuth 2.0 Access Token для получения** `OAuth2AccessToken`, **если клиент еще не был авторизован.**
    - `authorization_code`: Запускает перенаправление Authorization Request для инициирования потока.
        
    - `client_credentials`: Токен доступа получается напрямую с Token Endpoint.
        
- Дополнительные типы грантов поддерживаются путем включения расширенных типов грантов.
    
- Если существующий `OAuth2AccessToken` истек, он обновляется (или продлевается).


Следующий пример использует `OAuth2AuthorizedClientManager` по умолчанию для настройки `RestClient`, способного получать доступ к защищенным ресурсам, помещая Bearer-токены в заголовок `Authorization` каждого запроса:

**Настройка `RestClient` с `ClientHttpRequestInterceptor`**
```java
@Configuration
public class RestClientConfig {

	@Bean
	public RestClient restClient(
		OAuth2AuthorizedClientManager authorizedClientManager
	) {
		OAuth2ClientHttpRequestInterceptor requestInterceptor =
			new OAuth2ClientHttpRequestInterceptor(
				authorizedClientManager
			);

		return RestClient.builder()
				.requestInterceptor(requestInterceptor)
				.build();
	}

}
```

## 2.1 Предоставление `clientRegistrationId`

`OAuth2ClientHttpRequestInterceptor` **использует** `ClientRegistrationIdResolver` ***для определения того, какой клиент используется для получения токена доступа.*** 

По умолчанию `RequestAttributeClientRegistrationIdResolver` используется для получения `clientRegistrationId` из `HttpRequest#attributes()`.

Следующий пример демонстрирует предоставление `clientRegistrationId` через атрибуты:

**Предоставление `clientRegistrationId` через атрибуты**
```java
import static org.springframework.security.oauth2.client.web.client.RequestAttributeClientRegistrationIdResolver.clientRegistrationId;

@Controller
public class ResourceController {

	private final RestClient restClient;

	public ResourceController(RestClient restClient) {
		this.restClient = restClient;
	}

	@GetMapping("/")
	public String index() {
		String resourceUri = "...";

		String body = this.restClient.get()
			.uri(resourceUri)
			.attributes(clientRegistrationId("okta"))   //(1)
			.retrieve()
			.body(String.class);

		// ...

		return "index";
	}
}
```

|№|Описание|
|---|---|
|(1)|`clientRegistrationId()` — это статический метод в `RequestAttributeClientRegistrationIdResolver`.|

В качестве альтернативы можно предоставить пользовательский `ClientRegistrationIdResolver`. 

Следующий пример настраивает пользовательскую реализацию, которая получает `clientRegistrationId` от текущего пользователя.

**Настройка `ClientHttpRequestInterceptor` с пользовательским `ClientRegistrationIdResolver`**
```java
@Configuration
public class RestClientConfig {

	@Bean
	public RestClient restClient(
	OAuth2AuthorizedClientManager authorizedClientManager
	) {
		OAuth2ClientHttpRequestInterceptor requestInterceptor =
		new OAuth2ClientHttpRequestInterceptor(
			authorizedClientManager
		);
		requestInterceptor
			.setClientRegistrationIdResolver(
				clientRegistrationIdResolver()
			);

		return RestClient.builder()
				.requestInterceptor(requestInterceptor)
				.build();
	}


	private static ClientRegistrationIdResolver clientRegistrationIdResolver() 
	{
		return (request) -> {
			Authentication authentication = 
			SecurityContextHolder
				.getContext().getAuthentication();
				
			return (authentication 
				instanceof 
				OAuth2AuthenticationToken principal) 
			? rincipal
				.getAuthorizedClientRegistrationId() 
			: null;
		};
	}

}
```

## 2.2 Предоставление `principal`
`OAuth2ClientHttpRequestInterceptor` использует `PrincipalResolver` **для определения того, какое имя принципала связано с токеном доступа**, что позволяет приложению выбирать, как ограничить область действия сохраняемого `OAuth2AuthorizedClient`

По умолчанию `SecurityContextHolderPrincipalResolver` используется для получения текущего принципала из `SecurityContextHolder`.

В качестве альтернативы `principal` может быть получен из `HttpRequest#attributes()` путем настройки `RequestAttributePrincipalResolver`, как показано в следующем примере:

**Настройка `ClientHttpRequestInterceptor` с `RequestAttributePrincipalResolver`**
```java
@Configuration
public class RestClientConfig {

	@Bean
	public RestClient restClient(
		OAuth2AuthorizedClientManager authorizedClientManager
	) {
		OAuth2ClientHttpRequestInterceptor 
		requestInterceptor =
			new OAuth2ClientHttpRequestInterceptor(
				authorizedClientManager
			);
			
		requestInterceptor.setPrincipalResolver(
			new RequestAttributePrincipalResolver()
		);

		return RestClient.builder()
				.requestInterceptor(requestInterceptor)
				.build();
	}

}
```

Следующий пример демонстрирует предоставление имени `principal` через атрибуты, что ограничивает область действия `OAuth2AuthorizedClient` приложением, а не текущим пользователем:

**Предоставление имени `principal` через атрибуты**
```java
import static org.springframework.security.oauth2.client.web.client.RequestAttributeClientRegistrationIdResolver.clientRegistrationId;
import static org.springframework.security.oauth2.client.web.client.RequestAttributePrincipalResolver.principal;

@Controller
public class ResourceController {

	private final RestClient restClient;

	public ResourceController(RestClient restClient) {
		this.restClient = restClient;
	}

	@GetMapping("/")
	public String index() {
		String resourceUri = "...";

		String body = this.restClient.get()
				.uri(resourceUri)
				.attributes(clientRegistrationId("okta"))
				.attributes(principal("my-application"))   // (1)
				.retrieve()
				.body(String.class);

		// ...

		return "index";
	}

}
```

|№|Описание|
|---|---|
|(1)|`principal()` — это статический метод в `RequestAttributePrincipalResolver`.|

## 2.3 Обработка ошибок (Handling Failure)
**Если токен доступа недействителен** по какой-либо причине (например, истекший токен), **может быть полезно обработать ошибку**, удалив токен доступа, чтобы его нельзя было использовать снова. 

==Вы можете настроить перехватчик делать это автоматически,==**предоставив** `OAuth2AuthorizationFailureHandler` для удаления токена доступа.

Следующий пример использует `OAuth2AuthorizedClientRepository` для настройки `OAuth2AuthorizationFailureHandler`, который удаляет недействительный `OAuth2AuthorizedClient` в контексте `HttpServletRequest`:

**Настройка `OAuth2AuthorizationFailureHandler` с использованием `OAuth2AuthorizedClientRepository`**
```java
@Configuration
public class RestClientConfig {

	@Bean
	public RestClient restClient(
		OAuth2AuthorizedClientManager 
			authorizedClientManager,
		OAuth2AuthorizedClientRepository 
			authorizedClientRepository
	) {

		OAuth2ClientHttpRequestInterceptor requestInterceptor =
			new OAuth2ClientHttpRequestInterceptor(
				authorizedClientManager
			);

		OAuth2AuthorizationFailureHandler 
		authorizationFailureHandler =
			OAuth2ClientHttpRequestInterceptor
				.authorizationFailureHandler(
					authorizedClientRepository
				);
		requestInterceptor
			.setAuthorizationFailureHandler(
				authorizationFailureHandler
			);

		return RestClient.builder()
				.requestInterceptor(requestInterceptor)
				.build();
	}

}
```

В качестве альтернативы **можно использовать** `OAuth2AuthorizedClientService` **для удаления недействительного** `OAuth2AuthorizedClient` вне контекста `HttpServletRequest`, как показано в следующем примере:

**Настройка `OAuth2AuthorizationFailureHandler` с использованием `OAuth2AuthorizedClientService`**
```java
@Configuration
public class RestClientConfig {

	@Bean
	public RestClient restClient(
	OAuth2AuthorizedClientManager authorizedClientManager,
	OAuth2AuthorizedClientService authorizedClientService
	) {

		OAuth2ClientHttpRequestInterceptor requestInterceptor =
			new OAuth2ClientHttpRequestInterceptor(
				authorizedClientManager
			);

		OAuth2AuthorizationFailureHandler 
		authorizationFailureHandler =
			OAuth2ClientHttpRequestInterceptor
			.authorizationFailureHandler(
				authorizedClientService
			);
		requestInterceptor
			.setAuthorizationFailureHandler(
				authorizationFailureHandler
			);

		return RestClient.builder()
				.requestInterceptor(requestInterceptor)
				.build();
	}

}
```

# 3. Интеграция с WebClient для сред сервлетов
Поддержка OAuth 2.0 Client интегрируется с `WebClient` с помощью `ExchangeFilterFunction`.

`ServletOAuth2AuthorizedClientExchangeFilterFunction` предоставляет механизм для запроса защищенных ресурсов с использованием `OAuth2AuthorizedClient` и включения связанного `OAuth2AccessToken` в качестве Bearer-токена. 

Он напрямую использует `OAuth2AuthorizedClientManager` и, следовательно, наследует следующие возможности:
- Запрашивается `OAuth2AccessToken`, если клиент еще не был авторизован.
    
    - `authorization_code`: Запускает перенаправление Authorization Request для инициирования потока.
        
    - `client_credentials`: Токен доступа получается напрямую с Token Endpoint.
        
- Если `OAuth2AccessToken` истек, он обновляется (или продлевается), если доступен `OAuth2AuthorizedClientProvider` для выполнения авторизации.
    

Следующий код показывает пример того, как настроить `WebClient` с поддержкой OAuth 2.0 Client:
```java
@Bean
WebClient webClient(OAuth2AuthorizedClientManager authorizedClientManager) {
	ServletOAuth2AuthorizedClientExchangeFilterFunction oauth2Client =
			new ServletOAuth2AuthorizedClientExchangeFilterFunction(authorizedClientManager);
	return WebClient.builder()
			.apply(oauth2Client.oauth2Configuration())
			.build();
}
```

---
## 3.1 Предоставление авторизованного клиента
`ServletOAuth2AuthorizedClientExchangeFilterFunction` определяет клиента для использования (для запроса), получая `OAuth2AuthorizedClient` из `ClientRequest.attributes()` (атрибутов запроса).

Следующий код показывает, как установить `OAuth2AuthorizedClient` в качестве атрибута запроса:
```java
@GetMapping("/")
public String index(@RegisteredOAuth2AuthorizedClient("okta") OAuth2AuthorizedClient authorizedClient) {
	String resourceUri = ...

	String body = webClient
			.get()
			.uri(resourceUri)
			.attributes(oauth2AuthorizedClient(authorizedClient))   // (1)
			.retrieve()
			.bodyToMono(String.class)
			.block();

	...

	return "index";
}
```

|№|Описание|
|---|---|
|(1)|`oauth2AuthorizedClient()` — это статический метод в `ServletOAuth2AuthorizedClientExchangeFilterFunction`.|

Следующий код показывает, как установить `ClientRegistration.getRegistrationId()` в качестве атрибута запроса:
```java
@GetMapping("/")
public String index() {
	String resourceUri = ...

	String body = webClient
			.get()
			.uri(resourceUri)
			.attributes(clientRegistrationId("okta"))   // (1)
			.retrieve()
			.bodyToMono(String.class)
			.block();

	...

	return "index";
}
```

|№|Описание|
|---|---|
|(1)|`clientRegistrationId()` — это статический метод в `ServletOAuth2AuthorizedClientExchangeFilterFunction`.|

Следующий код показывает, как установить `Authentication` в качестве атрибута запроса:
```java
@GetMapping("/")
public String index() {
	String resourceUri = ...

	Authentication anonymousAuthentication = new AnonymousAuthenticationToken(
			"anonymous", "anonymousUser", AuthorityUtils.createAuthorityList("ROLE_ANONYMOUS"));
	String body = webClient
			.get()
			.uri(resourceUri)
			.attributes(authentication(anonymousAuthentication))   // (1)
			.retrieve()
			.bodyToMono(String.class)
			.block();

	...

	return "index";
}
```

|№|Описание|
|---|---|
|(1)|`authentication()` — это статический метод в `ServletOAuth2AuthorizedClientExchangeFilterFunction`.|

> Рекомендуется проявлять осторожность с этой функцией, поскольку все HTTP-запросы будут получать токен доступа, привязанный к предоставленному принципалу.

## 3.2 Выбор авторизованного клиента по умолчанию
Если ни `OAuth2AuthorizedClient`, ни `ClientRegistration.getRegistrationId()` не предоставлены в качестве атрибута запроса, `ServletOAuth2AuthorizedClientExchangeFilterFunction` может определить, какой клиент использовать по умолчанию, в зависимости от своей конфигурации.

Если настроено `setDefaultOAuth2AuthorizedClient(true)` и пользователь аутентифицировался с помощью `HttpSecurity.oauth2Login()`, используется `OAuth2AccessToken`, связанный с текущим `OAuth2AuthenticationToken`.

Следующий код показывает конкретную конфигурацию:
```java
@Bean
WebClient webClient(
OAuth2AuthorizedClientManager authorizedClientManager
) {
	ServletOAuth2AuthorizedClientExchangeFilterFunction oauth2Client =
		new ServletOAuth2AuthorizedClientExchangeFilterFunction(
		authorizedClientManager
	);
	
	oauth2Client.setDefaultOAuth2AuthorizedClient(true);
	return WebClient.builder()
			.apply(oauth2Client.oauth2Configuration())
			.build();
}
```

> Будьте осторожны с этой функцией, поскольку все HTTP-запросы получают токен доступа.

В качестве альтернативы, если настроено `setDefaultClientRegistrationId("okta")` с допустимым `ClientRegistration`, используется `OAuth2AccessToken`, связанный с `OAuth2AuthorizedClient`.

Следующий код показывает конкретную конфигурацию:
```java
@Bean
WebClient webClient(
OAuth2AuthorizedClientManager authorizedClientManager
) {
	ServletOAuth2AuthorizedClientExchangeFilterFunction oauth2Client =
			new ServletOAuth2AuthorizedClientExchangeFilterFunction(
			authorizedClientManager
		);
	oauth2Client.setDefaultClientRegistrationId("okta");
	return WebClient.builder()
			.apply(oauth2Client.oauth2Configuration())
			.build();
}
```

> Будьте осторожны с этой функцией, поскольку все HTTP-запросы получают токен доступа.
