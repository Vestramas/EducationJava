Authorization Server — это не то, с чем сталкивается каждый разработчик. 
> **Большинство пользуется готовыми решениями** (Keycloak, Auth0, Okta). 

==Spring Security Authorization Server нужен==, когда вы **создаете свой собственный сервер авторизации**.

### Что можно разобрать
Эти страницы содержат ключевые концепции, без понимания которых вы не сможете настроить даже простейший Authorization Server.

| Страница                  | Действие                                 | Почему                                                                                                                                                                                                                     |
| ------------------------- | ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Index (корень)**        | 📝 **Перевести и законспектировать**     | Содержит список всех поддерживаемых фич (Authorization Code, Client Credentials, DPoP, PKCE, mutual-TLS). Это ваша карта возможностей.                                                                                     |
| **Getting Started**       | 📝 **Перевести полностью с пояснениями** | Самый важный раздел! Здесь показана **минимальная конфигурация** для запуска Authorization Server. Вы должны понять, зачем нужны `RegisteredClientRepository`, `JWKSource`, `AuthorizationServerSettings` — это фундамент. |
| **Configuration Model**   | 📝 **Перевести и разобрать**             | Описывает, как кастомизировать **каждую конечную точку** (authorization, token, introspection, revocation). Без этого вы не сможете настроить сервер под свои нужды.                                                       |
| **Core Model Components** | ⏸️ **Оставить как справочник**           | Там перечислены компоненты (`RegisteredClient`, `OAuth2Authorization`, `OAuth2Token`). Это важно, но читать это лучше **перед практикой**, а не вместо нее.                                                                |
| **Protocol Endpoints**    | ⏸️ **Оставить как справочник**           | Детальные настройки каждой конечной точки. Это чистая документация API. Вы будете обращаться к ней, когда начнете кастомизировать конкретный endpoint.                                                                     |
