# 1. Зависимости
Будем разрабатывать MVC приложение, которое будет защищено. Приложение будет запускаться на сервере Tomcat.

После создания модуля, в родительском pom (SecurityExamples), можно будет описать общие зависимости, а также указывать, какие версии других зависимостей будут использоваться.

Так например, все проекты у нас будут строится на основе Spring MVC + Spring Security

# 2. Docker-compose
Чтобы проще было запускать наше приложение Spring Framework (без Boot), мы поднимем в докере Tomcat сервер, и сделаем том на папку Webapps (где будет находиться war файл приложения) в корне нашего приложения

```yaml
version: "3.5"  
services:  
  security_tomcat:  
    image: tomcat:10.1.50-jdk17-temurin-jammy  
    container_name: security_tomcat  
    ports:  
      - "8081:8080"  
    volumes:  
      - ./volumes/tomcat/webapps:/usr/local/tomcat/webapps  
  
volumes:  
  webapps:
```
