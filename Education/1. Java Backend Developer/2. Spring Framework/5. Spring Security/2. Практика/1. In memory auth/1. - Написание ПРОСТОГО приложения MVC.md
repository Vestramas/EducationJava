Напишем простое MVC приложение, которое потом будет "защищать" при помощи Spring Security

---
# 0. ИДЕЯ
Мы будем писать один проект, который будет разбит на логические состовляющие - пакеты

В каждом из пакетов будет описана своя конфигурация, в зависимости от реализуемого метода Spring Security

Так, мы создадим пакеты внутри главного пакета:
- пакет1
- пакет2
- пакет3 
- и так далее

И в корне главного пакета, будет находиться класс, который будет сканировать определенный пакет, тем самым меняя контекст и конфигурацию.

Этот класс, будет указан в качестве класса конфигурации сервлетов в классе конфигурации `DispatcherServelt` при помощи метода `getServletConfigClasses()`

# 2. Класс конфигурации Web приложения
Теперь опишем "главный" класс приложения, в котором будет находится конфигурация Web приложения

А именно, у нас тут будет находится класс, **который указывает где брать Views для отображения пользователю**

```java
package security_examples.in_memory_auth.config.web;  
  
import org.springframework.context.annotation.Configuration;  
import org.springframework.web.servlet.config.annotation.EnableWebMvc;  
import org.springframework.web.servlet.config.annotation.ViewResolverRegistry;  
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;  
  
@Configuration  
@EnableWebMvc  
public class WebConfig implements WebMvcConfigurer {  
    @Override  
    public void configureViewResolvers(ViewResolverRegistry registry) {  
        registry.jsp("/pages/", ".jsp");  
    }  
}
```

# 4. Главный класс приложения
Теперь, напишем класс, который будет **СОСТАВЛЯТЬ КОНТЕКСТ ПРИЛОЖЕНИЯ по сканируемому пакету**
```java
package security_examples;  
  
import org.springframework.context.annotation.AnnotationConfigApplicationContext;  
import org.springframework.context.annotation.ComponentScan;  
import org.springframework.context.annotation.Configuration;  
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;  
  
@Configuration  
@ComponentScan({"security_examples.in_memory_auth"})  
public class SpringSecurityMainApp implements WebMvcConfigurer {  
}
```

Метод main не нужен, так как Tomcat запускает приложение, а не наш класс. 
Когда приложение будет запущено, Spring создаст контекст, а затем создается DispathcerSerlvet который достает из контекста Spring нужные классы

# 5. Конфигурация DispatcherServlet
Так как мы используем Java-based конфигурацию, а не web.xml для конфигурации DispatcherServlet, то нужно написать код.
> (см [[2. Практическое резюме - Java конфигурация]] для Spring MVC)

```java
package security_examples.in_memory_auth.config.web;  
  
import security_examples.SpringSecurityMainApp;  
import org.springframework.web.servlet.support.AbstractAnnotationConfigDispatcherServletInitializer;  
  
public class DispatcherServletConfig extends AbstractAnnotationConfigDispatcherServletInitializer {  
    @Override  
    protected Class<?>[] getRootConfigClasses() {  
        return new Class[0];  
    }  
  
    @Override  
    protected Class<?>[] getServletConfigClasses() {  
        return new Class[]{SpringSecurityMainApp.class};  
    }  
  
    @Override  
    protected String[] getServletMappings() {  
        return new String[]{"/"};  
    }  
}
```
