# 1. Класс конфигурации безопаности
Напишем класс, в котором будет происходить самая простая аутентификация. При помощи In Memory Auth
```java
package security_examples.in_memory_auth.config.security;  
  
import org.springframework.context.annotation.Bean;  
import org.springframework.context.annotation.Configuration;  
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;  
import org.springframework.security.core.userdetails.User;  
import org.springframework.security.core.userdetails.UserDetailsService;  
import org.springframework.security.provisioning.InMemoryUserDetailsManager;  
  
@Configuration  
@EnableWebSecurity  
public class WebSecurityConfig {  
  
    @Bean  
    public UserDetailsService userDetailsService() {  
        InMemoryUserDetailsManager manager =  
                new InMemoryUserDetailsManager();  
        manager  
            .createUser(  
                User.withDefaultPasswordEncoder()  
                    .username("user")  
                    .password("password")  
                    .roles("USER")  
                    .build()  
            );  
        return manager;  
    }  
}
```


Также можно явно указать цепочку фильтрации:
```java
@Bean  
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {  
    http  
        .authorizeHttpRequests(auth -> auth  
            .requestMatchers("/login").permitAll()  
            .anyRequest().authenticated()  
        )  
        .httpBasic(Customizer.withDefaults());  
  
    return http.build();  
}
```

> Такой подход работает только на этапе разработке.

# 2. Регистрация бина springSecurityFilterChain
Как мы уже знаем, весь Sping Security работает на фильтрах. И Tomcat (и другие контейнеры сервлетов) также имеют фильтры.

Поэтому, чтобы отработали фильтры безопасности, нужно чтобы фильтр из контейнера сервлетов, вызвал цепочку фильтров Security.

Для этого нужно зарегистрировать бин `springSecurityFilterChain` (тип DelegatingFilterProxy), который содержит в себе фильтры безопасности

Именно этот бин, и используется для вызова бина, который содержит цепочку фильтров безопасности 


Чтобы  **сделать это** в конфигурации Java **с помощью поддержки** `WebApplicationInitializer` Spring в среде Servlet 3.0+.

Но так как мы уже реализовали `WebApplicationInitializer` 
(через класс `DispatcherServletConfig`, который ПО ЦЕПОЧКЕ реализует `WebApplicationInitializer`)
```java
public class DispatcherServletConfig extends AbstractAnnotationConfigDispatcherServletInitializer {  
    @Override  
    protected Class<?>[] getRootConfigClasses() {  
        return new Class[0];  
    }  
  
    @Override  
    protected Class<?>[] getServletConfigClasses() {  
        return new Class[]{SpringSecurityMainApp.class};  
    }  
  
    @Override  
    protected String[] getServletMappings() {  
        return new String[]{"/"};  
    }  
}
```

То нам нужно зарегистрировать наш фильтр в контексте
Для этого делаем это:
```java
public class SecurityWebApplicationInitializer  
        extends AbstractSecurityWebApplicationInitializer {  
}
```

И все!
(см [[2. - Конфигурация Java-based#1.3 AbstractSecurityWebApplicationInitializer с Spring MVC]])

Хотя если сделать как  [[2. - Конфигурация Java-based#1.2 AbstractSecurityWebApplicationInitializer без существующего Spring]] то это тоже сработает, и никакой ошибки не будет

## 2.1 ГЛАВНОЕ УСЛОВИЕ ОДНОГО КОНТЕКСТА
Главное, чтобы класс конфигурации приложения (`WebConfig`) И класс конфигурации безопасности (`WebSecurityConfig`) , ДОЛЖНЫ БЫТЬ В ОДНОМ КОНТЕКСТЕ

Но так как мы контекстом управляем через сканирование в классе `SpringSecurityMainApp`, который сканирует весь пакет, и составляет контекст, и этот класс (`SpringSecurityMainApp`) указан в `DispatcherServlet` в одном контексте, то мы не нарушили правило одного контекста

# 3. Доработка контроллера
Добавим логику, которая будет отображать данные текущего пользователя, который заходит на страницу
```java
package security_examples.in_memory_auth.controller;  
  
import org.springframework.security.authentication.UsernamePasswordAuthenticationToken;  
import org.springframework.security.core.Authentication;  
import org.springframework.security.core.context.SecurityContextHolder;  
import org.springframework.stereotype.Controller;  
import org.springframework.web.bind.annotation.GetMapping;  
  
@Controller  
public class MainController {  
  
    @GetMapping("/hello")  
    public String helloPageHtml() {  
        Authentication clientAuth = 
			SecurityContextHolder
		        .getContext()
		        .getAuthentication();  
  
        //Получение данных текущего пользователя  
        String username = clientAuth.getName();  
        String principal = clientAuth.getPrincipal().toString();  
          
        System.out.println(
	        "Username: " + username + 
	        "\nPassword: " + principal
	    );  
  
        return "hello_page";  
    }  
}
```
