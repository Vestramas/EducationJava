# 1. Model 
Опишем нашу модель данных, которая будет сохраняться в БД.

Разобьем модель данных, на две составляющие:
- Базовая инфа о сохраняемом объекте
- Информацию пользователя

Это удобно, так как при добавлении новых классов, не придется "раздувать" новые классы, а просто наследоваться от базового класса, содержащего общую сохраняемую инфу

## 1.1 BaseEntity
Опишем базовую сущность, которая будет определять общие параметры записи

> [!NOTE] `@MappedSuperclass`
> Используется **для обозначения класса, чья информация о сопоставлении с базой данных применяется к его подклассам**, ***но сам класс не является сущностью (Entity) и не имеет отдельной таблицы в базе данных***. 


```java
package basic_auth_hardcoded_one_role.model;  
  
import jakarta.persistence.*;  
import lombok.Getter;  
import lombok.Setter;  
import org.hibernate.annotations.CreationTimestamp;  
import org.hibernate.annotations.UpdateTimestamp;  
import org.springframework.data.annotation.CreatedBy;  
import org.springframework.data.annotation.LastModifiedBy;  
  
import java.time.LocalDateTime;  
  
@MappedSuperclass  
@Setter  
@Getter  
public abstract class BaseEntity {  
    @Id  
    @GeneratedValue(strategy = GenerationType.IDENTITY)  
    private Integer id;  
  
    @CreationTimestamp  
    @Column(name = "created_at")  
    private LocalDateTime createdDateTime;  
  
    @UpdateTimestamp  
    @Column(name = "updated_at")  
    private LocalDateTime updatedDateTime;  
  
    @CreatedBy  
    @Column(name = "created_by")  
    private String createdBy;  
  
    @LastModifiedBy  
    @Column(name = "updated_by")  
    private String updatedBy;  
	
	//Чтобы использовать в User
    protected BaseEntity() {}  
    protected BaseEntity(  
            LocalDateTime createdDateTime,  
            LocalDateTime updatedDateTime,  
            String createdBy,  
            String updatedBy) {  
        this.createdDateTime = createdDateTime;  
        this.updatedDateTime = updatedDateTime;  
        this.createdBy = createdBy;  
        this.updatedBy = updatedBy;  
    }  
}
```

И от этого АБСТРАКТНОГО класса, будем наследоваться, и дополнять специфическими полями

Заранее создадим перечисление, которое будет хранить роли пользователей

Эти роли должны попасть в `authorities` (тип `GrantedAuthority`) в объекте аутентифицированного пользователя [`Authentication`]([[0. - Компоненты Аутентификации#2.4 Authentication]])

Для этого, создадим enum и реализуем интерфейс `GrantedAuthority`
```java
package basic_auth_hardcoded_one_role.model;  
  
import org.springframework.security.core.GrantedAuthority;  
  
//Чтобы когда UserDetailService загружал пользователя, то он заполнял его роли в Authentication  
public enum Role implements GrantedAuthority {  
    ROLE_USER, 
    ROLE_ADMIN;  
  
    @Override  
    public String getAuthority() {  
        return name();  
    }  
}
```

```java
package basic_auth_hardcoded_one_role.model;  
  
import jakarta.persistence.*;  
import lombok.Getter;  
import lombok.Setter;  
  
import java.time.LocalDateTime;  
  
@Entity  
@Getter  
@Setter  
@Table(name = "users")  
public class User extends BaseEntity{  
    private String name;  
    private String username;  
    private String password;  
  
    @Enumerated(EnumType.STRING)  
    private Role role;  
  
    private boolean isEnabled;  
  
    public User(String name, String username, String password, Role role, boolean isEnabled) {  
        super(  
                LocalDateTime.now(),  
                LocalDateTime.now(),  
                username,  
                username  
        );  
        this.name = name;  
        this.username = username;  
        this.password = password;  
        this.role = role;  
        this.isEnabled = isEnabled;  
    }  
    public User() {}  
}
```

Так как этот класс, уже будет сохраняться в БД, его мы помечаем аннотацией `@Entity` и указываем название таблицы `@Table`.

Класс наследуется от `BaseEntity`, чтобы включить в себя те базовые поля


# 4. Конфигурация JPA + HIbernate
Теперь, когда у нас есть данные которые мы будем сохранять, нужно сконфигурировать JPA + Hibernate, так с их помощью мы будем сохранять данные

## 4.1 Описание конфигурационных свойств
Опишем файл `.properties`, в котором будут указаны параметры для подключения к БД, настройки Hibernate и JPA

```properties
# ---------- DataSource ----------  
db.url=jdbc:postgresql://host.docker.internal:5532/spring-security  
db.username=postgres  
db.password=postgres  
db.driver=org.postgresql.Driver  
  
# ---------- Hibernate ----------  
hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect  
hibernate.hbm2ddl.auto=update  
hibernate.show_sql=true  
hibernate.format_sql=true  
  
# ---------- JPA ----------  
jpa.packagesToScan=basic_auth_hardcoded_one_role.model
```

#### Hibernate
- **`hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect`**  
    Указывает Hibernate, с какой **системой управления базами данных (СУБД)** он работает.  
    `PostgreSQLDialect` содержит правила перевода HQL (Hibernate Query Language) в SQL, специфичный для PostgreSQL.  
    Благодаря этому Hibernate генерирует корректные для вашей базы данных запросы (например, правильный синтаксис для автоинкремента, работы с последовательностями и т.д.).
    
- **`hibernate.hbm2ddl.auto=update`**  
    Определяет, как Hibernate управляет **схемой базы данных** при запуске.  
    Значение `update` означает: *«Сравни текущую структуру таблиц в БД с описанной в Entity-классах. Если появились новые поля — добавь их. Если структура совпадает — ничего не делай»*.  
    Это позволяет **сохранять существующие данные** при перезапуске, в отличие от `create`, который полностью пересоздаёт таблицы.
    
- **`hibernate.show_sql=true`**  
    Включает вывод **всех SQL-запросов**, которые Hibernate генерирует и отправляет в базу данных.  
    Вы будете видеть их в консоли или логах. Это очень полезно при разработке и отладке, чтобы понимать, что именно происходит на уровне SQL.
    
- **`hibernate.format_sql=true`**  
    Делает выводимые SQL-запросы **форматированными и читаемыми**.  
    Вместо длинной строки в одну линию вы увидите красивый многострочный SQL с отступами, который легко читать и анализировать.

#### JPA
- **`jpa.packagesToScan=basic_auth_hardcoded_one_role.model`**  
    Указывает Spring-контексту, в **каком пакете искать Entity-классы** (***классы, помеченные аннотацией*** `@Entity`).  
    Spring просканирует пакет `basic_auth_hardcoded_one_role.model` и все его подпакеты, найдет ваши Entity-классы и передаст их Hibernate для управления.


> [!WARNING] Postgres на компе
> Так как **Postgres мы НЕ ЗАПУСКАЕМ В КОНТЕЙНЕРЕ,** то ***чтобы сервер из Docker, мог обращаться к БД на компе***, 
> **необходимо в пути указывать НЕ** `localhost:5532`, а `host.docker.internal:5532`

## 4.2 Класс конфигурации JPA
Для этого создадим класс `JpaConfig` в пакете `config`

Чтобы наш класс конфигурации подтянул данные из этого файла, нужно создать бин с типом `PropertySourcesPlaceholderConfigurer`

```java
@Bean  
public static PropertySourcesPlaceholderConfigurer properties() {  
    PropertySourcesPlaceholderConfigurer configurer =  
            new PropertySourcesPlaceholderConfigurer();  
  
    configurer.setLocation(  
		new FileSystemResource(
			"/usr/local/tomcat/webapps/config/application.properties"
		)  
    );  
  
    return configurer;  
}
```

> [!WARNING] ОБЯЗАТЕЛЬНО `STATIC`
> ОБЯЗАТЕЛЬНО использовать спецификатор `static`, так как этот бин должен появиться до того, как соберется бин `JpaConfig`, так как мы будем использовать аннотацию `@Value` для чтения из файла `.properties`. 
> 
> А чтобы считать данные из файла, ПУТЬ ДО ЭТОГО ФАЙЛА ДОЛЖЕН БЫТЬ ИЗВЕСТЕН

```java
package basic_auth_hardcoded_one_role.config.data;  
  
import org.springframework.beans.factory.annotation.Value;  
import org.springframework.context.annotation.Bean;  
import org.springframework.context.annotation.Configuration;  
import org.springframework.context.support.PropertySourcesPlaceholderConfigurer;  
import org.springframework.core.io.FileSystemResource;  
import org.springframework.data.jpa.repository.config.EnableJpaRepositories;  
import org.springframework.jdbc.datasource.DriverManagerDataSource;  
import org.springframework.orm.jpa.JpaTransactionManager;  
import org.springframework.orm.jpa.LocalContainerEntityManagerFactoryBean;  
import org.springframework.orm.jpa.vendor.HibernateJpaVendorAdapter;  
import org.springframework.transaction.PlatformTransactionManager;  
import org.springframework.transaction.annotation.EnableTransactionManagement;  
  
import javax.sql.DataSource;  
import java.util.Properties;  
  
@Configuration  
@EnableTransactionManagement  
@EnableJpaRepositories("basic_auth_hardcoded_one_role.repository")  
public class JpaConfig {  
  
    //---------- Data Source ----------  
    @Value("${db.url}")  
    private String url;  
  
    @Value("${db.username}")  
    private String username;  
  
    @Value("${db.password}")  
    private String password;  
  
    @Value("${db.driver}")  
    private String driver;  
  
    //---------- Hibernate ----------  
    @Value("${hibernate.dialect}")  
    private String dialect;  
  
    @Value("${hibernate.hbm2ddl.auto}")  
    private String ddl;  
  
    @Value("${hibernate.show_sql}")  
    private String showSql;  
  
    @Value("${hibernate.format_sql}")  
    private String formatSql;  
  
    @Value("${jpa.packagesToScan}")  
    private String packagesToScan;  
  
    @Bean  
    public static PropertySourcesPlaceholderConfigurer properties() {  
        PropertySourcesPlaceholderConfigurer configurer =  
                new PropertySourcesPlaceholderConfigurer();  
  
        configurer.setLocation(  
			new FileSystemResource(
				"/usr/local/tomcat/webapps/config/application.properties"
			)  
        );  
  
        return configurer;  
    }  
  
    // ---------- 1. DataSource (данные для подключени к БД) ----------  
    @Bean  
    public DataSource dataSource() {  
        DriverManagerDataSource ds = new DriverManagerDataSource();  
  
        ds.setDriverClassName(driver);  
        ds.setUrl(url);  
        ds.setUsername(username);  
        ds.setPassword(password);  
  
        return ds;  
    }  
  
    // ---------- 2. EntityManagerFactory ----------  
    /*
	    Для описания фабрики создания EntityManager, которые 
	    управляют сущностями (Entity): создают, удаляют, обновляют 
	    и прочие операции
	    
	    В качестве вендора (реалзиации/адаптера) используется Hibernate
    */
    @Bean  
    public LocalContainerEntityManagerFactoryBean entityManagerFactory() {  
        LocalContainerEntityManagerFactoryBean emf =  
                new LocalContainerEntityManagerFactoryBean();  
  
        emf.setDataSource(dataSource());  
        emf.setPackagesToScan(packagesToScan);  
  
        HibernateJpaVendorAdapter vendorAdapter = 
	        new HibernateJpaVendorAdapter(); 
	         
        emf.setJpaVendorAdapter(vendorAdapter);  
  
        Properties props = new Properties();  
        props.put("hibernate.dialect", dialect);  
        props.put("hibernate.hbm2ddl.auto", ddl);  
        props.put("hibernate.show_sql", showSql);  
        props.put("hibernate.format_sql", formatSql);  
  
        emf.setJpaProperties(props);  
  
        return emf;  
    }  
  
    // ---------- 3. TransactionManager ----------  
    /*
	    Указываем менеджер транзакций
    */
    @Bean  
    public PlatformTransactionManager transactionManager() {  
        JpaTransactionManager txManager =  
                new JpaTransactionManager();  
  
        txManager.setEntityManagerFactory(
	        entityManagerFactory().getObject()
	    );  
  
        return txManager;  
    }  
}
```

# 5. Описание сервиса
Теперь, опишем сервис, который будет использовать репозиторий для работы с БД
> Стандартная архитектура MVC, чтобы контроллер не содержал бизнес логику

Нам нужно всего несколько методов:
- Создание пользователя
- Получение всех пользователей
- Получение одного пользователя по имени (для Spring Security в дальнейшем)

```java
package basic_auth_hardcoded_one_role.service;  
  
import basic_auth_hardcoded_one_role.dto.UserDto;  
import basic_auth_hardcoded_one_role.model.User;  
import basic_auth_hardcoded_one_role.repository.UserRepository;  
import basic_auth_hardcoded_one_role.util.mapper.UserMapper;  
import org.springframework.beans.factory.annotation.Autowired;  
import org.springframework.stereotype.Service;  
  
import java.time.LocalDateTime;  
import java.time.ZoneId;  
import java.util.List;  
import java.util.Optional;  
import java.util.stream.Collectors;  
  
@Service  
public class UserService {  
    private final UserRepository userRepository;  
    private final UserMapper mapper;  
  
    @Autowired  
    public UserService(
	    UserRepository userRepository, 
	    UserMapper userMapper
	) {  
        this.userRepository = userRepository;  
        this.mapper = userMapper;  
    }  
  
    public List<UserDto> getUsers() {  
        return userRepository.findAll()
			.stream()
				.map( u-> 
					mapper.userToUserDto(u)
				)
				.collect(Collectors.toList());  
    }  
  
    public Optional<User> getByUserName(String userName) {  
        return userRepository.getUserByUsername(userName);  
    }  
  
    public UserDto createUser(UserDto dto) {  
        dto.setCreatedDateTime(
	        LocalDateTime.now(ZoneId.of("Europe/Moscow"))
	    );  
        dto.setCreatedBy(dto.getUsername());  
  
        User user = userRepository.save(mapper.userDtoToUser(dto));  
        return mapper.userToUserDto(user);  
    }  
}
```

## 6.2 Jackson - для конвертации DTO из JSON
Теперь, сконфигурируем Jackson, чтобы приложение могло конвертировать JSON->Object и наоборот

Плюс, нам нужно настроить ObjectMapper, чтобы он:
- Поддерживал десериализацию времени
- Работал по политике нейминга полей `snake_case`

```java
package basic_auth_hardcoded_one_role.config.web;  
  
import com.fasterxml.jackson.databind.ObjectMapper;  
import com.fasterxml.jackson.databind.PropertyNamingStrategies;  
import com.fasterxml.jackson.datatype.jsr310.JavaTimeModule;  
import org.springframework.context.annotation.Configuration;  
import org.springframework.http.converter.HttpMessageConverter;  
import org.springframework.http.converter.json.MappingJackson2HttpMessageConverter;  
import org.springframework.web.servlet.config.annotation.EnableWebMvc;  
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;  
  
import java.util.List;  
  
@Configuration  
@EnableWebMvc  
public class WebConfig implements WebMvcConfigurer {  
    @Override  
    public void configureMessageConverters(
	    List<HttpMessageConverter<?>> converters
    ) {  
        ObjectMapper objectMapper = new ObjectMapper();  
          
        //Поддержка десериализации времени  
        objectMapper.registerModule(new JavaTimeModule());  
          
        // Устанавливаем стратегию именования  
        objectMapper.setPropertyNamingStrategy(
	        PropertyNamingStrategies.SNAKE_CASE
	    );  
  
          
        MappingJackson2HttpMessageConverter jsonConverter =  
                new MappingJackson2HttpMessageConverter(objectMapper);  
  
        converters.add(jsonConverter);  
    }  
}
```
