Producer можно настроить на транзакционность и идемпонетность, для того чтобы
- обеспечить семантику exactly-once

## 1. Параметры конфигурации фабрки продюсеров
Чтобы сконфигурировать транзакционного продюсера, необходимо в свойства добавить два важных параметра
```java
props.put(  
	ProducerConfig.TRANSACTIONAL_ID_CONFIG,  
	"transact-id-1");  
props.put(  
	ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG,  
	true);
```

### Что делает каждый параметр:
|Параметр|Значение|
|---|---|
|`transactional.id`|Уникальный идентификатор транзакционного Producer. Kafka сохраняет состояние транзакций по нему. Без этого Producer не считается транзакционным.|
|`enable.idempotence`|Автоматически включает retry + sequence number. Требование Kafka для транзакций.|

⚠️ **Если указать `transactional.id`, Kafka автоматически включает идемпотентность**, но явное указание не вредит.

## 2. Префикс для Spring приложения (Spring-specific)
Spring Kafka поддерживает **пул продюсеров**, создаваемых `DefaultKafkaProducerFactory`.

Если приложение масштабируется (несколько потоков или несколько продюсеров), то необходимо задать **префикс**.

Для этого, можно установить префикс в фабрике продюсера
```java
DefaultKafkaProducerFactory<String, UserInfo> factory =  
	new DefaultKafkaProducerFactory<>(props); 

// 2️⃣Добавление префикса для транзационных продюсеров 
factory.setTransactionIdPrefix("transaction-id-prefix-");
```

В результате Spring создаст транзакционные ID вида:
```java
transaction-id-prefix-0
transaction-id-prefix-1
transaction-id-prefix-2
...
```

Префикс можно задать:
- в `ProducerFactory`
- или в `KafkaTemplate` (эффект одинаковый)

## 3. Управление транзакциями
Чтобы управлять транзакциями в Prodcer, есть 3 способа:
1) Используя менеджер транзакций и @Transactional
2) Используя локальные транзакции через метод `kafkaTemplate.executeInTransactiona()`
3) Синхронизация с существующими транзакциями 

### Используя менеджер транзакций и @Transactional 
Это самый часто используемый способ управления транзакцией чисто для Kafka

Для его использования необходимо две вещи:
- Создать бин менеджера транзакций `KafkaTransactionManager`
- Аннотировать метод отправки сообщения `@Transactional`

В результате, при помощи аннотации `@Transactional` Spring создаст proxy-объект, который будет управлять транзакцией через менеджер транзакций. А внутри этого менеджера, вызывается метод, описанный в классе для отправки сообщения
(см [[9. - РЕЗЮМЕ (habr)]])

#### 1. Создание бина и конфигурация фабрики продюсера
```java
@Bean  
public ProducerFactory<String, UserInfo> producerFactory(){  
    Map<String, Object> props = new HashMap<>();  
    props.put(  
            ProducerConfig.BOOTSTRAP_SERVERS_CONFIG,  
            "localhost:29092,localhost:39092,localhost:49092"  
    );  
    props.put(  
            ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,  
            StringSerializer.class  
    );  
    props.put(  
            ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,  
            CustomUserInfoSerializer.class  
    );  
  
    // 1️⃣Добавление Producer ID для транзакций и включение идемпотентности 
    props.put(  
            ProducerConfig.TRANSACTIONAL_ID_CONFIG,  
            "transact-id-1");  
    props.put(  
            ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG,  
            true);  
  
    DefaultKafkaProducerFactory<String, UserInfo> factory =  
            new DefaultKafkaProducerFactory<>(props); 
    
    // 2️⃣Добавление префикса для транзационных продюсеров 
    factory.setTransactionIdPrefix("transaction-id-prefix-");  
    return factory;  
}

@Bean("transactionKafkaManager")   
public KafkaTransactionManager<String, UserInfo> kafkaTransactionManager( 
        ProducerFactory<String, UserInfo> producerFactory) {  
    return new KafkaTransactionManager<>(producerFactory);  
}

// KafkaTempalte использует ту же фабрику, что и менеджер транзакций
@Bean  
public KafkaTemplate<String, UserInfo> kafkaTemplate(  
        ProducerFactory<String, UserInfo> producerFactory){  
    return new KafkaTemplate<>(producerFactory);  
}
```

#### 2. Использование аннотации @Transactional 
После этого, можно использовать транзакционную отправку
```java
@Transactional("transactionKafkaManager")  
public void produceMessage(){   
	try {  
		kafkaTemplate.send(message).get(10, TimeUnit.SECONDS);  
	} catch (Exception e) {  
		System.out.println(
			"Exception on send Message in transaction");  
		e.printStackTrace();  
	}  
```

> [!NOTE] Для работы `@Transactional ` нужна анотация `@EnableTransactionManagement` над классом конфигурации
> 


В остальном, конфигурация Producer идентичная обычному Producer.

### 2. Использование локальной транзакции
Также, можно использовать локальные Kafka транзакции.

Это **прямой низкоуровневый доступ** к транзакции Producer.
> ==Spring не участвует==, только Kafka.
> ==менеджер транзакций НЕ нужен== для локальной транзакции


Для этого также необходимы те же параметры для фабрики Producer.

Но вместо использования аннотации `@Transactional` используется метод kafkaTemplate:
```java
kafkaTemplate.executeInTransaction()
```
в параметр метода передается функциональный интерфейс

Блок кода внутри выполняется внутри транзакции
```java
public void produceMessage(){  
	System.out.println("[PRODUCER] In produceMessage");  

	kafkaTemplate.executeInTransaction(kt -> {  
		try {  
			Message<UserInfo> message = MessageBuilder  
				.withPayload(
					new UserInfo(
						"Tom in Transaction",
						100))  
				.setHeader(
					KafkaHeaders.TOPIC, 
					"topic-spring-1")  
				.setHeader(
					"StringHeader", 
					"string")  
				.setHeader(
					"CustomHeader", 
					new CustomHeader(
						"transactionHeader", 
						UUID.randomUUID(), 
						17))  
				.build();  

			kt.send(message).get(10, TimeUnit.SECONDS);  

		} catch (Exception e) {  
			System.out.println(
			"Exception on send Message in transaction");  
		}
```

#### Особенности:
|Свойство|Значение|
|---|---|
|Spring НЕ создаёт proxy|Да|
|НЕ участвует в declarative transactions|Да|
|Используется чистая Kafka транзакция|Да|
|Можно использовать в любом месте|Да|

Обычно — лучший способ, если не нужна интеграция с БД.

### 3. Синхронизация транзакции Kafka с существующей
Типовой сценарий:
> В одном методе сохраняем в БД и отправляем в Kafka.  
> Если ошибка — откатываем и БД, и Kafka.

Spring позволяет «привязать» Kafka Producer к существующей транзакции (например, JDBC).


Для этого в Spring Kafka есть механизм - Синхронизация Kafka транзакций

Если уже есть транзакция к некому ресурсу, то 
- если Producer настроен на транзакционность
- тогда транзакция к Kafka будет выполняться внутри существующей открытой транзакции

**Для этого достаточно** чтобы фабрика Producer была настроена на транзакционность и:
- либо НЕ БЫЛО менеджера транзакций для Kafka
- либо в аннотации @Transactional был явно указан менеджер транзакций НЕ ОТ Kafka

Тогда Spring будет откатывать транзакции к некому ресурсу и Kafka

#### Важные нюансы синхронизации
|Особенность|Описание|
|---|---|
|Работает **только в send()**|executeInTransaction использовать нельзя|
|Требует транзакционный Producer|`transactional.id` обязателен|
|Spring создаёт «транзакционный синхронизатор»|Привязывает Kafka транзакцию к JDBC транзакции|
|Работает только при `send()` через `KafkaTemplate`|Низкоуровневый Producer не подходит|
