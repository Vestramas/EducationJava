# 2. Создание файла конфигурации

Создадим папку `src/main/resources` если ее нет.
В этой папке создадим файл `hibernate.properties`
```
# Config Data Source for Hibernate
hibernate.driver_class=org.postgresql.Driver
hibernate.connection.url=jdbc:postgresql://localhost:5432/myHibernateDB
hibernate.connection.username=postgres
hibernate.connection.password=12345
  
# Config Hibernate
hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect
hibernate.show_sql=true
```

> [!NOTE] ВНИМАНИЕ!
> Ранее, при использовании чистого `Hibernate` мы использовали `hibernate.current_session_context_class=thread`
> 
> ОДНАКО при ИСПОЛЬЗОВАНИИ `Spring` нам НЕ НУЖНО использовать это свойство, поскольку `Spring` сам реализует ==SpringSessionContext==

# 3. Подключение Hibernate к Spring
Для этого необходимо создать бин-компонент `Hibernate` в конфигурационном классе Spring, для того чтобы Spring смог сам его использовать

Для начала добавим следующие аннотации к классу конфигуратору
```java
@PropertySource("classpath:hibernate.properties")
@EnableTransactionManagement
```

> [!NOTE] Аннотации
> `@PropertySource("classpath:hibernate.properties")` - указываем ИСТОЧНИК данных (откуда брать инфу)
> 
> `@EnableTransactionManagement` - СПЕЦИАЛЬНАЯ ==аннотация== для того, ==чтобы== `Spring` ==САМ УПРАВЛЯЛ ТРАНЗАКЦИЯМИ== (открывал, выполнял, закрывал)

```java
public class AppContext implements WebMvcConfigurer{
	private final ApplicationContext applicationContext;
	private final Environment environment; // Специальный класс
	
	@Autowired
	public AppContext(ApplicationContext applicationContext, Environment environment) 
	{
		this.applicationContext = applicationContext;
		this.environment = environment; // Внедрение класса
	}
```

> [!NOTE] Класс `Enviroment`
> Данный класс нужен для того, чтобы работать с файлами `.properties`

Теперь необходимо настроить ==функцию, которая возвращает ИСТОЧНИК ДАННЫХ==. 
В ней мы ==УКАЗЫВАЕМ== какие ==ПАРАМЕТРЫ НЕОБХОДИМО ИСПОЛЬЗОВАТЬ== для ==ПОДКЛЮЧЕНИЯ К БД==
```java
@Bean
public DataSource dataSource() {
	DriverManagerDataSource driverManagerDataSource = 
		new DriverManagerDataSource();
		
	driverManagerDataSource.setDriverClassName(
		Objects.requireNonNull(
			environment.getProperty("hibernate.driver_class")
		)
	);
	
	driverManagerDataSource.setUrl(
		environment.getProperty("hibernate.connection.url")
	);

	driverManagerDataSource.setUsername(
		environment.getProperty("hibernate.connection.username")
	);
	
	driverManagerDataSource.setPassword(
		environment.getProperty("hibernate.connection.password")
	);

	return driverManagerDataSource;
}
```

Теперь необходимо создать функции для КОНФИГУРАЦИИ самого `Hibernate`.
***
1. Для начала ==необходимо ДОСТАТЬ конфигурационные параметры== из файла `hibernate.properties`
Здесь мы указываем диалект БД (SQL), и опцию для отображения SQL-запросов 
```java
private Properties hibernateProperties() {
	Properties properties = new Properties();
	
	properties.put(
		"hibernate.dialect", 
		environment.getRequiredProperty("hibernate.dialect"));
	
	properties.put(
		"hibernate.show_sql", 
		environment.getRequiredProperty("hibernate.show_sql"));
	
	return properties;
}
```

***
2. Теперь необходимо написать ==функцию для получения БИНА Сессии==

- `setPackagesToScan()` используется для определения пакета классов, которые необходимо отслеживать `Hibernate`'ом. 
- `setDataSource()` указываем сессии какую БД мы используем и данные для подключения
- `setHibernateProperties()` указываем параметры САМОГО `Hibernate`

Отслеживаться будут классы, ПОМЕЧЕННЫЕ АННОТАЦИЕЙ `@Entity`
```java
@Bean
public LocalSessionFactoryBean sessionFactory() {
	LocalSessionFactoryBean sessionFactory = new LocalSessionFactoryBean();
	
	sessionFactory.setDataSource(dataSource());
	sessionFactory.setPackagesToScan("ru.my.spring.crud.myModel");
	sessionFactory.setHibernateProperties(hibernateProperties());
	
	return sessionFactory;
}
```
***
3. Чтобы дать Spring'у ВОЗМОЖНОСТЬ САМОСТОЯТЕЛЬНО УПРАВЛЯТЬ ТРАНЗАКЦИЯМИ, мы должны сказать ему как это делать.
Мы указываем что при создании транзакции, необходимо использовать `sessionFactory()`, ==для получения необходимой сессии и связывании ее с транзакцией==
```java
@Bean
public PlatformTransactionManager hibernateTransactionManager() {
	HibernateTransactionManager transactionManager = new HibernateTransactionManager();
	
	transactionManager.setSessionFactory(sessionFactory().getObject());
	
	return transactionManager;
}
```

# 6. Изменение класса Person под класс-сущность

Поскольку в Hibernate мы работаем с классами как с сущностями в БД, необходимо изменить обычный класс Java под класс-сущность, с которым Hibernate сможет работать

```java
@Entity
@Table(name = "Persons")
public class Person {
	@Id
	@Column(name = "p_id")
	@GeneratedValue(strategy = GenerationType.IDENTITY)
	private int id;
	
	@Column(name = "p_name")
	@NotEmpty(message = "Name can`t not be empty!")
	@Size(min = 2, max = 30, message = "Name can be length between 2 and 30 character!")
	private String name;
	
	@Column(name = "p_email")
	@NotEmpty(message = "Email can`t not be empty!")
	@Email(message = "Email should be valid format!")
	private String email;
	
	@Column(name = "p_age")
	@Min(value = 0, message = "Age can`t be negative value!")
	private int age;
	
	@Column(name = "p_adress")
	@Pattern(regexp = "[A-Z]\\w+, [A-Z]\\w+, \\d{6}", message = "Format will be 'Coutry, City, Index(6 numbers)'")
	private String adress;
```

Теперь перепишем методы, которые ранее использовали `JdbcTemplate`

> [!NOTE] Транзакции в `Spring`
> Работа с транзакциями `Spring` отличается от `Hibernate`.
> 
> ==Транзакции== в `Spring` мы ==НЕ ОТКРЫВАЕМ САМОСТОЯТЕЛЬНО==, для этого ==используется аннотация== `@Transactional` из пакета `org.springframework.transaction.annotation.Transactional`
> 
> С ее помощью `Spring` ==будет следить за выполнением транзакции в методе== (открывать/выполнять/закрывать)
> 
> Если ИЗМЕНЕНИЯ в БД НЕ ВНОСЯТСЯ, то нужно указывать ПАРАМЕТР аннотации `readOnly = true`

```java
@Transactional(readOnly = true)
public List<Person> getList(){
	Session session = sessionFactory.getCurrentSession();
	
	List<Person> listPerson = session.createQuery("FROM Person", Person.class).getResultList();
	
	return listPerson;
}

@Transactional(readOnly = true)
public Person getPerson(int id) {
	Session session = sessionFactory.getCurrentSession();
	return session.get(Person.class, id);
}

@Transactional
public void saved(Person pers) {
	Session session = sessionFactory.getCurrentSession();
	session.save(pers);
}

@Transactional
public void update(int id, Person updatePerson) {
	Session session = sessionFactory.getCurrentSession();
	
	Person persOld = session.get(Person.class, id);
	
	persOld.setName(updatePerson.getName());
	persOld.setAge(updatePerson.getAge());
	persOld.setEmail(updatePerson.getEmail());
	persOld.setAdress(updatePerson.getAdress());
}

@Transactional
public void delete(int id) {
	Session session = sessionFactory.getCurrentSession();
	Person persDelete = session.get(Person.class, id);
	session.remove(persDelete);
}
```
