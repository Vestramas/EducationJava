Системы поиска, сбора, и визуализации логов/метрик/данных *строятся по 3-х уровневой архитектуре:*
1. Поисковой движок
2. Инструменты сбора данных
3. Инструменту визуализации

## 1. Поисковой движок / Хранилище данных
Это «мозг» системы. Он индексирует данные (логи, метрики, трейсы) и позволяет быстро искать по ним.

| Технология            | Назначение                                                                                                                 | Для чего используют                                                                                                                                                     |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Elastic<br>search** | Поисковый движок на базе Lucene (как у Solr). Хранит **документы в JSON**, строит инвертированные индексы.                 | Классический выбор **для** <br>- ***логов***, <br>- ***журналов событий***, <br>- ***любых текстовых данных***. <br><br>Умеет агрегации («сгруппируй ошибки по часам»). |
| **Grafana Loki**      | Горизонтально масштабируемое хранилище логов. Не индексирует содержимое логов, только **метки (labels)** как в Prometheus. | Когда логи вторичны по сравнению с метриками. <br><br>Экономит место, но поиск по тексту медленнее, чем в Elasticsearch.                                                |
| **Prometheus**        | Хранилище **метрик** (чисел с метками времени), а не логов. Своих логов не хранит.                                         | Мониторинг состояния CPU, памяти, RPS, ошибок. Логи туда не складывают.                                                                                                 |

> **Важно:** 
> - Loki и Prometheus часто работают в связке с Grafana, 
> - а Elasticsearch — с Kibana.

---
## 2. Инструменты сбора данных
Они забирают логи из файлов, сокетов, контейнеров, обрабатывают и отправляют в хранилище.

| Технология                                 | Назначение                                                                                                                                              | Для чего используют                                                                                           |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **Logstash**                               | **Тяжёлый конвейер обработки**: ввод → фильтры (grok, regex, mutate) → вывод. <br><br>Поддерживает 200+ плагинов.                                       | Сложная **обработка логов на лету** (парсинг, обогащение географией, переименование полей). Требует ресурсов. |
| **Beats** <br>(Filebeat, Metricbeat и др.) | **Лёгкие агенты-«шпионы».** Filebeat читает файлы логов, Metricbeat собирает метрики системы. <br><br>Отправляют напрямую в Elasticsearch или Logstash. | Установка на каждый сервер. <br><br>Для простого сбора без тяжелой обработки.                                 |
| **Promtail** <br>(для Loki)                | *Аналог Filebeat, но для Loki.* <br><br>Читает логи и добавляет к ним метки (например, `job=nginx`, `host=web-01`).                                     | Сбор логов в экосистеме Grafana Loki.                                                                         |
| **Grafana Agent**                          | Универсальный сборщик метрик, логов и трейсов для Grafana Stack. Может заменять Promtail и другие компоненты.                                           | Единый агент, если вы уже используете Grafana.                                                                |

> **Связки:**
> - Filebeat → (опционально Logstash) → Elasticsearch
>     
> - Promtail → Loki → Grafana
>     
> - Logstash → Elasticsearch (Logstash может и метрики принимать, но редко)
>

## 3. Визуализация данных

Веб-интерфейс для построения дашбордов, поиска и анализа.

| Технология  | Назначение                                                                                                               | Для чего используют                                                                    |
| ----------- | ------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------- |
| **Kibana**  | ==Родной интерфейс для Elasticsearch.== Умеет поиск по логам, дашборды, машинное обучение (платно).                      | Если хранилище — Elasticsearch, то Kibana — естественный выбор.                        |
| **Grafana** | ==Универсальный визуализатор.== <br><br>Подключается к Prometheus, Loki, Elasticsearch, PostgreSQL и ещё 50+ источникам. | Мониторинг «всё в одном»: метрики (Prometheus) + логи (Loki/Elastic) + трейсы (Tempo). |

> **Забавный факт:** 
> Grafana может работать поверх Elasticsearch — тогда Kibana не нужна. 
> 
> И наоборот, Loki можно смотреть через Kibana? Нет, только через Grafana.
