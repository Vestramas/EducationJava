## 1.2 Подключение к Elastic по HTTPS
А в самой документации указано, **что для безопасности используется HTTP Basic** ([Securing HTTP client applications | Elastic Docs](https://www.elastic.co/docs/deploy-manage/security/httprest-clients-security#http-clients))


Поэтому, чтобы **подключиться к ElasticSearch есть два пути:**
- *Отключить безопасность* через файл `/config/elasticsearch.yml` и указать в `xpack.security,enabled = false`
  (не рекомендуется в продакшене)
- *Создать пользователя, и ходить с Basic Auth Token*

### 1.2.1 Создание пользователя
Мы отключать безопасность не будем, поэтому создадим пользователя через встроенные утилиты
```bash
elasticsearch-setup-passwords -h
```


### 2.1.1 Автоматическое конфигурирование
Для автоматической настройки, нужно получить `enrollment token `(регистрационный токен)

Чтобы его получить, воспользуемся подсказкой ниже окна ввода. Там будем команда
```bash
bin\elasticsearch-create-enrollment-token.bat --scope kibana
```
> НЕ ЗАБУДЬ УБРАТЬ РАСШИРЕНИЕ УТИЛИТЫ, если выполняешь в докере
