
Logstash — это серверный инструмент для сбора, обработки и маршрутизации данных. Он принимает события из разных источников, преобразует их и отправляет в системы хранения или анализа, чаще всего в Elasticsearch.

### Основная идея
1. Input — получение данных из источника.
    
2. Filter — преобразование и обогащение событий.
    
3. Output — отправка данных в целевую систему.

![[Pasted image 20260610163250.png]]

Logstash может **ДИНАМИЧЕСКИ ОБЪЕДИНЯТЬ ДАННЫЕ из разнородных источников и нормализовать их в места назначения** (destination) по вашему выбору. 

Для принятия и отправки данных из различных источников, используются кодеки (кодировщики)

### Для чего нужен Logstash
- централизованный сбор логов и событий;
    
- парсинг текстовых логов в структурированные поля;
    
- нормализация данных под ECS;
    
- маршрутизация данных в разные системы;
    
- обогащение события геоданными, метаданными и т.д.;
    
- буферизация и повышение устойчивости конвейера.

## Logstash и Elastic Cloud Serverless

Вы будете использовать выходной плагин Logstash для Elasticsearch (`logstash-output-elasticsearch`), чтобы отправлять данные в Elastic Cloud Serverless.

Обратите внимание на следующие отличия Elasticsearch Serverless от Elastic Cloud Hosted и самостоятельного (self-managed) Elasticsearch:

|Отличие|Пояснение|
|---|---|
|**API-ключи вместо паролей**|Используйте API-ключи для доступа к Elastic Cloud Serverless из Logstash, так как он не поддерживает нативную аутентификацию пользователей. Любые настройки безопасности, основанные на пользователях, в конфигурации вашего выходного плагина Elasticsearch будут проигнорированы и могут вызвать ошибки|
|**DLM вместо ILM**|Elastic Cloud Serverless использует потоки данных (data streams) и управление жизненным циклом данных (Data Lifecycle Management — DLM) вместо управления жизненным циклом индексов (Index Lifecycle Management — ILM). Любые настройки ILM в конфигурации вашего выходного плагина Elasticsearch будут проигнорированы и могут вызвать ошибки|
|**Мониторинг через интеграцию**|Мониторинг Logstash доступен через интеграцию Logstash в Elastic Observability на Elastic Cloud Serverless|
|**Известная проблема с портом**|Настройка `hosts` в `logstash-output-elasticsearch` по умолчанию использует порт `:9200`. Установите значение порта `:443` вместо этого|

# 1. Компоненты Logstash
Logstash построен на JVM и использует pipeline — конвейеры обработки данных.

Конвейер (pipeline) обработки событий Logstash состоит из трёх этапов: **вход (inputs) → фильтры (filters) → выход (outputs)**.

- **Входы (Inputs)** генерируют события.
    
- **Фильтры (Filters)** изменяют события.
    
- **Выходы (Outputs)** отправляют события дальше.

*Входы и выходы поддерживают* **кодеки (codecs)** — они позволяют кодировать или декодировать данные на входе или выходе из конвейера, без использования отдельного фильтра.

## Входы (Inputs)

> [!NOTE] То, откуда получаем данные
> **Входы** используются ==ДЛЯ ПОЛУЧЕНИЯ ДАННЫХ в Logstash==. 

Некоторые из наиболее часто используемых входов:

| Вход       | Описание                                                                                                                                                                                                                           |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **file**   | **Читает из файла** в файловой системе, очень похоже на команду UNIX `tail -0F`                                                                                                                                                    |
| **syslog** | **Слушает известный порт 514 для syslog-сообщений** и *парсит их согласно формату RFC3164*                                                                                                                                         |
| **redis**  | **Читает из Redis-сервера**, используя Redis-каналы и Redis-списки. <br><br>Redis часто используется как "брокер" (посредник) в централизованной установке Logstash, создавая очередь событий от удалённых "отправителей" Logstash |
| **beats**  | **Обрабатывает события, отправленные Beats** (например, Filebeat, Metricbeat)                                                                                                                                                      |

Для получения дополнительной информации о доступных входных плагинах обратитесь к [документации по Input Plugins](https://www.elastic.co/guide/en/logstash/current/input-plugins.html).

## Фильтры (Filters)

> [!NOTE] Операции обработки
> **Фильтры** — это промежуточные ==УСТРОЙСТВА ОБРАБОТКИ В КОНВЕЕРЕ== Logstash. 

Вы можете комбинировать фильтры с условными операторами (conditionals), чтобы выполнить действие над событием, если оно соответствует определённым критериям. Некоторые полезные фильтры:

| Фильтр     | Описание                                                                                                                                                                                                                                                                                                                               |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **grok**   | **Парсит и структурирует произвольный текст**. <br><br>В настоящее время *это лучший способ в Logstash преобразовать неструктурированные данные логов во что-то структурированное* и доступное для запросов. <br><br>Со 120 встроенными в Logstash паттернами, скорее всего, вы найдёте тот, который соответствует вашим потребностям! |
| **mutate** | Выполняет **общие преобразования над полями событий**. <br><br>Вы можете переименовывать, удалять, заменять и изменять поля в ваших событиях                                                                                                                                                                                           |
| **drop**   | **Полностью УДАЛЯЕТ событие**, например, события уровня DEBUG                                                                                                                                                                                                                                                                          |
| **clone**  | **СОЗДАЕТ КОПИЮ события,** возможно, добавляя или удаляя поля                                                                                                                                                                                                                                                                          |
| **geoip**  | Добавляет *информацию о географическом расположении IP-адресов* (также отображает потрясающие диаграммы в Kibana!)                                                                                                                                                                                                                     |

Для получения дополнительной информации о доступных фильтрах обратитесь к [документации по Filter Plugins](https://www.elastic.co/guide/en/logstash/current/filter-plugins.html).

## Выходы (Outputs)

> [!NOTE] Отправка данных к место назначения
> **Выходы** — это финальная фаза конвейера Logstash. Событие может пройти через несколько выходов, но после завершения обработки всеми выходами выполнение события считается законченным. 


Некоторые часто используемые выходы:

| Выход             | Описание                                                                                                                                                                                                                                                                  |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **elasticsearch** | **Отправляет данные событий в Elasticsearch**. Если вы планируете сохранять ваши данные в эффективном, удобном и легко доступном для запросов формате… Elasticsearch — это правильный путь. Период. Да, мы предвзяты :)                                                   |
| **file**          | **Записывает данные событий в файл** на диске                                                                                                                                                                                                                             |
| **graphite**      | **Отправляет данные событий в Graphite** — популярный инструмент с открытым исходным кодом *для хранения и **построения графиков метрик***                                                                                                                                |
| **statsd**        | **Отправляет данные событий в StatsD** — сервис, который "слушает статистику, такую как счётчики и таймеры, отправленную по UDP, и отправляет агрегаты в один или несколько подключаемых бэкенд-сервисов". Если вы уже используете StatsD, это может быть полезно для вас |

Для получения дополнительной информации о доступных выходах обратитесь к [документации по Output Plugins](https://www.elastic.co/guide/en/logstash/current/output-plugins.html).

## Кодеки (Codecs)

> [!NOTE] Как сериализуем данные
> **Кодеки** — это, по сути, потоковые фильтры, которые могут работать как часть входа или выхода. Кодеки позволяют легко отделить транспортировку ваших сообщений от процесса сериализации.

| Кодек         | Описание                                                                                                         |
| ------------- | ---------------------------------------------------------------------------------------------------------------- |
| **json**      | Кодирует или декодирует данные **в формате JSON**                                                                |
| **multiline** | Объединяет **многострочные текстовые события**, такие как исключения Java и сообщения стектрейса, в одно событие |

Для получения дополнительной информации о доступных кодеках обратитесь к [документации по Codec Plugins](https://www.elastic.co/guide/en/logstash/current/codec-plugins.html).
