# 1. dependencyManagement
`dependencyManagement` — это мощный инструмент Maven **для централизованного УПРАВЛЕНИЯ ВЕРСИЯМИ зависимостей** ***в многомодульных проектах.***

### ВАЖНОЕ ЗАМЕЧАНИЯ
`dependencyManagement` **не добавляет зависимость в проект**. Он только **управляет версиями**. Зависимость по-прежнему нужно явно указать в дочернем модуле (просто без версии).

Если вы хотите добавить зависимость **во все модули сразу**, используйте секцию `<dependencies>` (без Management) в родительском POM.

---
## 1.1 Суть проблемы без `dependencyManagement`
Представьте, у вас есть монорепозиторий с тремя модулями:
```
my-project/
├── pom.xml (родительский)
|
├── module-a/
│   └── pom.xml
├── module-b/
│   └── pom.xml
└── module-c/
    └── pom.xml
```

**Каждому модулю нужен Spring Security (или другая зависимость).** 

Без `dependencyManagement` вы пишете в каждом модуле:
**module-a/pom.xml:**
```xml
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-web</artifactId>
    <version>6.4.3</version>
</dependency>
```

**module-b/pom.xml:**
```xml
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-web</artifactId>
    <version>6.3.1</version> <!-- ❗❗❗ Ошибка! Версия отличается ❗❗❗-->
</dependency>
```

**Проблемы:**
1. Версии могут разъехаться — конфликты, баги, ClassNotFoundException.
    
2. Чтобы обновить версию, нужно менять её в каждом модуле.
    
3. Легко забыть обновить какой-то модуль.

---
## 1.2 Решение с `dependencyManagement`
Родительский POM задаёт **версии для всех модулей сразу**:

**Родительский pom.xml:**
```xml
<dependencyManagement>

    <dependencies>
    
        <dependency>
            <groupId>org.springframework.security</groupId>
            <artifactId>spring-security-web</artifactId>
            <version>6.4.3</version>
        </dependency>
        
        <dependency>
            <groupId>org.springframework.security</groupId>
            <artifactId>spring-security-config</artifactId>
            <version>6.4.3</version>
        </dependency>
        
    </dependencies>
    
</dependencyManagement>
```

**Дочерние модули указывают зависимость БЕЗ версии:**
***module-a/pom.xml:***
```xml
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-web</artifactId>
    <!-- Версия берётся из dependencyManagement родителя -->
</dependency>
```

***module-b/pom.xml:***
```xml
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-web</artifactId>
    <!-- Та же версия, что и в module-a, гарантированно -->
</dependency>
```

---
## 1.3 Что даёт `dependencyManagement`

|Преимущество|Как работает|
|---|---|
|**Единая версия**|Все модули используют одну версию библиотеки|
|**Централизованное обновление**|Меняете версию в одном месте — обновляется везде|
|**Контроль транзитивных зависимостей**|Можно зафиксировать версию библиотеки, которую тянет другая зависимость|
|**Чистота дочерних POM**|Дочерние модули не содержат версий, только координаты|
