`<scope>` в Maven определяет **на каком этапе жизненного цикла приложения** будет доступна зависимость. 

Это важный механизм контроля над тем, что попадает в итоговый WAR/JAR-файл.

## 1.1 Зачем нужен?
Без `<scope>` **Maven добавляет зависимость везде**: 
- при компиляции, 
- при тестировании, 
- и в итоговый артефакт. 

==Но не все библиотеки нужны везде.== 

Например:
- `junit` нужен **только** для тестов. В продакшене он не должен занимать место.
    
- `jakarta.servlet-api` нужен **только** при компиляции. На сервере Tomcat уже предоставляет свою версию этого API.
    
- Драйвер базы данных нужен всегда — и при компиляции, и на сервере.

Scope позволяет указать: «Эту библиотеку использовать только там-то».

---
## 1.2 Виды scope

| Scope                        | Доступна при компиляции | Доступна при тестировании | Попадает в WAR/JAR | Когда использовать                                                              |
| ---------------------------- | ----------------------- | ------------------------- | ------------------ | ------------------------------------------------------------------------------- |
| **`compile`** (по умолчанию) | ✅                       | ✅                         | ✅                  | Большинство библиотек (Spring, Hibernate).                                      |
| **`provided`**               | ✅                       | ✅                         | ❌                  | API, которые предоставляет сервер (Servlet API).                                |
| **`runtime`**                | ❌                       | ✅                         | ✅                  | Драйверы БД, реализация SLF4J.                                                  |
| **`test`**                   | ❌                       | ✅                         | ❌                  | Библиотеки для тестов (JUnit, Mockito, H2).                                     |
| **`system`**                 | ✅                       | ✅                         | ❌                  | Редко. Зависимость лежит в файловой системе. Указать путь через `<systemPath>`. |
| **`import`**                 | —                       | —                         | —                  | Только в `dependencyManagement`. Импорт BOM (Bill of Materials).                |
