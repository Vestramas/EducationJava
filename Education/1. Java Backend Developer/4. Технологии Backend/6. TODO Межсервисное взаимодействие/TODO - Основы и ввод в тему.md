**API (Application Programming Interface)** — это интерфейс, с помощью которого приложения или сервисы обмениваются данными и взаимодействуют друг с другом. В веб-разработке API обычно реализуется через HTTP и предоставляет набор эндпоинтов, к которым клиент (фронтенд, мобильное приложение, другой сервис) может делать запросы. API определяет, _какие_ данные можно получать или отправлять, _как_ это делать и _в каком формате_. Он скрывает внутреннюю логику приложения и предоставляет безопасный, стандартизированный способ взаимодействия.

Основные типы API:

- **REST API** — ресурсно-ориентированный интерфейс на HTTP.
    
- **GraphQL** — гибкие запросы, клиент сам выбирает нужные поля.
    
- **gRPC** — высокопроизводительный бинарный протокол для микросервисов.
    
- **WebSocket API** — двусторонняя связь в реальном времени.
    

Что обычно есть в API:

- **Эндпоинты** (например, `/users`, `/orders/5`).
    
- **Методы**: GET, POST, PUT, PATCH, DELETE.
    
- **Формат данных**: JSON чаще всего.
    
- **Авторизация**: JWT, OAuth, API ключи.
    
- **Коды ответов**: 200, 400, 401, 404, 500 и др.
    

Почему API важен:

- Разделяет фронтенд и backend.
    
- Позволяет подключать мобильные приложения, микросервисы, интеграции.
    
- Делает систему расширяемой и легко поддерживаемой.
    
- Позволяет масштабировать архитектуру.
