Порождающие - **Отвечают за удобное и безопасное создание новых объектов** или даже целых семейств объектов.

Они решают проблему, когда простое `new MyClass(...)` перестает работать из-за сложной логики инициализации, зависимости от внешних условий или необходимости контролировать количество экземпляров.

| Паттерн              | Основная проблема                                 | Решение                                    |
| -------------------- | ------------------------------------------------- | ------------------------------------------ |
| **Singleton**        | Глобальное состояние, единый ресурс               | Один экземпляр + статический метод доступа |
| **Builder**          | Много параметров, сложная пошаговая инициализация | Отделить конструирование от представления  |
| **Factory Method**   | *Неизвестный заранее тип создаваемого объекта*    | *Делегировать создание подклассам*         |
| **Abstract Factory** | Нужно *переключать целые семейства объектов*      | Фабрика, создающая семейство продуктов     |
| **Prototype**        | Дорогое или сложное создание объекта              | Клонирование существующего экземпляра      |

### 1. Одиночка (Singleton)
**Задача:** Гарантировать, что у класса есть только **один экземпляр**, и предоставить к нему глобальную точку доступа. 

**Пример:** Логгер, пул соединений с БД, конфигурация приложения.

```java
public class DatabaseConnection {
    private static volatile DatabaseConnection instance;
    private String url = "jdbc:postgresql://localhost:5432/mydb";

    private DatabaseConnection() {} // Приватный конструктор

    public static DatabaseConnection getInstance() {
        if (instance == null) {
            synchronized (DatabaseConnection.class) {
                if (instance == null) {
                    instance = new DatabaseConnection();
                }
            }
        }
        return instance;
    }
    
    public void query(String sql) { /* ... */ }
}

// Использование
DatabaseConnection conn = DatabaseConnection.getInstance();
```

### 2. Строитель (Builder) - создать пошагово (через методы)
**Задача:** ==Создавать сложные объекты пошагово==, используя один и тот же процесс для разных представлений. Решает проблему "телескопического конструктора" (много параметров).  

**Пример:** Создание Email с обязательными и опциональными полями, сложный JSON/XML объект.

```java
public class Pizza {
    private final String size;
    private final boolean cheese;
    private final boolean pepperoni;
    
    private Pizza(Builder builder) {
        this.size = builder.size;
        this.cheese = builder.cheese;
        this.pepperoni = builder.pepperoni;
    }
    
    public static class Builder {
        private final String size; // обязательный
        private boolean cheese = false;
        private boolean pepperoni = false;
        
        public Builder(String size) { this.size = size; }
        public Builder cheese(boolean val) { cheese = val; return this; }
        public Builder pepperoni(boolean val) { pepperoni = val; return this; }
        public Pizza build() { return new Pizza(this); }
    }
}

// Использование
Pizza pizza = new Pizza.Builder("Large")
                    .cheese(true)
                    .pepperoni(true)
                    .build();
```

### 3. Фабричный метод (Factory Method) - определить интерфейс для создания объекта, НО ПОЗВОЛИТЬ подклассам решать какой именно объект 
**Задача:**
==Определить интерфейс для создания объекта==, **но позволить подклассам решать**, какой именно класс инстанцировать.  

**Пример:** Приложение логирует в файл, но в тестовой среде — в память.
```java
// Продукт
interface Transport { void deliver(); }

class Truck implements Transport { 
	public void deliver() { 
		System.out.println("Delivery by road"); 
	} 
}
class Ship implements Transport { 
	public void deliver() { 
		System.out.println("Delivery by sea"); 
	} 
}


// Создатель
abstract class Logistics {
    abstract Transport createTransport(); // Фабричный метод
    
    public void planDelivery() {
        Transport t = createTransport();
        t.deliver();
    }
}

class RoadLogistics extends Logistics {
    Transport createTransport() { return new Truck(); }
}
class SeaLogistics extends Logistics {
    Transport createTransport() { return new Ship(); }
}

// Использование
Logistics logistics = new RoadLogistics();
logistics.planDelivery(); // "Delivery by road"
```

### 4. Абстрактная фабрика (Abstract Factory) - определять семейство классов, по выбранной фабрике

**Задача:** 
Создавать **семейства связанных объектов** (например, виджетов для Windows и для macOS) без привязки к конкретным классам. 

**Решение:** Позволяет переключать целые семейства продуктов одной строкой кода.

```java
// Абстрактные продукты
interface Button { void paint(); }
interface Checkbox { void check(); }

// Конкретные продукты для Windows
class WindowsButton implements Button { 
	public void paint() { 
		System.out.println("Windows button"); 
	} 
}
class WindowsCheckbox implements Checkbox { 
	public void check() { 
		System.out.println("Windows checkbox"); 
	} 
}

// Конкретные продукты для Mac
class MacButton implements Button { 
	public void paint() { 
		System.out.println("Mac button"); 
	} 
}
class MacCheckbox implements Checkbox { 
	public void check() { 
		System.out.println("Mac checkbox"); 
	} 
}


// АБСТРАКТНАЯ ФАБРИКА
interface GUIFactory {
    Button createButton();
    Checkbox createCheckbox();
}

// КОНКРЕТНЫЕ ФАБРИКИ
class WindowsFactory implements GUIFactory {
    public Button createButton() { return new WindowsButton(); }
    public Checkbox createCheckbox() { return new WindowsCheckbox(); }
}

class MacFactory implements GUIFactory {
    public Button createButton() { return new MacButton(); }
    public Checkbox createCheckbox() { return new MacCheckbox(); }
}



// Клиент
class Application {
    private Button button;
    private Checkbox checkbox;
    
    // ПРИ СОЗДАНИИ КАКОГО-ТО КЛАССА, 
    // МЫ МОЖЕТ ПОДСТАВИТЬ ЛЮБУЮ РЕАЛИЗАЦИЮ ФАБРИКИ
    public Application(GUIFactory factory) {
        button = factory.createButton();
        checkbox = factory.createCheckbox();
    }
}

// Использование (переключение ОС)
Application app = new Application(new MacFactory());
```

### 5. Прототип (Prototype)
**Задача:** **Копировать существующие объекты**, не создавая зависимость от их классов. Решает проблему дорогого создания объекта (например, из БД) или необходимости скрыть конструктор.  
**Пример:** Клонирование сложного документа, сцены в игре.
```java
interface Copyable { 
	Copyable copy(); 
}

class User implements Copyable {
    private String name;
    private int age;
    
    public User(String name, int age) { 
	    this.name = name; 
	    this.age = age; 
	}
    
    public User copy() { 
	    return new User(this.name, this.age); 
	} // Глубокое копирование
    
    @Override
    public String toString() { return "User{" + name + ", " + age + "}"; }
}

// Кэш прототипов
class UserCache {
    private static Map<String, User> cache = new HashMap<>();
    
    static {
        User user = new User("Default", 18);
        cache.put("default", user);
    }
    
    public static User getClone(String id) {
        return cache.get(id).copy();
    }
}

// Использование
User user1 = UserCache.getClone("default");
User user2 = user1.copy(); // еще один
```
