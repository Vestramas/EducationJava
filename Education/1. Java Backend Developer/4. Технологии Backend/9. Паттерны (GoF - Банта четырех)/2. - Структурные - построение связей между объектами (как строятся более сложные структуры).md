**Основная задача:** 
Обеспечить ==гибкое построение связей между классами и объектами, позволяя им работать вместе, не создавая жесткой связанности==. 

**Решают проблемы неудобного API**, несовместимых интерфейсов и необходимости расширения функциональности без изменения кода.

| Паттерн       | Основная проблема                                | Решение                                        |
| ------------- | ------------------------------------------------ | ---------------------------------------------- |
| **Adapter**   | Несовместимые интерфейсы                         | Обертка с нужным интерфейсом                   |
| **Bridge**    | "Картезианское произведение" классов             | Разделение абстракции и реализации             |
| **Decorator** | Добавление функций без наследования              | Вложенные обертки с расширенным поведением     |
| **Facade**    | Сложность подсистемы                             | Простой унифицированный интерфейс              |
| **Proxy**     | Контроль доступа, ленивая загрузка               | Заместитель с дополнительной логикой           |

### 1. Адаптер (Adapter) - совместить НЕ СОВМЕСТИМЫЕ ИНТЕРФЕЙСЫ

АДАПТЕР НАСЛЕДУЕТ НОВЫЙ ИНТЕРФЕЙС, И ВНУТРИ СЕБЯ ИСПОЛЬЗУЕТ СТАРЫЙ. 
Использует старый интерфейс в новом (наследуемом)

**Задача:** ==Сделать несовместимые интерфейсы совместимыми== — как переходник для евророзетки в США. Позволяет классам работать вместе, если их интерфейсы не совпадают.

**Пример:** Старая система логирования пишет в файл, новая требует REST API.

```java
// Существующий класс с неудобным интерфейсом
class OldLogger {
    void writeToFile(String message, String filename) {
        System.out.println("Writing '" + message + "' to " + filename);
    }
}

// Целевой интерфейс, который нужен клиенту
interface NewLogger {
    void log(String message);
}


// Адаптер
class LoggerAdapter implements NewLogger {
    private OldLogger oldLogger;
    private String defaultFilename = "app.log";
    
    public LoggerAdapter(OldLogger oldLogger) {
        this.oldLogger = oldLogger;
    }
    
    @Override
    public void log(String message) {
        oldLogger.writeToFile(message, defaultFilename);
    }
}

// Использование
OldLogger legacy = new OldLogger();
NewLogger logger = new LoggerAdapter(legacy);

logger.log("User logged in"); // "Writing 'User logged in' to app.log"
```

### 2. Декоратор (Decorator) - ДИНАМИЧЕСКИ ДОБАВЛЯТЬ "ОБЯЗАНОСТИ" ОБЪЕКТУ

**Задача:** **Динамически добавлять новые обязанности объекту**, не изменяя его код и не используя наследование. Решает проблему "разрастания подклассов" для комбинаций функций.

**Пример:** Кофейня — базовый кофе можно дополнить молоком, сиропом, взбитыми сливками.

```java
// Компонент
interface Coffee {
    double cost();
    String description();
}

class SimpleCoffee implements Coffee {
    public double cost() { return 2.0; }
    public String description() { return "Simple coffee"; }
}

// Абстрактный декоратор НАСЛЕДНИК ТОГО, КОМУ НАКИДЫВАЕМ ОБЯЗАНОСТИ
abstract class CoffeeDecorator implements Coffee {
    protected Coffee decoratedCoffee;
    
    public CoffeeDecorator(Coffee coffee) { 
	    this.decoratedCoffee = coffee; 
	}
}

// Конкретные декораторы
class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) { 
	    super(coffee); 
	}
    public double cost() { 
	    return decoratedCoffee.cost() + 0.5; 
	}
    public String description() { 
	    return decoratedCoffee.description() + ", milk"; 
	}
}

class SyrupDecorator extends CoffeeDecorator {
    public SyrupDecorator(Coffee coffee) { 
	    super(coffee); 
	}
    public double cost() { 
	    return decoratedCoffee.cost() + 0.7; 
	}
    public String description() { 
	    return decoratedCoffee.description() + ", syrup"; 
	}
}

// Использование — комбинируем декораторы
Coffee coffee = new SimpleCoffee();
coffee = new MilkDecorator(coffee);
coffee = new SyrupDecorator(coffee);
System.out.println(coffee.description() + " = $" + coffee.cost());
// "Simple coffee, milk, syrup = $3.2"
```

### 3. Фасад (Facade) - скрывает детали реализации, предоставляя единый интерфейс для работы

**Задача:** **Предоставить простой интерфейс к сложной системе** классов или библиотек. 

Решает проблему сложности — *клиенту не нужно знать о внутренних деталях и зависимостях.*

**Пример:** Компьютер — вместо того чтобы включать питание, загружать BIOS, запускать ОС по отдельности, есть кнопка "Power".

```JAVA
// Сложная подсистема
class CPU {
    void start() { System.out.println("CPU started"); }
    void execute() { System.out.println("Executing"); }
}

class Memory {
    void load() { System.out.println("Loading OS into memory"); }
}

class HardDrive {
    void read() { System.out.println("Reading boot sector"); }
}

// Фасад
class ComputerFacade {
    private CPU cpu;
    private Memory memory;
    private HardDrive hardDrive;
    
    public ComputerFacade() {
        this.cpu = new CPU();
        this.memory = new Memory();
        this.hardDrive = new HardDrive();
    }
    
    public void startComputer() {
        System.out.println("=== Starting computer ===");
        hardDrive.read();
        memory.load();
        cpu.start();
        cpu.execute();
        System.out.println("=== Computer ready ===");
    }
}

// Использование — клиенту нужен только один метод
ComputerFacade computer = new ComputerFacade();
computer.startComputer();
```

### 4. Мост (Bridge)

**Задача:** **Разделить абстракцию и реализацию**, чтобы их можно было изменять независимо. 

Решает проблему "картезианского произведения" классов (когда у вас N вариаций одного измерения и M другого — нужно не N×M классов, а N+M).

**Пример:** Разные устройства (TV, Radio) и разные пульты управления (Basic, Advanced).

```java
// Реализация (device)
interface Device {
    void turnOn();
    void turnOff();
    void setVolume(int percent);
}

class TV implements Device {
    public void turnOn() { System.out.println("TV ON"); }
    public void turnOff() { System.out.println("TV OFF"); }
    public void setVolume(int percent) { System.out.println("TV volume: " + percent); }
}

class Radio implements Device {
    public void turnOn() { System.out.println("Radio ON"); }
    public void turnOff() { System.out.println("Radio OFF"); }
    public void setVolume(int percent) { System.out.println("Radio volume: " + percent); }
}



// Абстракция (remote)
abstract class RemoteControl {
    protected Device device;
    
    public RemoteControl(Device device) { this.device = device; }
    abstract void togglePower();
    abstract void volumeUp();
}

class BasicRemote extends RemoteControl {
    public BasicRemote(Device device) { super(device); }
    public void togglePower() { 
        System.out.print("Basic remote: "); 
        device.turnOn(); 
    }
    public void volumeUp() { device.setVolume(10); }
}

// Использование — можно комбинировать любые Remote с любыми Device
Device tv = new TV();
RemoteControl remote = new BasicRemote(tv);
remote.togglePower(); // "Basic remote: TV ON"
```

### 5. Заместитель (Proxy)

**Задача:** **Контролировать доступ к другому объекту**, ==добавляя косвенный уровень. ==

==Решает проблемы ленивой загрузки==, контроля доступа, логирования или кэширования.

**Пример:** Загрузка тяжелых 3D-моделей — сначала показываем заглушку.

```java
// Общий интерфейс
interface Image {
    void display();
}

// Реальный объект (тяжелый)
class RealImage implements Image {
    private String filename;
    
    public RealImage(String filename) {
        this.filename = filename;
        loadFromDisk();
    }
    
    private void loadFromDisk() {
        System.out.println("Loading " + filename + " from disk...");
        // тяжелая операция
    }
    
    public void display() {
        System.out.println("Displaying " + filename);
    }
}

// Заместитель
class ProxyImage implements Image {
    private String filename;
    private RealImage realImage;
    
    public ProxyImage(String filename) { this.filename = filename; }
    
    public void display() {
        if (realImage == null) { // ленивая загрузка
            realImage = new RealImage(filename);
        }
        realImage.display();
    }
}

// Использование
Image image = new ProxyImage("photo.jpg");
// файл еще не загружен
image.display(); // теперь загрузится и отобразится
image.display(); // повторно — без загрузки
```
