**Основная задача:** 
Обеспечить **эффективное взаимодействие между объектами**, распределяя обязанности и инкапсулируя поведение.

Решают проблемы сложных связей, алгоритмов, которые нужно менять, и необходимости слабой связанности между объектами.

---
### 1. Стратегия (Strategy)

**Задача:** **Определить семейство алгоритмов**, инкапсулировать каждый и сделать их взаимозаменяемыми.

Решает проблему, когда нужно использовать разные варианты алгоритма в одном объекте без условных операторов.

**Пример:** Разные способы оплаты (карта, PayPal, криптовалюта).
```java
// Интерфейс стратегии
interface PaymentStrategy {
    void pay(double amount);
}

// Конкретные стратегии
class CreditCardPayment implements PaymentStrategy {
    private String cardNumber;
    
    public CreditCardPayment(String cardNumber) { 
	    this.cardNumber = cardNumber; 
	}
    
    @Override
    public void pay(double amount) {
        System.out.printf(
	        "Paid $%.2f using credit card %s\n", 
	        amount, 
	        cardNumber);
    }
}

class PayPalPayment implements PaymentStrategy {
    private String email;
    
    public PayPalPayment(String email) { 
	    this.email = email; 
	}
    
    @Override
    public void pay(double amount) {
        System.out.printf(
	        "Paid $%.2f using PayPal account %s\n", 
	        amount, 
	        email);
    }
}


// Контекст
class ShoppingCart {
    private PaymentStrategy paymentStrategy;
    
    public void setPaymentStrategy(PaymentStrategy strategy) {
        this.paymentStrategy = strategy;
    }
    
    public void checkout(double total) {
        paymentStrategy.pay(total);
    }
}

// Использование
ShoppingCart cart = new ShoppingCart();

cart.setPaymentStrategy(new CreditCardPayment("1234-5678"));
cart.checkout(99.99);

cart.setPaymentStrategy(new PayPalPayment("user@example.com"));
cart.checkout(49.99);
```

### 2. Наблюдатель (Observer)

**Задача:** ==Создать механизм подписки==, позволяющий одним объектам (наблюдателям) следить за изменениями других объектов (субъектов). 

Решает проблему оповещения многих объектов об изменении состояния без жесткой привязки.

**Пример:** Система уведомлений: подписчики получают новости из издательства.

```java
// Наблюдатель
interface Observer {
    void update(String news);
}

// Субъект (издатель)
interface Subject {
    void attach(Observer observer);
    void detach(Observer observer);
    void notifyObservers();
}

class NewsAgency implements Subject {
    private List<Observer> observers = new ArrayList<>();
    private String news;
    
    public void setNews(String news) {
        this.news = news;
        notifyObservers();
    }
    
    @Override
    public void attach(Observer observer) { observers.add(observer); }
    
    @Override
    public void detach(Observer observer) { observers.remove(observer); }
    
    @Override
    public void notifyObservers() {
        for (Observer observer : observers) {
            observer.update(news);
        }
    }
}

// Конкретные наблюдатели
class EmailSubscriber implements Observer {
    private String email;
    
    public EmailSubscriber(String email) { this.email = email; }
    
    @Override
    public void update(String news) {
        System.out.println("Sending email to " + email + ": " + news);
    }
}

class SMSSubscriber implements Observer {
    private String phone;
    
    public SMSSubscriber(String phone) { this.phone = phone; }
    
    @Override
    public void update(String news) {
        System.out.println("Sending SMS to " + phone + ": " + news);
    }
}

// Использование
NewsAgency agency = new NewsAgency();
agency.attach(new EmailSubscriber("alice@example.com"));
agency.attach(new SMSSubscriber("+1234567890"));
agency.setNews("Java 21 released!"); // Оба получат уведомление
```

### 3. Цепочка обязанностей (Chain of Responsibility)

**Задача:** ==Передавать запрос по цепочке обработчиков==, пока один из них не обработает запрос. 

Решает проблему, когда отправитель не знает, какой именно обработчик нужен.

**Пример:** Система поддержки: запрос проходит через уровни поддержки L1 → L2 → L3.
```java
// Базовый обработчик
abstract class SupportHandler {
    protected SupportHandler next;
    
    public void setNext(SupportHandler next) { this.next = next; }
    
    public abstract void handle(String issue, int severity);
}

class Level1Support extends SupportHandler {
    @Override
    public void handle(String issue, int severity) {
        if (severity <= 2) {
            System.out.println("L1 Support resolved: " + issue);
        } else if (next != null) {
            next.handle(issue, severity);
        }
    }
}

class Level2Support extends SupportHandler {
    @Override
    public void handle(String issue, int severity) {
        if (severity <= 4) {
            System.out.println("L2 Support resolved: " + issue);
        } else if (next != null) {
            next.handle(issue, severity);
        }
    }
}

class Level3Support extends SupportHandler {
    @Override
    public void handle(String issue, int severity) {
        System.out.println("L3 Support resolved: " + issue);
    }
}

// Использование
Level1Support l1 = new Level1Support();
Level2Support l2 = new Level2Support();
Level3Support l3 = new Level3Support();
l1.setNext(l2);
l2.setNext(l3);

l1.handle("Password reset", 1);  // L1
l1.handle("Database down", 5);   // L3
```

### 4. Состояние (State)

**Задача:** 
==Позволить объекту ИЗМЕНЯТЬ свое ПОВЕДЕНИЕ, ПРИ ИЗМЕНЕНИИ внутреннего СОСТОЯНИЯ==, как будто он меняет класс. 

Решает проблему огромных условных конструкций (if/switch).

**Пример:** Заказ в интернет-магазине (новый, оплачен, доставляется, завершен).
```JAVA
// Контекст
class Order {
    private OrderState state;
    
    public Order() { state = new NewOrderState(); }
    
    void setState(OrderState state) { this.state = state; }
    
    void pay() { state.pay(this); }
    void ship() { state.ship(this); }
    void complete() { state.complete(this); }
}

// ИНТЕРФЕЙС СОСТОЯНИЯ
interface OrderState {
    void pay(Order order);
    void ship(Order order);
    void complete(Order order);
}

class NewOrderState implements OrderState {
    @Override
    public void pay(Order order) {
        System.out.println("Order paid, moving to paid state");
        order.setState(new PaidOrderState());
    }
    @Override
    public void ship(Order order) { 
	    System.out.println("Cannot ship unpaid order"); 
	}
    @Override
    public void complete(Order order) { 
	    System.out.println("Cannot complete unpaid order"); 
	}
}

class PaidOrderState implements OrderState {
    @Override
    public void pay(Order order) { 
	    System.out.println("Already paid"); 
	}
    @Override
    public void ship(Order order) {
        System.out.println("Order shipped, moving to shipped state");
        order.setState(new ShippedOrderState());
    }
    @Override
    public void complete(Order order) { 
	    System.out.println("Ship before completing"); 
	}
}

class ShippedOrderState implements OrderState {
    @Override
    public void pay(Order order) { 
	    System.out.println("Already paid"); 
	}
    @Override
    public void ship(Order order) { 
	    System.out.println("Already shipped");
	}
    @Override
    public void complete(Order order) {
        System.out.println("Order completed");
        order.setState(new CompletedOrderState());
    }
}

class CompletedOrderState implements OrderState {
    @Override
    public void pay(Order order) { 
	    System.out.println("Order already completed"); 
	}
    @Override
    public void ship(Order order) { 
	    System.out.println("Order already completed"); 
	}
    @Override
    public void complete(Order order) { 
	    System.out.println("Already completed"); 
	}
}

// Использование
Order order = new Order();
order.pay();    // "Order paid, moving to paid state"
order.ship();   // "Order shipped, moving to shipped state"
order.complete(); // "Order completed"
order.complete(); // "Already completed"
```

### 5. Шаблонный метод (Template Method)

**Задача:** 
==Определить скелет алгоритма в базовом классе==, **позволяя подклассам переопределять отдельные шаги**. 

Решает проблему дублирования кода при одинаковой структуре алгоритма.

**Пример:** Приготовление напитков (чай и кофе) — одинаковые шаги с разными деталями.

```java
// Абстрактный класс с шаблонным методом
abstract class BeverageMaker {
    // Шаблонный метод (final, чтобы нельзя было изменить структуру)
    public final void makeBeverage() {
        boilWater();
        brew();
        pourInCup();
        addCondiments();
    }
    
    protected void boilWater() { System.out.println("Boiling water"); }
    protected void pourInCup() { System.out.println("Pouring into cup"); }
    
    // Шаги, которые должны реализовать подклассы
    protected abstract void brew();
    protected abstract void addCondiments();
}


class TeaMaker extends BeverageMaker {
    @Override
    protected void brew() { 
	    System.out.println("Steeping tea bag"); 
	}
    @Override
    protected void addCondiments() { 
	    System.out.println("Adding lemon"); 
	}
}

class CoffeeMaker extends BeverageMaker {
    @Override
    protected void brew() { 
	    System.out.println("Dripping coffee through filter"); 
	}
    @Override
    protected void addCondiments() { 
	    System.out.println("Adding sugar and milk"); 
	}
}

// Использование
BeverageMaker tea = new TeaMaker();
tea.makeBeverage();
// Boiling water -> Steeping tea bag -> Pouring into cup -> Adding lemon
```

### 6. Посетитель (Visitor)

**Задача:** 

**Добавлять новые операции над объектами**, не изменяя сами объекты. 

Решает проблему, когда нужно выполнить разные действия для разных типов, но не хочется засорять классы этими методами.

**Пример:** Разные операции для геометрических фигур (расчет площади, отрисовка, экспорт в JSON).

```java
// Элементы (фигуры)
interface Shape {
    void accept(Visitor visitor);
}

class Circle implements Shape {
    private double radius;
    
    public Circle(double radius) { this.radius = radius; }
    public double getRadius() { return radius; }
    
    @Override
    public void accept(Visitor visitor) { visitor.visit(this); }
}

class Rectangle implements Shape {
    private double width, height;
    
    public Rectangle(double width, double height) { this.width = width; this.height = height; }
    public double getWidth() { return width; }
    public double getHeight() { return height; }
    
    @Override
    public void accept(Visitor visitor) { visitor.visit(this); }
}


// Посетитель
interface Visitor {
    void visit(Circle circle);
    void visit(Rectangle rectangle);
}

class AreaCalculator implements Visitor {
    private double totalArea = 0;
    
    @Override
    public void visit(Circle circle) {
        double area = Math.PI * circle.getRadius() * circle.getRadius();
        totalArea += area;
        System.out.printf("Circle area: %.2f\n", area);
    }
    
    @Override
    public void visit(Rectangle rectangle) {
        double area = rectangle.getWidth() * rectangle.getHeight();
        totalArea += area;
        System.out.printf("Rectangle area: %.2f\n", area);
    }
    
    public double getTotalArea() { return totalArea; }
}

class JsonExporter implements Visitor {
    private StringBuilder json = new StringBuilder("[");
    
    @Override
    public void visit(Circle circle) {
        if (json.length() > 1) {
	        json.append(",");
	        json.append(
		        String.format("{\"type\":\"circle\",\"radius\":%.2f}", 
		        circle.getRadius()
		    )
        }
        
    }
    
    @Override
    public void visit(Rectangle rectangle) {
        if (json.length() > 1) {
	        json.append(",");
	        json.append(
		        String.format(
		    "{\"type\":\"rectangle\",\"width\":%.2f,\"height\":%.2f}", 
                rectangle.getWidth(), 
	            rectangle.getHeight()));
    }
    
    public String getJson() { return json.append("]").toString(); }
}

// Использование
List<Shape> shapes = Arrays.asList(new Circle(5), new Rectangle(4, 6));
AreaCalculator areaCalc = new AreaCalculator();
JsonExporter jsonExporter = new JsonExporter();

for (Shape shape : shapes) {
    shape.accept(areaCalc);
    shape.accept(jsonExporter);
}

System.out.println("Total area: " + areaCalc.getTotalArea());
System.out.println("JSON: " + jsonExporter.getJson());
```
