# 1. Быстрый поиск по синтаксис-помощнику
**Второй способ работы с синтакс-помощником — контекстный поиск (Ctrl+F1)** (первый тупо самим читать про каждый элемент)

Этот способ удобен, когда нужно быстро разобраться в конкретном выражении прямо из кода.

**Как это работает:**
1. В модуле формы (или другом модуле) установить курсор на интересующее выражение (например, `Элементы`).
    
2. Нажать **Ctrl + F1**.
    
3. Откроется окно со списком разделов синтакс-помощника, где встречается это выражение. Полный путь к разделу показан через слеш (`/`).
    
4. Если список велик, можно уточнить поиск:
    - Нажать **Ctrl + F** в окне списка.
        
    - Ввести ключевое слово (например, `УправляемаяФорма`), чтобы найти нужный раздел.
        
    - Нажать _Искать_ → нужный раздел выделится.
        
5. Кнопка _Показать_ откроет описание выбранного раздела в нижней части окна.
    
6. Чтобы увидеть это описание в дереве содержания, нажать кнопку **Найти текущий элемент в дереве** (над окном описания).

## **Фильтрация по режиму выполнения:**
Можно ограничить показываемые объекты теми, что доступны в текущем режиме (тонкий клиент, веб-клиент, сервер):

- Меню _Сервис → Параметры_ → закладка _Справка_.
    
- Отметить нужные режимы исполнения.
    

Это особенно полезно, когда код выполняется на сервере — важно видеть только серверные объекты.

# 2. **Анализ кода с помощью отладчика**
Отладчик позволяет понять работу кода без необходимости заранее знать контекст — достаточно остановиться в нужном месте и посмотреть, какие объекты и значения доступны.

**Как использовать на примере обработчика `МатериалыКоличествоПриИзменении`:**

1. **Установить точку останова**  
    В модуле формы документа открыть процедуру и дважды щелкнуть в служебной области слева от первой строки.
    
2. **Запустить отладку**  
    _Отладка → Начать отладку_ (или кнопка на панели). Конфигуратор запустит 1С:Предприятие в режиме отладки.
    
3. **Воспроизвести ситуацию**  
    Открыть документ, изменить поле _Количество_. Выполнение остановится на точке останова.
    
4. **Изучить выражения**
    - Выделить слово `Элементы` → нажать _Вычислить выражение_ (Shift+F9).
        
    - В окне видно: `Элементы` — это коллекция `ВсеЭлементыФормы`.
        
    - Раскрыть объект, найти `Материалы` (тип `ТаблицаФормы`).
        
    - Раскрыть `Материалы`, найти `ТекущиеДанные` — это данные текущей строки табличной части.
        
5. **Шагнуть внутрь процедуры**  
    Нажать _Шагнуть в_ — переход в процедуру `РассчитатьСумму` общего модуля.  
    Выделить `СтрокаТабличнойЧасти` → _Вычислить выражение_. Видно объект `ДанныеФормыЭлементКоллекции`, но сумма ещё не пересчитана.
    
6. **Выполнить строку**  
    _Шагнуть через_ — код выполняется. Подвести курсор к `Количество` или `Сумма` — всплывающая подсказка покажет новые значения. Сумма пересчитана.
    
7. **Продолжить или завершить**  
    _Продолжить отладку_ (F5) или _Завершить_.


## **Полезные приёмы:**
- `ЭтотОбъект` в модуле формы — показывает полный контекст текущей формы со всеми расширениями.
    
- Аналогично `ЭтотОбъект` работает в модулях объектов и наборов записей.

**Важно:** изменения кода во время отладки не компилируются автоматически — для проверки правок нужно обновить конфигурацию базы данных и перезапустить отладку.

# 3. Движения по регистрам
## 3.1 Очистка движений
Очищать движения регстров нужно следующим образом
```java
Движения.<Регистр>.Очистить();
Движения.<Регистр>.Записать();
Движения.<Регистр>.Записывать = Истина;
```

`Очистить()` - ОЧИЩАЕТ движения, которые были вместе с объектом(формой)

> Если галочка будет стоять, то если выполнить просто метод `Записать()` - запишутся эти подтянутые изменения
