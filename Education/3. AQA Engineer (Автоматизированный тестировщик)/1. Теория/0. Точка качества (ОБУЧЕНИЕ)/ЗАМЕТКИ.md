1. При Assert обязательно используй AssertMessage. (Стиль Точка качества)
	
2. Используй Page Object паттерн для работы с страницами и написанием тестов
	1. **Элементы (локаторы) не** должны быть **доступны за пределами класса** страницы
	2. **Методы для работы с элементами должны быть доступны** вне класса страницы
	3. *Методы не должны возвращать элементы/страницы*
	4. Каждый класс страницы должен быть независимым
	5. *Имена* элементов и методов должны соответствовать *их назначению*
	6. *«Ассерты» не должны размещаться в классах страниц*. Все проверки только в классе теста

3. Для идентификации страницы, используй уникальный идентификатор. **НЕ ИСПОЛЬЗУЙ title() или ссылку**

4. Конфиги и тестовые данные. РАЗДЕЛЯЙ, ДЛЯ КОНФИГУРАЦИОННЫХ ДАННЫХ - один файл, для тестовых -  другой
   
   НЕ ХАРДКОДЬ ДАННЫЕ В КОДЕ
   
   Используются JSON или XML. Обычно JSON. 
   И спец классы:
	   - Утилитный для любых JSON файла (методы для работы с JSON) (дженерик, чтобы потом мапить в класс конфига и класс тестовых данных)
	   - Специальные классы для конфига и тестовые, которые используют методы утилитарного
