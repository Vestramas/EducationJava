# 1. Основы Selenium: Настройка и команды браузера

## 1.1 Методы открытия браузера, страницы

**Открытие браузера с помощью Selenium WebDriver — это фундаментальный шаг** в автоматизации тестирования веб-приложений. 

> [!NOTE] WebDriver - интерфейс для работы с браузером
> `WebDriver` предоставляет **интерфейс для взаимодействия с браузерами, позволяя программно управлять сессией браузера**: открывать окна, переходить по страницам и имитировать действия пользователя.

Ниже приведен пример кода на Java, демонстрирующий, как открыть браузеры (Google Chrome, Firefox и Microsoft Edge) с помощью Selenium WebDriver.

**Предварительные требования:**
- JDK
- Библиотека Selenium WebDriver
- Исполняемые файлы драйверов для браузеров (ChromeDriver, GeckoDriver, EdgeDriver)

---
### Открытие браузера через WebDriver (Java-код)
В коде ниже, НЕОБХОДИМЫЙ ДРАЙВЕР БРАУЗЕРА ПОДКЛЮЧАЕТСЯ ВРУЧНУЮ, БЕЗ Maven, поэтому в коде устанавливают системное свойство с указанием `.jar`-файла

```java
import org.openqa.selenium.WebDriver;
// Раскомментируйте импорт для нужного браузера
import org.openqa.selenium.chrome.ChromeDriver;
// import org.openqa.selenium.firefox.FirefoxDriver;
// import org.openqa.selenium.edge.EdgeDriver;

public class OpenBrowserExample {
    public static void main(String[] args) {

        // Для Google Chrome:
        System.setProperty(
	        "webdriver.chrome.driver", "путь/к/chromedriver");
	        
        WebDriver driver = new ChromeDriver();
        
        driver.get("https://www.google.com");
        System.out.println("Google Chrome запущен успешно.");
        
        driver.quit();

        // Для Firefox (раскомментировать):
        // System.setProperty("webdriver.gecko.driver", "путь/к/geckodriver");
        // WebDriver driver = new FirefoxDriver();
        // driver.get("https://www.google.com");
        // System.out.println("Firefox запущен успешно.");
        // driver.quit();

        // Для Microsoft Edge (раскомментировать):
        // System.setProperty("webdriver.edge.driver", "путь/к/edgedriver");
        // WebDriver driver = new EdgeDriver();
        // driver.get("https://www.google.com");
        // System.out.println("Microsoft Edge запущен успешно.");
        // driver.quit();
    }
}
```

**Что происходит в коде:**
1. Мы указываем системе путь к исполняемому файлу драйвера через `System.setProperty()`.
    
2. Создаем объект драйвера (например, `new ChromeDriver()`), который открывает браузер.
    
3. С помощью метода `get()` переходим на нужный URL.
    
4. Закрываем браузер методом `quit()`.
    

> **Важно:** Замените `"путь/к/chromedriver"` на актуальный путь к файлу на вашем диске.

---
### Открытие веб-страниц: онлайн и офлайн

**Онлайн-страницы**  
Используйте `get()` с полным URL (HTTP или HTTPS).
```java
driver.get("https://www.apress.com");
```

**Офлайн-страницы (локальные HTML-файлы)**  
Используйте протокол `file:///`.
```java
driver.get("file:///путь/к/вашему/локальному/файлу.html");
```

Это удобно для тестирования статичных страниц без доступа к интернету.

## 1.2 Управление браузером
### Управление сессией: `close()` и `quit()`
Это критически важные команды для корректного завершения тестов и освобождения ресурсов.

- **`driver.close()`** — *закрывает* **текущую** вкладку или окно браузера (то, которое в фокусе). Если открыто несколько вкладок, остальные останутся открытыми.
    
- **`driver.quit()`** — *закрывает* **все** окна, связанные с данной сессией WebDriver, и полностью завершает сессию. 
  **Рекомендуется использовать именно `quit()` в конце каждого теста**, чтобы не оставлять "висячих" процессов браузера в памяти.

---
### Управление размером и положением окна

Это нужно для тестирования адаптивности (responsive design) и поведения приложения в разных условиях.

**Максимизация:**
```java
driver.manage().window().maximize();
```

**Минимизация (сворачивание):**
```java
driver.manage().window().minimize();
```

**Установка конкретного размера:**
```java
import org.openqa.selenium.Dimension;

driver.manage().window().setSize(new Dimension(1024, 768));
```

**Полноэкранный режим (Fullscreen):** (скрывает панель адреса и вкладки)
```java
driver.manage().window().fullscreen();
```

**Установка положения окна на экране:**
```java
import org.openqa.selenium.Point;

driver.manage().window().setPosition(new Point(0, 0)); // В левый верхний угол
```

---
**Пример**: установка размера и положения одновременно
```java
import org.openqa.selenium.Dimension;
import org.openqa.selenium.Point;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.firefox.FirefoxDriver;

public class SetBrowserSizeAndPosition {
    public static void main(String[] args) {
        WebDriver driver = new FirefoxDriver();
        driver.get("https://apress.com");

        // Устанавливаем позицию окна (x=30, y=30)
        driver.manage().window().setPosition(new Point(30, 30));
        // Устанавливаем размер окна (450x500 пикселей)
        driver.manage().window().setSize(new Dimension(450, 500));

        System.out.println(
	        "Размер и положение окна установлены");
        driver.quit();
    }
}
```

Это дает вам точный контроль над окружением, чтобы тестировать отображение сайта под разные разрешения экранов.

---
### Получение позиции браузера

Знание текущего положения окна браузера может быть полезно, особенно когда вы *запускаете тесты, связанные с экранными координатами*, или когда нужно проверить, что окно находится в ожидаемом месте при работе с несколькими окнами или мониторами.

Получение позиции позволяет узнать текущее расположение окна на экране.
```java
Point position = driver.manage().window().getPosition();

System.out.println(
	"Позиция браузера - X: " + position.getX() + 
	", Y: " + position.getY());
```

**Получив объект `Point` с координатами, вы можете делать проверки (assertions) в тестах**. Это особенно полезно, когда расположение окна может влиять на поведение или видимость UI-элементов.

---
### Получение размера окна

Знание текущего размера окна критически **важно для тестирования адаптивного дизайна** (responsive design), ***позволяя проверить, что ваше приложение корректно отображается при разных размерах.***

Получение размера окна помогает понять, в каких текущих размерах области просмотра (viewport) отображается ваше приложение.
```java
Dimension size = driver.manage().window().getSize();

System.out.println(
	"Размер окна - Ширина: " + size.getWidth() + 
	", Высота: " + size.getHeight());
```

Получив размеры, вы можете проверить, корректно ли отображается приложение для данного размера окна. Это помогает убедиться, что адаптивный дизайн работает правильно и обеспечивает единообразный пользовательский опыт на всех устройствах.

## 1.3 Навигация по веб-страницам с помощью Selenium WebDriver
Навигация по страницам — фундаментальная часть процесса автоматизированного тестирования. 

*Selenium WebDriver предлагает интуитивные команды для перемещения вперед и назад по истории браузера, обновления текущей страницы и даже перехода на новый URL в том же окне.*

### Навигация назад (Back)
Чтобы **имитировать нажатие кнопки "Назад" в браузере, используйте метод** `navigate().back()`. 

Это позволяет протестировать поведение приложения, когда пользователь возвращается на предыдущую страницу.
```java
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;

public class BackClickExample {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.get("https://example.com");

        // Выполняем клик по ссылке
        WebElement link = driver.findElement(By.id("link-id"));
        link.click();

        // Возвращаемся назад
        driver.navigate().back();

        driver.quit();
    }
}
```

После перехода на вторую страницу вызов `navigate().back()` возвращает вас на первую. Эта команда необходима при тестировании многостраничных сценариев, чтобы убедиться, что каждая страница сохраняет свое состояние или выполняет правильные действия при возврате пользователя.

---
### Навигация вперед (Forward)
Иногда нужно снова перейти вперед по истории браузера. Команда `navigate().forward()` имитирует нажатие кнопки "Вперед".

```java
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;

public class ForwardClickExample {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.get("https://example.com");

        WebElement link = driver.findElement(By.id("link-id"));
        link.click();

        // Возвращаемся назад
        driver.navigate().back();

        // Идем вперед
        driver.navigate().forward();

        driver.quit();
    }
}
```

Эта команда полезна для тестирования полного потока навигации пользователя, чтобы убедиться, что переход вперед работает корректно и состояние страницы правильно сохраняется или восстанавливается.

---
### Обновление страницы (Refresh)
Обновление или **перезагрузка текущей страницы — частое действие пользователя,** которое можно автоматизировать **с помощью метода** `navigate().refresh()`. 

Это полезно для тестирования поведения приложения при ручном обновлении страницы.
```java
// Переход по URL
driver.get("https://www.apress.com");

// Выполняем некоторые действия, затем обновляем страницу
driver.navigate().refresh();
```

Обновление страницы **позволяет убедиться, что состояние приложения корректно сбрасывается или что динамический контент загружается ожидаемым образом.** *Это действие критически важно для тестирования страниц с динамическим контентом или данными в реальном времени,* проверяя, что данные обновляются правильно без ручного вмешательства пользователя.

Используя эти команды навигации, вы можете тщательно тестировать навигационные потоки и поведение контента вашего веб-приложения, обеспечивая плавный и интуитивно понятный пользовательский опыт. Методы навигации WebDriver позволяют воспроизводить реальные взаимодействия пользователя, повышая надежность и покрытие ваших автоматизированных тестов.
