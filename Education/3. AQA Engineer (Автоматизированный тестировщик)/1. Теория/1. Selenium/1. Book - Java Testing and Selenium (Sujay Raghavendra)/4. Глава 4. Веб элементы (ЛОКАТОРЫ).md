В этой главе рассматривается важнейшая концепция локаторов веб-элементов в Selenium WebDriver — фундаментальный аспект веб-автоматизации. 

Вы узнаете, **что такое локаторы, почему они незаменимы в Selenium,** и как они связаны с DOM (Document Object Model) — структурой, которая позволяет Selenium взаимодействовать с элементами на странице.

*Далее следует подробное рассмотрение* в**осьми основных типов локаторов в Selenium**: их практическое применение, синтаксис и сценарии использования. От простых ID и Class Name до сложных XPath и CSS-селекторов — каждый тип описан для полного понимания.

В конце рассматриваются типичные проблемы, возникающие при работе с локаторами, и даются рекомендации по их решению для повышения надежности и эффективности скриптов автоматизации.

---
## Что такое локаторы веб-элементов?

> [!NOTE] Локаторы - строка для идентификации и работы с элементом страницы
> **Локаторы веб-элементов** — это инструменты, используемые в веб-автоматизации **для идентификации и взаимодействия с элементами на веб-странице**. 

Этими элементами могут быть 
- кнопки, 
- текстовые поля, 
- чекбоксы, 
- выпадающие меню, 
- ссылки и многое другое. 

**Локаторы указывают на эти элементы внутри DOM**, позволяя автоматизированным скриптам выполнять действия: клики, ввод текста или извлечение информации.

Локаторы обеспечивают корректную работу функциональности веб-приложения и пользовательского интерфейса. Они широко используются в различных сценариях тестирования: функциональном, регрессионном, кросс-браузерном и т.д.

---
### Почему локаторы важны в веб-автоматизации?

1. **Точность автоматизации** — ==позволяют точно идентифицировать элементы на странице==.
    
2. **Кросс-браузерная совместимость** — помогают тестам работать одинаково в разных браузерах.
    
3. **Поддерживаемость** — правильно написанные локаторы проще обновлять при изменении приложения.
    
4. **Переиспользуемость** — локаторы можно использовать в разных тест-кейсах.
    
5. **Масштабируемость** — позволяют покрывать тестами всё больше функциональности.
    
6. **Эффективность** — автоматизация ускоряет процесс тестирования.

## Понимание DOM (Document Object Model)


> [!NOTE] DOM (Document Object Model) - страница (документ) состоит из объектом
> **DOM** — это **структурированное представление содержимого веб-страницы.** 
> 
> Это как карта, которую браузер использует для интерпретации и отображения страницы.

Представьте ***веб-страницу в виде*** **древовидной структуры**. Каждый элемент (заголовки, абзацы, изображения, кнопки) — это **узел (node)** в этом дереве. 

DOM организует эти узлы иерархически, где корневым узлом является HTML-документ.

### Основы HTML и DOM

HTML — стандартный язык для создания веб-страниц. Веб-элементы определяются с помощью тегов.
```ytml
<p>Это абзац.</p>
```
- `<p>` — открывающий тег
    
- `</p>` — закрывающий тег
    
- Текст — содержимое

---
### Атрибуты
Атрибуты предоставляют **дополнительную информацию об элементе** и указываются внутри открывающего тега.
```html
<img src="image.jpg" alt="Красивое изображение">
```

- `src="image.jpg"` — атрибут с именем `src` и значением `image.jpg`
    
- `alt="Красивое изображение"` — атрибут `alt` со значением

Атрибуты `id` и `name` особенно важны для локаторов, так как они позволяют уникально идентифицировать элементы.

---
### Структура DOM-дерева

**Узлы (Nodes):**
- **Корневой узел** — представляет весь документ (`document`)
    
- **Элементные узлы** — представляют HTML-теги
    
- **Атрибутные узлы** — связаны с элементами и содержат информацию о них
    
- **Текстовые узлы** — содержат текст внутри элементов

---
### Отношения в DOM

- **Родитель-ребенок (parent-child)**: элемент вложен в другой. Например, `<div>` содержит внутри `<p>`.
    
- **Соседние (siblings)**: элементы на одном уровне, у одного родителя.

**Пример HTML:**
```html
<!DOCTYPE html>
<html>
<head>
    <title>Наш документ</title>
</head>
<body>
    <div id="content">
        <h1>Добро пожаловать</h1>
        <p>Это абзац.</p>
        <a href="https://example.com">Посетить Example</a>
    </div>
</body>
</html>
```

**Структура:**
- `<html>` — корневой элемент, дети: `<head>` и `<body>`
    
- `<title>` — ребенок `<head>`
    
- Внутри `<body>`: 
	`<div>` — ребенок, который содержит `<h1>`, `<p>`, `<a>` как своих детей
    
- `<a>` имеет атрибут `href="https://example.com"`

# 1. Типы локаторов в Selenium WebDriver

Рассмотрим все основные локаторы: `ID`, `Name`, `XPath`, `CSS-селекторы`, `Link` `Text`, `Partial Link Text`, `Tag Name` и `Class Name`.

## 1. ID Locator
Ищет элемент п**о уникальному атрибуту `id`**. Это самый быстрый и надежный способ, так как ID должен быть уникальным в пределах документа.

**Java-синтаксис:**
```java
WebElement element = 
	driver.findElement(
		By.id("elementId")
	);
```

**HTML-пример:**
```html
<button id="submitBtn">Отправить</button>
```

**Java-код:**
```java
WebElement submitButton = 
	driver.findElement(
		By.id("submitBtn")
	);
```

---
## 2. Name Locator

Ищет элемент **по атрибуту `name`**. Часто используется в формах.

**Java-синтаксис:**
```java
WebElement element = 
	driver.findElement(
		By.name("elementName")
	);
```

**HTML-пример:**
```java
<input type="text" name="username">
```

**Java-код:**
```java
WebElement usernameInput = 
	driver.findElement(
		By.name("username")
	);
```

---
## 3. Link Text Locator - точное совпадения текста

Ищет ссылку **по точному видимому тексту**.

**Java-синтаксис:**
```java
WebElement element = 
	driver.findElement(
		`By.linkText("Точный текст ссылки")
	);
```

**HTML-пример:**
```html
<a href="login.html">Войти</a>
```

**Java-код:**
```java
WebElement loginLink = 
	driver.findElement(
		By.linkText("Войти")
	);
```

---
## 4. Partial Link Text Locator - часть текста (подстрока)

Ищет ссылку **по части видимого текста** (подстроке).

**Java-синтаксис:**

## 5. Tag Name Locator 

Ищет **элементы по имени тега**. ***Возвращает список***, так как тегов может быть много.

```java
List<WebElement> listItems =

	);
```

## 7. XPath Локаторы

XPath — мощный **язык запросов для навигации по XML/HTML-документу.**

---
### Абсолютный XPath

Начинается с корневого узла (`/`) и идет по полному пути. **Хрупкий** — любое изменение структуры сломает локатор.

**Java-синтаксис:**
```java
WebElement element = 
	driver.findElement(
		By.xpath("/html/body/div/p")
	);
```

**HTML-пример:**
```html
<html>
<body>
    <div>
        <p>Текст абзаца</p>
    </div>
</body>
</html>
```

**Java-код:**
```java
WebElement paragraph = 
	driver.findElement(
		By.xpath("/html/body/div/p")
	);
```

---
### Относительный XPath

Начинается с `//` и ищет элемент в любом месте документа. **Более гибкий** и устойчивый к изменениям.

**Java-синтаксис:**
```java
WebElement element = 
	driver.findElement(
		By.xpath("//tag[@attribute='value']")
	);
```

**HTML-пример:**
```html
<div>
    <button id="submit">Отправить</button>
</div>
```

**Java-код:**
```java
WebElement submitButton = 
	driver.findElement(
		By.xpath("//button[@id='submit']")
	);
```

---
### XPath по атрибутам

Ищет элемент по любому атрибуту (`id`, `name`, `type`, кастомным и т.д.).

**Java-синтаксис:**
```java
WebElement element = 
	driver.findElement(
		By.xpath("//tag[@attribute='value']")
	);
```

**HTML-пример:**
```html
<input type="text" name="username">
```

**Java-код:**
```java
WebElement usernameField = 
	driver.findElement(
		By.xpath("//input[@name='username']")
	);
```

---
### Позиционные фильтры в XPath (ИНДЕКСЫ)

Позволяют выбирать элементы по их позиции среди соседей (индексация с **1**).

**Java-синтаксис:**
```java
WebElement element = 
	driver.findElement(
		By.xpath("(//parent/child)[position]")
	);
```

**HTML-пример:**
```html
<ul>
    <li>Элемент 1</li>
    <li>Элемент 2</li>
    <li>Элемент 3</li>
</ul>
```

**Java-код:**
```java
WebElement secondItem = 
	driver.findElement(
		By.xpath("(//ul/li)[2]")
	);
// Выберет второй элемент списка
```

---
### XPath с логическими операторами

#### AND — все условия должны быть истинны
```java
WebElement element = 
	driver.findElement(
		By.xpath("//input[@type='email' and @required]")
	);
```

#### OR — хотя бы одно условие истинно
```java
WebElement element = 
	driver.findElement(
		By.xpath("//button[@id='confirm' or @id='proceed']")
	);
```

#### NOT — исключает элементы, удовлетворяющие условию
```java
WebElement element = 
	driver.findElement(
		By.xpath("//input[@type='checkbox' and not(@checked)]")
	);
// Находит НЕотмеченные чекбоксы
```

## 8. CSS-селекторы (CSS Selectors)

CSS (каскадные таблицы стилей) используются на веб-страницах, написанных на HTML или XML, *для определения отображения элементов: макета, цветов, шрифтов и т.д*. CSS применяет стили к одному или группе элементов. 

**CSS-селекторы** используются ***для идентификации и выбора этих элементов на странице.***

CSS-селекторы имеют различные формы, позволяющие разработчикам точно нацеливаться на элементы. Рассмотрим основные типы селекторов и их использование в Selenium.

---
### Типы CSS-селекторов и их применение

#### Базовые селекторы (Basic Selectors)
Базовые селекторы — самые простые, напрямую нацеленные на элементы по их типу, классу или ID.

---
##### 1. Селектор по типу (Type Selector)
Ищет элементы по имени тега.

**Java-синтаксис:**
```java
By.cssSelector("tagName")
```

**HTML-пример:**
```html
<p>Это абзац.</p>
```

**Java-код:**
```java
WebElement paragraph = 
	driver.findElement(By.cssSelector("p"));
```

---
##### 2. Селектор по классу (Class Selector)

Ищет элементы по атрибуту `class`. Используется точка (`.`) перед именем класса.

**Java-синтаксис:**
```java
By.cssSelector(".className")
```

**HTML-пример:**
```html
<div class="alert">Предупреждение!</div>
```

**Java-код:**
```java
WebElement alertMessage = 
	driver.findElement(By.cssSelector(".alert"));
```

---
##### 3. Селектор по ID (ID Selector)

Ищет элемент по атрибуту `id`. Используется решетка (`#`) перед значением ID.

**Java-синтаксис:**
```java
By.cssSelector("#idValue")
```

**HTML-пример:**
```html
<button id="submitBtn">Отправить</button>
```

**Java-код:**
```java
WebElement submitButton = 
	driver.findElement(
		By.cssSelector("#submitBtn")
	);
```

---
##### 4. Универсальный селектор (Universal Selector)

Выбирает **все** элементы на странице. Используется редко в Selenium из-за низкой производительности.

**Java-синтаксис:**
```java
By.cssSelector("*")
```

---
#### Комбинаторы (Combinators)

Комбинаторы устанавливают отношения между элементами, позволяя выбирать элементы на основе их иерархии.

##### 5. Селектор потомка (Descendant Selector) — пробел

Выбирает элемент, **который является потомком (на любой глубине вложенности)** указанного родителя.

**Java-синтаксис:**
```java
By.cssSelector("родитель потомок")
```

**HTML-пример:**
```html
<div class="container">
    <p>Абзац внутри контейнера.</p>
</div>
```

**Java-код:**
```java
WebElement paragraph = 
	driver.findElement(
		By.cssSelector(".container p")
	);
```

---
##### 6. Селектор прямого ребенка (Child Selector) — `>`

Выбирает элементы, которые являются **непосредственными** **детьми** (**первый уровень вложенности**) указанного родителя.

**Java-синтаксис:**
```java
By.cssSelector("родитель > ребенок")
```

**HTML-пример:**
```html
<ul>
    <li>Элемент списка 1</li>
    <li>Элемент списка 2</li>
</ul>
```

**Java-код:**
```java
List<WebElement> listItems = 
	driver.findElements(
		By.cssSelector("ul > li")
	);
```

---
##### 7. Селектор соседнего брата (Adjacent Sibling) — `+`

Выбирает элемент, который **непосредственно следует** з***а указанным элементом на том же уровне.***

**Java-синтаксис:**
```java
By.cssSelector("предыдущий + следующий")
```

**HTML-пример:**
```html
<h2>Заголовок</h2>
<p>Первый абзац после заголовка.</p>
```

**Java-код:**
```java
WebElement firstParagraph = 
	driver.findElement(By.cssSelector("h2 + p"));
```

---
##### 8. Селектор всех следующих братьев (General Sibling) — `~`

Выбирает **все** элементы, которые следуют за указанным на том же уровне.

**Java-синтаксис:**
```java
By.cssSelector("брат ~ брат")
```

**HTML-пример:**
```html
<h2>Заголовок</h2>
<p>Абзац 1.</p>
<p>Абзац 2, тоже после заголовка.</p>
```

**Java-код:**
```java
List<WebElement> paragraphs = 
	driver.findElements(By.cssSelector("h2 ~ p"));
```

---
#### Селекторы по атрибутам (Attribute Selectors)

Позволяют выбирать элементы на основе их атрибутов и значений.

##### 9. Наличие атрибута (Presence)
Выбирает элементы, у которых есть указанный атрибут (независимо от его значения).

**Java-синтаксис:**
```java
By.cssSelector("[attribute]")
```

**HTML-пример:**
```html
<input type="text" required>
<input type="password">
```

**Java-код:**
```java
List<WebElement> requiredInputs = 
	driver.findElements(
		By.cssSelector("input[required]")
	);
```
---
##### 10. Точное значение (Exact Value)

Выбирает элементы, у которых значение атрибута точно совпадает с указанным.

**Java-синтаксис:**
```java
By.cssSelector("[attribute='value']")
```

**HTML-пример:**
```html
<button type="submit">Отправить</button>
<button type="button">Отмена</button>
```

**Java-код:**
```java
WebElement submitButton = 
	driver.findElement(By.cssSelector("button[type='submit']"));
```
---
##### 11. Частичное совпадение: содержит (Contains) — `*=`

Выбирает элементы, у которых значение атрибута **содержит** указанную подстроку.

**Java-синтаксис:**
```java
By.cssSelector("[attribute*='value']")
```

**HTML-пример:**
```html
<a href="https://example.com/profile/user">Профиль</a>
```

**Java-код:**
```java
WebElement profileLink = driver.findElement(By.cssSelector("a[href*='/profile/']"));
```

---
##### 12. Частичное совпадение: начинается с (Begins with) — `^=`

Выбирает элементы, у которых значение атрибута **начинается** с указанной подстроки.

**Java-синтаксис:**
```java
By.cssSelector("[attribute^='value']")
```

**HTML-пример:**
```html
<a href="https://example.com/dashboard">Дашборд</a>
```

**Java-код:**
```java
WebElement dashboardLink = 
	driver.findElement(
		By.cssSelector(
			"a[href^='https://example.com/dashboard']")
	);
```
---
##### 13. Частичное совпадение: заканчивается на (Ends with) — `$=`

Выбирает элементы, у которых значение атрибута **заканчивается** указанной подстрокой.

**Java-синтаксис:**
```java
By.cssSelector("[attribute$='value']")
```

**HTML-пример:**
```html
<img src="logo.png">
```

**Java-код:**
```java
List<WebElement> pngImages = 
	driver.findElements(By.cssSelector("img[src$='.png']"));
```
---
##### 14. Специфичность (Specificity) — `|=`

Выбирает элементы, у которых значение атрибута **точно равно** указанному значению **ИЛИ** начинается с этого значения, за которым следует дефис (`-`). Часто используется для языковых кодов.

**Java-синтаксис:**
```java
By.cssSelector("[attribute|='value']")
```

**HTML-пример:**
```html
<p lang="en">English Content</p>
<p lang="en-us">American English Content</p>
<p lang="en-gb">British English Content</p>
```

**Java-код:**
```java
// Найти все элементы с lang="en-us" или начинающиеся с "en-"
List<WebElement> americanEnglish = driver.findElements(By.cssSelector("p[lang|='en-us']"));
// Найти все элементы с lang="en" или начинающиеся с "en-"
WebElement generalEnglish = driver.findElement(By.cssSelector("p[lang|='en']"));
```
---
#### Псевдоклассы (Pseudo-Classes)

##### 15. Первый ребенок (`:first-child`)

Выбирает **первый** дочерний элемент внутри родителя.

**HTML-пример:**
```html
<ul id="social-media-links">
    <li>Facebook</li>
    <li>Twitter</li>
    <li>Instagram</li>
</ul>
```
**Java-код:**
```java
WebElement firstLink = 
	driver.findElement(
		By.cssSelector("#social-media-links li:first-child")
	);
// Выберет "Facebook"
```

##### 16. Последний ребенок (`:last-child`)

Выбирает **последний** дочерний элемент внутри родителя.

**Java-код:**
```java

WebElement lastLink = driver.findElement(By.cssSelector("#social-media-links li:last-child"));
// Выберет "Instagram"
```

---
##### 17. N-ный ребенок (`:nth-child(n)`)

Выбирает дочерний элемент по его **порядковому номеру** (индексация с 1).

**Java-код:**
```java
WebElement secondLink = 
	driver.findElement(
		By.cssSelector("#social-media-links li:nth-child(2)")
	);
// Выберет "Twitter" (второй элемент)
```
