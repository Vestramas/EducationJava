# Гиперссылки (Hyperlinks)

Гиперссылки — это веб-элементы, которые помогают пользователям перемещаться по веб-странице или переходить на совершенно другой сайт. 

Этот **навигационный элемент встроен в теги привязки** `<a>` в HTML и представляет собой средство для перемещения по Всемирной паутине или для доступа к данным, которые можно транслировать или загружать.

Гиперссылки в основном ассоциируются с меню, кнопками, изображениями и документами, стилизованными с помощью CSS и JavaScript. 

Гиперссылки также называют просто _ссылками_. Ниже приведен синтаксис создания ссылки:
```html
<a href="URL">текст_ссылки</a>
```
<a href="URL">текст_ссылки</a>


Рассмотрим следующий фрагмент HTML для поиска ссылок в предстоящем контенте:

```html
<div class="container">Selenium
    <a href="java.html" id="java" data-info="javalink">Java</a>
    <a href="python.html" id="python" data-info="pythonlink">Python</a>
    <a href="csharp.html" id="csharp" data-info="csharplink">CSharp</a>
</div>
```

## Поиск гиперссылок

### Гиперссылка по ID
Вы можете **найти гиперссылку, используя атрибут `id` в теге `<a>`.** 

Веб-приложения теперь доступны на многих языках, и `id` — это лучший способ найти ссылку, которая не меняется.

```java
WebElement linkJava = driver.findElement(By.id("java"));
```
**Использование атрибута `id` гарантирует, что будет найден правильный элемент,** даже если другие атрибуты, такие как `name` или `class`, дублируются или являются динамическими.

---
### Гиперссылка по тексту (Link Text)
Этот метод очень полезен **для поиска гиперссылки по её видимому тексту**, когда текст статичен и уникален. 

Этот метод зависит от языка и может не подходить для многоязычных сайтов, если тест не адаптирован для разных языков.

```java
WebElement linkPython = 
	driver.findElement(
		By.linkText("Python")
	);
```

---
### Гиперссылка по частичному тексту (Partial Link Text)
Когда вы *не знаете полный текст* или **только его часть** является стабильной и постоянной, вы можете **использовать частично видимый текст для поиска ссылок.** 

Этот метод подходит для работы с длинными или динамическими текстами ссылок, где только часть остается неизменной.

```java
WebElement linkPartial = driver.findElement(By.partialLinkText("Shar"));
```

----
### Гиперссылка по XPath
Это более гибкий способ поиска гиперссылок, особенно когда вы не можете найти ссылки с помощью более простых локаторов или имеете дело со сложной DOM-структурой.

```java
WebElement linkCSharp = 
	driver.findElement(By.xpath("//a[@id='csharp']"));
```

---
### n-я гиперссылка
Вы можете использовать метод n-й гиперссылки, когда имеете дело со списком похожих или идентичных ссылок, где прямые атрибуты могут быть недоступны или бесполезны. 

Этот подход основан на индексе и может быть неэффективен при изменении порядка или количества ссылок; следовательно, он требует тщательного управления и проверки используемого индекса.

```java
WebElement nthLink = driver.findElements(By.tagName("a")).get(1);
```

---
### Получение всех гиперссылок
Когда вы хотите получить все гиперссылки в приложении для выполнения проверок или взаимодействия с несколькими ссылками последовательно. 

С помощью этого метода вы получаете список всех ссылок, по которому можно итерироваться для выполнения различных операций или проверок на каждой ссылке.

```java
List<WebElement> allLinks = driver.findElements(By.tagName("a"));
```

# 2. Тестирование гиперссылок

Гиперссылки являются основным источником навигации и доступа к различным ресурсам и информации. 

Тестирование гиперссылок важно по нескольким причинам:

- **Точная навигация:** *Тест гарантирует, что пользователи будут направлены в правильное место назначения*. Точная навигация является ключевым звеном пользовательского опыта, направляя пользователей по намеченным маршрутам и обеспечивая доступ к соответствующему контенту и функциям без неправильного направления или ошибок.
    
- **Целостность ссылок:** *Выявление и исправление битых или недействительных ссылок* имеет решающее значение для поддержания целостности гиперссылок.
    
- **Тестирование ссылок на скачивание:** ***Гиперссылки, связанные с данными, которые инициируют загрузку файлов для пользователя, должны гарантировать, что они инициируют правильное действие и предоставляют ожидаемые файлы.***
    
- **Доступность ресурсов:** *Ссылки должны быть протестированы, чтобы убедиться, что они ведут к предполагаемым ресурсам*, гарантируя пользователям правильный доступ к контенту.
    
- **Безопасность:** Тестирование позволяет узнать, не подвергают ли какие-либо ссылки пользователей угрозам безопасности.

## Проверка действительной гиперссылки (Check for a Valid Hyperlink)

Вы можете проверить, действительна ли гиперссылка, получив URL с помощью атрибута `href` и проверив его формат. Вы можете отправить запрос, чтобы проверить доступность ответа. Запросы отправляются по протоколу HTTP к URL, а в ответ возвращается код состояния. Это гарантирует, что гиперссылка не только правильно отформатирована, но и ведет к предполагаемому месту назначения.

```java
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.By;
import java.net.HttpURLConnection;
import java.net.URL;
public class ValidateHyperlink {
    public static void main(String[] args) throws Exception {
        WebDriver driver = new ChromeDriver();
        driver.get("your_website_url");
        
        
        WebElement link = driver.findElement(By.id("java"));
        String href = link.getAttribute("href");
        
        HttpURLConnection connection = 
	        (HttpURLConnection) new URL(href).openConnection();
	        
        connection.setRequestMethod("HEAD");
        int responseCode = connection.getResponseCode();
        
        if (responseCode == 200) {
            System.out.println("Valid Hyperlink");
        } else {
            System.out.println("Invalid Hyperlink");
        }
        driver.quit();
    }
}
```

Коды состояния HTTP — это трехзначные числа, возвращаемые серверами для указания статуса отправленного запроса (см. Таблицу 5-1). Они делятся на пять классов на основе первой цифры.

**Таблица 5-1. Коды состояния HTTP**

|Диапазон|Описание|Примеры кодов|
|---|---|---|
|**1xx (Информационные)**|Запрос получен, и процесс продолжается.|100 Continue, 101 Switching Protocols|
|**2xx (Успешные)**|Запрос успешно получен, понят и принят.|200 OK, 201 Created|
|**3xx (Перенаправление)**|Для завершения запроса необходимо выполнить дальнейшие действия.|300 Multiple Choices, 301 Moved Permanently|
|**4xx (Ошибки клиента)**|Запрос содержит неверный синтаксис или не может быть выполнен сервером.|400 Bad Request, 404 Not Found|
|**5xx (Ошибки сервера)**|Серверу не удалось выполнить действительный запрос.|500 Internal Server Error, 502 Bad Gateway|

## Проверка битых изображений (Check for Broken Images)

При тестировании изображений необходимо найти их (обычно вложенные в тег привязки) и проверить атрибут `src`. Чтобы убедиться, что изображение загружается правильно, вы должны отправить HTTP-запрос к URL изображения и проверить ответ. Если код состояния равен 200, это указывает на то, что изображение доступно и загружается корректно, аналогично проверке ссылок в предыдущем случае. Код состояния 404 указывает на то, что изображение повреждено. Используя этот метод, вы гарантируете, что изображения внутри гиперссылок отображаются правильно и не приводят к появлению иконок битых изображений, сохраняя визуальную целостность веб-страницы.

```java
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.By;
import java.net.HttpURLConnection;
import java.net.URL;
import java.util.List;
public class ValidateImages {
    public static void main(String[] args) throws Exception {
        System.setProperty(
	        "webdriver.chrome.driver", "path/to/chromedriver");
	        
	        
        WebDriver driver = new ChromeDriver();
        driver.get("your_website_url");
        
        
        List<WebElement> images = 
	        driver.findElements(By.tagName("img"));
        
        for (WebElement img : images) {
            String src = img.getAttribute("src");
            
            HttpURLConnection connection = 
	            (HttpURLConnection) new URL(src)
		            .openConnection();
		            
            connection.setRequestMethod("HEAD");
            int responseCode = connection.getResponseCode();
            
            if (responseCode == 200) {
                System.out.println(src + " - Image is valid");
            } else if (responseCode == 404) {
                System.out.println(src + " - Image is broken");
            } else {
                System.out.println(
	                src + " - Image status is " + responseCode);
            }
        }
        driver.quit();
    }
}
```

## Гиперссылки с Data-атрибутами

При работе с динамическими или похожими стандартными атрибутами вам необходимо искать гиперссылки, используя пользовательские _data-атрибуты_, что обеспечивает гибкий и настраиваемый подход. 

Эти пользовательские атрибуты предоставляют стабильный и уникальный способ поиска гиперссылок для различных взаимодействий и проверок, гарантируя, что вы найдете правильную гиперссылку в тесте.

```java
WebElement dataLink = 
	driver.findElement(
		By.cssSelector("a[data-info='pythonLink']")
	);
```
