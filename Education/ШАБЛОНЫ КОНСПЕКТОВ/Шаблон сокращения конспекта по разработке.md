Вот тебе даны несколько файлов конспекта по теме ...

Выдели из всех этих конспектов информацию, и распредели по структуре описанной ниже. Информацию по этой технологии/возможности должна быть максимально полной, для понимания того как написать полноценного работоспособного приложения/реализации технологии в проекте.

Вот сама структура:
# 1. Описание технологии/темы
Здесь опиши суть технологии/темы, для чего она нужна, как работает, ключевые ее моменты. Если уместно рисуй UML-диаграммы.

# 2. Зависимости необходимые для этой технологии/темы (опционально)
Опиши зависимости, которые нужно подключить к проекту, чтобы использовать эту технологию

# 3. КАРКАСНАЯ конфигурация и код
## 3.1 Ключевые классы
Здесь опиши ключевые классы, для построения жизнеспособного приложения: зачем каждый из них нужен, пример кода как он конфигурируется и где они используется 
## 3.2 Каркасная конфигурация
Затем, опиши принцип написания КАРКАСНОГО кода по этой технологии, то с чего всегда стоит начинать разработку. 
- Базовая конфигурация (properties,yaml), 
- Каркасный код, то что обязательно должно быть чтобы эта технология работала
- Основные "структурные части" (блоки кода/методы/дополнительные классы/бины, которые добавляются к базовой конфигурации, и описание того что они делают)

Все что описано в этом разделе должно быть максимально готово к использованию. 

## 3.3 Аннотации (опционально)
Опиши, какие аннотации обычно используются в конфигурации, и где они устанавливаются.

Опиши их основные параметры для построения работоспособного приложения. И приведи примеры их использования

# 3. Дополнительная конфигурация (опционально)
Здесь опиши альтернативные способы конфигурации: параметры, классы, которые меняют или добавляют функционал в каркасной конфигурации. И описание того что именно они добавляют/меняют
